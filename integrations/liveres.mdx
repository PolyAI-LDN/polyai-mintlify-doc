LiveRes by Zonal is a comprehensive restaurant booking and table management system. This integration allows your PolyAI voice assistant to check availability, create reservations, modify bookings, and cancel reservations directly through the LiveRes API.

## Prerequisites

Before you begin, you need:

- LiveRes account with API access
- API credentials (API key and secret)
- LiveRes venue ID
- PolyAI account with integration permissions

## Configuration

To configure the LiveRes integration, you'll need to provide the following parameters:

### Required parameters

- **API key**: Your LiveRes API authentication key
- **API secret**: Your LiveRes API secret for secure communication
- **Venue ID**: The unique identifier for your restaurant venue in LiveRes
- **Base URL**: The LiveRes API endpoint (typically provided by Zonal)

### Optional parameters

- **Default party size**: Default number of guests if not specified
- **Booking duration**: Default reservation duration in minutes
- **Time zone**: Venue time zone for accurate booking times

## Features

The LiveRes integration supports the following operations:

### Check availability

Query available time slots for a specific date, time, and party size.

### Create reservation

Book a table with customer details including:
- Guest name
- Contact information (phone/email)
- Party size
- Date and time
- Special requests or notes

### Modify reservation

Update existing bookings:
- Change date or time
- Adjust party size
- Update customer details
- Add special requests

### Cancel reservation

Cancel existing reservations using the booking reference number.

### Retrieve reservation

Look up booking details using:
- Booking reference
- Customer phone number
- Customer email

## Error handling

The integration handles common scenarios:

- **No availability**: Suggests alternative times
- **Invalid booking reference**: Prompts for correct information
- **API timeout**: Retries with exponential backoff
- **Venue closed**: Informs customer of opening hours

## Best practices

- Always confirm booking details with the customer before finalizing
- Provide booking reference numbers for future modifications
- Set appropriate timeout values for API calls
- Monitor integration logs for any issues
