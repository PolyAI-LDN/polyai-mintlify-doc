Design My Night is a leading venue booking and management platform for restaurants, bars, and entertainment venues. This integration enables your PolyAI voice assistant to manage reservations and bookings through the Design My Night API.

## Prerequisites

Before you begin, you need:

- Design My Night account with API access
- API credentials (client ID and secret)
- Venue ID from Design My Night
- PolyAI account with integration permissions

## Configuration

To configure the Design My Night integration, you'll need to provide the following parameters:

### Required parameters

- **Client ID**: Your Design My Night API client identifier
- **Client secret**: Your API secret for authentication
- **Venue ID**: The unique identifier for your venue
- **API endpoint**: Design My Night API base URL

### Optional parameters

- **Default booking duration**: Standard reservation length in minutes
- **Minimum party size**: Minimum number of guests per booking
- **Maximum party size**: Maximum number of guests per booking
- **Advance booking days**: How far in advance customers can book

## Features

The Design My Night integration supports the following operations:

### Search availability

Check available time slots based on:
- Date and time
- Party size
- Specific areas or table types
- Special requirements

### Create booking

Make reservations with:
- Customer name and contact details
- Party size
- Date and time
- Special requests
- Dietary requirements
- Occasion (birthday, anniversary, etc.)

### Modify booking

Update existing reservations:
- Change date or time
- Adjust party size
- Update customer information
- Add or modify special requests

### Cancel booking

Cancel reservations using the booking reference or customer details.

### Retrieve booking

Look up existing bookings by:
- Booking reference number
- Customer email
- Customer phone number
- Date range

## Booking flow

A typical booking conversation flow:

1. Customer requests a reservation
2. Assistant checks availability
3. Assistant presents available options
4. Customer selects preferred time
5. Assistant collects customer details
6. Assistant confirms booking
7. Booking reference provided to customer

## Error handling

The integration handles various scenarios:

- **Fully booked**: Suggests alternative times or dates
- **Invalid party size**: Requests valid number of guests
- **Past date/time**: Prompts for future date
- **Venue closed**: Provides opening hours information
- **API errors**: Graceful fallback with retry logic

## Best practices

- Confirm all booking details before finalizing
- Provide clear booking references
- Set realistic timeout values
- Handle peak times with appropriate messaging
- Monitor booking success rates
