**Builder MCP** is PolyAI's authenticated MCP server for building agents. Point an MCP client — [Cursor](https://cursor.com), [Claude Code](https://www.claude.com/product/claude-code), Claude Desktop, or Codex — at it, and you can create, test, and deploy PolyAI agents from your IDE without writing a single API call. The client discovers every tool and its input schema automatically.

## Builder MCP vs Agent Studio MCP integrations

These are opposite directions, and keeping them straight is the fastest way to know which docs you need:

| | **Builder MCP** | **Agent Studio MCP integrations** |
| --- | --- | --- |
| **Direction** | An MCP client connects **in** to PolyAI | Your PolyAI agent connects **out** to a third-party server |
| **Who acts as the client** | Your IDE / AI coding tool | Your live PolyAI agent |
| **What it's for** | Build, test, and deploy agents | Give a live agent extra tools during a conversation |
| **Docs** | This page | [Agent Studio MCP integrations](/mcp/agent-studio-integrations) |

If you're managing agents from an IDE, you want Builder MCP. If you're extending what a running agent can do mid-conversation, you want [Agent Studio MCP integrations](/mcp/agent-studio-integrations).

## Why use it

- **No API plumbing.** The client discovers every tool and input schema from the server — you don't hand-write requests, hosts, or headers.
- **Conversational workflows.** Ask your assistant to "branch, update the greeting, test it, and publish to staging," and it chains the tool calls.
- **Enterprise lifecycle built in.** Variants, environments, and deployments are first-class — see the [build → test → deploy walkthrough](/mcp/builder/walkthrough).
- **One key, every tool.** A single [workspace-scoped API key](/mcp/authentication) authenticates the whole server.
