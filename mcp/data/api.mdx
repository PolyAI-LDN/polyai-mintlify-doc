The **Data API** is one read-only API for conversation data across the platform — transcripts, conversation search, a metrics catalog, and analytics. [Data MCP](/mcp/data/introduction) is built on it: the MCP server exposes these same endpoints as tools. Call the API directly when you're integrating outside an MCP client.

## Base URL

Endpoints live under `/public_api/api/v1`, exposed through the PolyAI Developer API. They're region-specific — use the base URL for the region your account lives in (self-serve accounts use Studio).

| Region | Base URL |
| ------ | ---------------------------------------------- |
| US     | `https://api.us.poly.ai/public_api/api/v1`     |
| UK     | `https://api.uk.poly.ai/public_api/api/v1`     |
| EU     | `https://api.eu.poly.ai/public_api/api/v1`     |
| Studio | `https://api.studio.poly.ai/public_api/api/v1` |

## Authenticate

Send your **account API key** in the `X-API-KEY` header — the same key you'd use for [Data MCP](/mcp/data/introduction). See [Authentication](/mcp/authentication).

```bash
curl "https://api.us.poly.ai/public_api/api/v1/metrics?project_id=your-project" \
  -H "X-API-KEY: $POLYAI_API_KEY"
```

## Endpoints

| Method | Path | What it does | Permission |
| ------ | ---- | ------------ | ---------- |
| `GET`  | `/metrics` | List the filterable metrics for a project. | Read conversations |
| `GET`  | `/transcripts/search` | Full-text search over transcript turns, with surrounding context. | Read conversations |
| `GET`  | `/transcripts/conversation/{conversation_id}` | Fetch all turns of one conversation. | Read conversations |
| `POST` | `/conversations/search` | Search conversations by metric filters, with sorting and pagination. | Read conversations |
| `POST` | `/analytics` | Aggregate a metric over time into buckets. **Experimental.** | Read analytics |

The request parameters mirror the matching [Data MCP tools](/mcp/data/capabilities) — for example, `/conversations/search` takes the same filters, sort, and pagination as the `search-conversations` tool.

<Note>
The `/analytics` endpoint is **experimental** — its schema may change.
</Note>

## Errors

| Status | Meaning |
| ------ | ------- |
| `400`  | Request validation error. |
| `401`  | Missing or invalid `X-API-KEY`. |
| `403`  | Authenticated, but missing the required permission. |
| `404`  | Not found or empty result — also returned when a credential can't access the account. |
| `422`  | Unsupported filter. |
| `502`  | An upstream data store failed. |
| `500`  | Unexpected internal error. |
