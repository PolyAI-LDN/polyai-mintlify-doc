PolyAI hosts its own MCP server so you can drive the platform from any [MCP](https://modelcontextprotocol.io/) client — [Claude Code](https://www.claude.com/product/claude-code), Cursor, or a custom agent — instead of writing direct API calls. The server exposes PolyAI's platform APIs as MCP **tools**: the client discovers them automatically and calls them on your behalf.

<Note>
This is the opposite direction from [connecting external MCP servers](/mcp/connect-external-servers). Here, an MCP client connects **in** to PolyAI. There, your agent connects **out** to a third-party server.
</Note>

## What it exposes

The server wraps PolyAI's build-and-monitor APIs and presents each operation as a tool. Broadly, that covers:

| Capability | Backed by |
| --- | --- |
| Create, duplicate, and manage agents, branches, and deployments | [Agents API](/api-reference/agents/introduction) |
| Edit behavior, knowledge base, variants, and real-time config | [Agents API](/api-reference/agents/introduction) |
| Manage connectors, phone numbers, and the voice library | [Agents API](/api-reference/agents/introduction) |
| Create alert rules and read active alerts | [Alerts API](/api-reference/alerts/introduction) |

See [Available tools](/mcp/server/tools) for the full breakdown, and the [API reference](/api-reference/introduction) for the underlying request and response shapes.

## Why use it

- **No API plumbing.** The client discovers every tool and its input schema from the server — you don't hand-write requests, base URLs, or auth headers per call.
- **Conversational workflows.** Ask an MCP-capable assistant to "create a branch, update the greeting, and publish to staging," and it chains the tool calls for you.
- **One key, every tool.** A single [workspace-scoped API key](/secrets/api-keys) authenticates the whole server.

## Get started

<Steps>
  <Step title="Get an API key">
    Create a workspace-scoped [API key](/secrets/api-keys) in Agent Studio. The full value is shown only once — copy it.
  </Step>
  <Step title="Find your regional endpoint">
    PolyAI runs the MCP server in every region. Pick the host that matches your workspace — see [Endpoints & authentication](/mcp/server/endpoints).
  </Step>
  <Step title="Connect your client">
    Add the server to Claude Code, Cursor, or your own client. See [Connect a client](/mcp/server/connect).
  </Step>
</Steps>
