The MCP server exposes each PolyAI platform operation as a tool. Your client discovers them automatically on connection — you don't configure tools by hand. The groups below mirror the underlying [platform APIs](/api-reference/introduction).

<Note>
Tool names and input schemas come from the server. To see exactly what your client discovered, ask it to list its available tools, or query the endpoint directly (see [Verify the connection](/mcp/server/endpoints#verify-the-connection)).
</Note>

## Build

Backed by the [Agents API](/api-reference/agents/introduction).

| Area | What you can do |
| --- | --- |
| **Agents** | List, create, duplicate, and delete agents |
| **Branches** | Create, merge, and delete branches |
| **Behavior** | Read and update agent behavior rules |
| **Knowledge base** | List, create, update, and delete knowledge base topics |
| **Variants** | Manage variant attributes and variants |
| **Real-time config** | Read and update config pages per environment |

## Deploy

Backed by the [Agents API](/api-reference/agents/introduction).

| Area | What you can do |
| --- | --- |
| **Deployments** | Publish drafts, promote between environments, and roll back |
| **Connectors** | List, update, and delete connectors |
| **Phone numbers** | Import, reassign, and delete phone numbers |
| **Outbound calls** | Trigger outbound calls and check their status |
| **Voice library** | List, register, and synthesize voices |

## Monitor

Backed by the [Alerts API](/api-reference/alerts/introduction).

| Area | What you can do |
| --- | --- |
| **Alert rules** | Create, list, update, and delete alert rules |
| **Active alerts** | Read currently firing alerts |

## Limiting which tools are exposed

If your client should only see a subset of tools, restrict discovery with the `tools` query parameter on the endpoint URL:

```
https://api.us.poly.ai/builder-mcp?tools=list-agents,create-branch
```

Only the listed tools are discovered. This keeps the client's context focused when you don't need the full surface.
