PolyAI widgets put your agent on your website in minutes. From **Deployment > Widget** in Agent Studio, create a widget, choose **Phone** or **Chat** as the type, brand it, copy the script tag, and ship it.

Phone and Chat widgets are built from the same unified Widget surface. The editor, styling controls, and embed model are the same. The widget type just determines whether visitors get a voice call or a text conversation.

![Add widget dialog with Phone vs Chat widget type selector](/images/widgets/add-widget-dialog.png)

## Widget types

<CardGroup cols={2}>
  <Card title="Phone widget (Polyphone)" icon="phone" href="/polyphone/introduction">
    Click-to-call voice on your website. Visitors talk to your agent directly in the browser over WebRTC – no number to dial, no app to install.
  </Card>
  <Card title="Chat widget (Webchat)" icon="message" href="/webchat/introduction">
    Text chat on your website. Visitors type to the same agent that answers your phone calls, with channel-specific styling and safety filters.
  </Card>
</CardGroup>

You can run both on the same site. Many teams ship a Phone widget on high-intent pages and a Chat widget elsewhere.

## How they compare

The editor is shared, so brand, copy, and disclaimer settings work the same way for both types. The differences are in the runtime experience and the JavaScript SDK each one exposes.

### Configuration

| | Phone | Chat |
|---|---|---|
| Created from **Deployment > Widget** | ✓ | ✓ |
| Widget name, website URL, variant, environment | ✓ | ✓ |
| Agent name | ✓ | ✓ |
| Color theme (Phone: Dark / Light) | ✓ | – |
| Primary color (Chat: hex) | – | ✓ |
| Disclaimer message + privacy & terms URLs | ✓ | ✓ |
| Consent button ("I consent and start chat") | – | ✓ |
| Header text | – | ✓ |
| Header image (logo) and agent avatar | – | ✓ |
| Bubble prompt style defaults | – | ✓ |

### Runtime experience

| | Phone | Chat |
|---|---|---|
| Modality | Voice (WebRTC, Opus 48 kHz) | Text |
| Launcher button on the host page | ✓ | ✓ |
| Mobile fullscreen takeover | ✓ | ✓ |
| Call states (connecting, in-call, ended) | ✓ | n/a |
| Mute / unmute, voice activity, call timer | ✓ | n/a |
| Microphone permission flow | ✓ | n/a |
| Cross-tab call lock (one active call per browser) | ✓ | n/a |
| Conversation transcript, attachments, suggestion chips | n/a | ✓ |
| Maintenance mode banner | n/a | ✓ |
| Configurable render mode (`default` or `fullscreen`) | n/a | ✓ |
| Header / minimize / close / privacy / terms toggles | n/a | ✓ |
| Prompt bubble next to the launcher | n/a | ✓ |
| Launcher animation (`bounce`, `grow`, `pulse`) | n/a | ✓ |
| iOS WebView mode (`data-platform="ios"`) | n/a | ✓ |
| Hosted **Test widget** preview link | ✓ | – (use in-editor preview) |

### JavaScript SDK

Both widgets expose a global API on the host page so you can drive them from your own code. The two surfaces don't overlap – Phone exposes telephony events, Chat exposes messaging events.

| | Phone (`PolyphoneAPI`) | Chat (`WebchatAPI`) |
|---|---|---|
| Lifecycle | `onReady` | `onReady`, `destroy`, `getConfig`, `isFullscreen` |
| Open / close | – (launcher only) | `openWidget`, `closeWidget`, `endSession`, `onOpen`, `onClosed`, `onSessionEnded` |
| Connection | `connect`, `disconnect`, `connectAndCall`, `isConnected`, `onConnecting`, `onConnected`, `onDisconnected`, `onRegistered`, `onUnregistered` | n/a (session managed automatically) |
| Call control | `startCall`, `endCall`, `mute`, `unmute`, `getCurrentSession`, `onRinging`, `onAnswered`, `onEnded`, `onFailed`, `onMuted`, `onUnmuted`, `onTranscription`, `onCaptureAudio`, `onAudioEnded` | n/a |
| Context / data | `sendInfo`, `onInfoSent`, `onInfoReceived` | `identifyUser`, `setContext` |
| Messaging | n/a | `onMessageSent`, `onMessageReceived` |
| Launcher | n/a | `showIcon`, `hideIcon`, `animateIcon`, `showBubble` |
| Errors | `onError` | (surfaces via `onSessionEnded` / fallback page) |
| Removing handlers | `off(eventName, handler)` | `off(eventName, handler)` |
| Script-tag config | – | `data-platform`, `data-render-mode`, `data-auto-open`, `data-show-icon`, `data-show-header` |

[Configure widget](/widgets/configure) and [Install on your site](/widgets/install) cover both types in one walkthrough, with Phone-specific and Chat-specific notes called out inline.

## Prerequisites

Before you create a widget you'll need:

- The web domain(s) where you'll embed it.
- Access to your website's HTML or your tag manager.
- A published variant for the agent that will respond.
- An **HTTPS** site. Required for Phone (microphone access), recommended for Chat.

For **Chat** widgets, complete [Chat configuration](/webchat/chat-configuration) (LLM, greeting, safety filters) before creating the widget.
