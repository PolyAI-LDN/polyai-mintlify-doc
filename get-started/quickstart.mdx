<Info>
  Before you begin, make sure you’ve signed up for an account. You can find detailed instructions [here](/get-started/account-creation/overview).
</Info>

Follow these steps to create an assistant on our platform:

### 1. Start the creation process

* After logging in, click **“Create Assistant”** to start the process.

![create-assistant](/images/02-02-01-01-create-assistant.png)

### 2. Set a name, response language, voice, and greeting line.

* Full page: [**Voice**](/voice/introduction)

* Set the general information for the assistant, including **assistant name**, **response language**, **voice**, and **the welcome greeting**.

<Info>
  You can also  [here](/get-started/account-creation/overview).
</Info>

* After completing these steps, click **"Next"**.

![general](/images/02-02-01-02-general.png)

### 3. Select personality, role, and additional info

* Next, select the **personality**, **role**, and enter the **additional info** for your assistant.

* After completing these steps, click **"Next"**.

![about](/images/02-02-01-03-about.png)

### 4. Select knowledge topics

* Full page: [**Knowledge base**](/knowledge-base/introduction)

* Knowledge refers to the information and context the assistant will have access to.

![knowledge](/images/02-02-01-04-knowledge.png)

### 5. \[Optional] Select a number for telephony integration

* Full page: [**Telephony Integration**](/telephony/introduction) section for more details.

* You may buy a number or connect an existing number to interact with your assistant. You may also skip this step and decide later.

![telephony](/images/02-02-01-05-telephony.png)

### 6. \[Advanced] Integrate functions into your knowledge base

* Full page: [**Functions**](/functions/introduction)

* **What they do**: Functions allow you to add dynamic, programmable behavior to your assistant, handling external API calls, retrieving specific data, and making decisions based on real-time context.

* Example use case: An assistant retrieving shipping information for a customer query about an order.

![functions](tba)

### 7. \[Advanced] Add a flow to your project

* Full page: [**Flows**](/flows/introduction)

* **What they do**: Flows are designed for managing multi-step conversations, like booking a table at a restaurant, resetting a password, or collecting feedback.

* Example use case: A customer reservation flow asking for the date, time, and number of people before confirming the booking.

![flows](tba)

By following these steps, you can create a personalized and effective assistant tailored to your needs.
