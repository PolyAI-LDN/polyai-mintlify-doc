Agent Studio is PolyAI's visual interface for building, deploying, and monitoring voice and chat agents. Configure knowledge, voice, integrations, and conversation flows without writing code. Most agents run on [Raven](/behavior/models/raven), PolyAI's proprietary LLM; [other models](/behavior/models/model-use) are also supported.

<div className="full-only">
<Note>
  If you are a developer looking to extend your agent with Python functions and API integrations, see [Extend with code](/extend/introduction) instead.
</Note>
</div>

For how Agent Studio fits alongside the ADK and REST APIs, see the [platform overview](/platform/introduction).

## Navigate the interface

The Agent Studio sidebar groups related features into focused sections. See [What's changed in Agent Studio](/releases/whats-changed-agent-studio) if you're coming from the previous Build/Channels/Configure layout.

<Note>
  Some sections are permission-gated or feature-flagged. If a section is missing, check your [role and permissions](/user-management/access-control-scope).
</Note>

### Studio Assistant

The natural-language builder *and* analyst. Describe what you want your agent to do and Studio Assistant configures it; ask questions about live performance and [Smart Analyst](/smart-analyst/introduction) answers from sampled conversations. See [Studio Assistant](/studio-assistant/introduction).

### Analytics

Review conversation data and track agent performance.

| Sub-section | What it does |
|---|---|
| [**Conversations**](/analytics/conversations/introduction) | Browse transcripts, listen to recordings, and diagnose individual calls |
| [**KPIs**](/analytics/kpis/introduction) | Track standard and custom metrics over time |
| [**Dashboards**](/analytics/dashboards/introduction) | Standard, safety, and custom dashboards |
| [**PolyScore**](/analytics/polyscore) | PolyAI's overall quality score |
| [**CSAT**](/analytics/csat/introduction) | In-call and webchat satisfaction surveys |

For natural-language Q&A over sampled conversations, use [Smart Analyst](/smart-analyst/introduction) inside Studio Assistant.

### Behavior

Define how your agent thinks and responds.

| Sub-section | What it does |
|---|---|
| [**General**](/behavior/general/agent) | Agent name, role, additional info, and behavior prompt |
| [**Language**](/behavior/language/multilingual) | Primary language, additional languages, translation overrides |
| [**Guardrails**](/behavior/guardrails/introduction) | Platform and custom guardrails |
| [**Models**](/behavior/models/model-use) | LLM selection and Raven configuration |

Advanced settings (safety filters, phrase filters, runtime configuration) sit behind an **Advanced** affordance.

### Knowledge

Give your agent the information it needs to answer questions.

| Sub-section | What it does |
|---|---|
| [**FAQs**](/knowledge/faqs/introduction) | FAQ topics with trigger questions, answers, and actions (previously Managed topics) |
| [**Sources**](/knowledge/sources/introduction) | Connected sources — URLs, PDFs, Zendesk, etc. (previously Connected knowledge) |
| [**Variants**](/knowledge/variants/introduction) | Per-entity knowledge variations |

### Flows, Tools, Simulation testing, Real-time config

| Section | What it does |
|---|---|
| [**Flows**](/flows/introduction) | Design multi-step conversation workflows |
| [**Tools**](/tools/introduction) | Reusable functions and API actions |
| [**Simulation testing**](/simulation-testing/introduction) | **Tests** (the test list) and **Test runs** (run history) for your agent (previously Test suite) |
| [**Real-time config**](/real-time-config/introduction) | Configuration data per environment, plus the schema builder (previously Real-time configuration + Configuration Builder) |

### Voice

Everything voice-specific.

| Sub-section | What it does |
|---|---|
| [**Settings**](/voice-channel/agent) | Voice per language, greeting, disclaimer, style prompt, ringing tone (merges Agent voice and Voice configuration) |
| [**Numbers**](/voice-channel/numbers/introduction) | Sandbox, Pre-release, and Live numbers and SIP addresses |
| [**Handoffs**](/voice-channel/handoffs) | Voice handoff destinations |
| [**Message templates**](/voice-channel/message-templates) | Voice message templates (previously SMS) |
| [**Audio library**](/voice-channel/audio-library) | Cached audio files (previously Audio management) |

Voice advanced settings group **Model**, **Call**, and **Speech** options behind an Advanced affordance.

### Messaging

Style prompt, greeting, and other text-channel settings. The chat widget moves out to [Widgets](/widgets/introduction).

### Integrations

| Tab | What it does |
|---|---|
| [**Apps**](/integrations/introduction) | App integrations (NICE, Twilio, Genesys, and others) |
| [**APIs**](/integrations/api/introduction) | API connections |
| [**MCP**](/integrations/introduction#mcp) | MCP integrations |

### Deployments

Promote your agent through Draft, Sandbox, Pre-release, and Live; promote and roll back. Version history, event types, authors, timestamps, and side-by-side diffs all live directly on the [Deployments](/environments-and-versions/introduction) page.

### Widgets

Configure the chat widget. See [Widgets](/widgets/introduction).

### Settings & admin

Project settings, user management, API keys, call data, and legal. See [Settings](/settings/introduction).
