The top toolbar runs across every page in Agent Studio. It surfaces who and what you're working on, and the actions you take most often — switching agents, swapping branches, opening Studio Assistant, previewing the agent, and promoting changes.

<Note>
The toolbar was redesigned for the [redesigned for 24 June 2026](/get-started/whats-new). The new toolbar adds a project selector, a Studio Assistant entry point, and a Share button, and consolidates every release action behind a single **Promote** button.
</Note>

## What's on it

| Control | Where | What it does |
|---------|-------|--------------|
| **Agent name** | Left | Shows the agent you're editing. Click to rename. |
| **Project selector** | Left, next to the agent name | Switch between projects in your workspace without going back to the workspace home. |
| **Branch selector** | Left, next to the project selector | Switch between Draft, Sandbox, Pre-release, Live, and any feature branches. See [Branches](/environments-and-versions/introduction). |
| **Chat with Studio Assistant** | Center / right | Opens the [Studio Assistant](/studio-assistant/introduction) chat panel — describe a change in natural language, ask the agent a question, or run [Smart Analyst](/smart-analyst/introduction) against your conversation data. |
| **Share** | Right | Generate a shareable preview link for the current branch. Useful for getting feedback from stakeholders without giving them workspace access. |
| **Preview agent** | Right | Open a live test conversation with the current branch in the Preview panel. Voice or chat depending on the project. |
| **Promote** | Right (primary button) | The single entry point for every release action. See [Promote](#promote-button) below. |
| **Global search** | Anywhere (keyboard shortcut) | Press <kbd>⌘ K</kbd> (Mac) or <kbd>Ctrl K</kbd> (Windows/Linux) to search across pages, settings, and knowledge from any screen. |

## Promote button

**Promote** consolidates every release action that used to live in different places in the old UI. Open it to see:

- **Release and branch history** — every version released to Sandbox, Pre-release, or Live, with author, timestamp, and the commit it points at.
- **Compare with Live** — side-by-side diff of the current branch against what's running in Live. See [Compare versions](/environments-and-versions/diffs).
- **Move to Staging** — promote the current branch into Sandbox or Pre-release.
- **Publish to Live** — promote the current Pre-release version into Live.
- **Run A/B test** — split traffic between two versions. See [A/B testing](/environments-and-versions/ab-testing).
- **Discard changes** — revert the branch back to its last released state.

Release actions require the right [role permission](/user-management/access-control-scope). If the option you need is greyed out, check your role.

## What was removed

- **Add/view numbers** has moved off the toolbar. Manage numbers under [Voice > Numbers](/voice-channel/numbers/introduction).
- The standalone **release history**, **compare**, **publish**, and **discard** buttons have been consolidated into [Promote](#promote-button).
