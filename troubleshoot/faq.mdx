Common questions about managed topics, rules, and agent configuration.

## Managed topics

Full page: [**Managed Topics**](/managed-topics/introduction)

<AccordionGroup>
  <Accordion title="How should I structure a topic?" icon="sitemap">
    Each topic should have:

    - **A clear name**: Use short, descriptive titles like "Refund policy" or "Store hours."
    - **Example queries**: Add 5-10 user questions to improve retrieval accuracy.
      - Example for *Refund policy*:
        - "How do I get a refund?"
        - "Can I return a product for a refund?"
    - **Content and actions**: Separate what the agent says (content) from what it does (actions).
      - Example for *Payment issues*:
        - **Content**: "Did you use a credit card or PayPal?"
        - **Action**: Trigger SMS with billing team details.

    For more, see [how to set up actions](/managed-topics/how-to-setup-action/introduction).
  </Accordion>
  <Accordion title="Should I use smaller or larger topics?" icon="scale-balanced">
    - **Larger topics**: Better for agents using newer LLM models, because they can handle more context.
    - **Smaller topics**: Easier for reporting and analysis, or for agents using older models.

    Balance scope and specificity based on your use case.
  </Accordion>
  <Accordion title="How can I handle clarification questions?" icon="circle-question">
    Add a disambiguation prompt in the content.

    **Example:**
    - **Topic**: "Booking issues"
    - **Content**: "Can you confirm if the booking was made online or over the phone?"
  </Accordion>
  <Accordion title="How do I manage out-of-scope queries?" icon="ban">
    Create an "Out-of-scope" topic or add instructions in [global rules](/agent-settings/rules).

    **Example response**: "I'm sorry, I can't help with that. Please contact support."
  </Accordion>
</AccordionGroup>

## Rules

Full page: [**Rules**](/agent-settings/rules)

<AccordionGroup>
  <Accordion title="What are global rules?" icon="globe">
    Global rules set consistent agent behavior across interactions. Use them for tone, scope, and task-specific instructions.

    **Examples:**
    - "Always remain professional."
    - "Only answer questions about `<service>`."
  </Accordion>
  <Accordion title="Should I include examples in global rules?" icon="lightbulb">
    Yes. Examples clarify how the agent should behave.

    **Example rule:**
    "If the user asks for repetition, respond with: 'Let me repeat that for you.'"
  </Accordion>
  <Accordion title="How should I plan for risky scenarios?" icon="shield-halved">
    Identify high-risk situations (like refunds or emergencies) and add clear rules or topics.

    **Example for refunds:**
    "Route all refund-related queries to a support specialist."

    <Warning>
      Always test risky scenario handling in [sandbox](/environments-and-versions/introduction) before deploying to production.
    </Warning>
  </Accordion>
</AccordionGroup>

## Actions

Full page: [**Actions**](/managed-topics/how-to-setup-action/introduction)

<AccordionGroup>
  <Accordion title="What are actions?" icon="bolt">
    Actions let the agent perform tasks like sending SMS, calling functions, or triggering handoffs. They are defined in managed topics, directly under the content field.

    **Example use case:** A user asks about refunds, and the agent sends an SMS with a refund link.
  </Accordion>
  <Accordion title="Can I combine multiple actions?" icon="layer-group">
    Yes. You can trigger multiple actions for a single topic, like sending an SMS and initiating a handoff.
  </Accordion>
  <Accordion title="What happens if an action fails?" icon="triangle-exclamation">
    Set up fallback actions, like redirecting to a global FAQ or retrying with a different template.

    <Tip>
      Use the [conversation diagnosis](/analytics/conversations/diagnosis) tool to debug failed actions.
    </Tip>
  </Accordion>
</AccordionGroup>

## Personality and role

Full page: [**About the agent**](/agent-settings/about)

<AccordionGroup>
  <Accordion title="How do I set the agent's personality?" icon="face-smile">
    Avoid generic prompts like "You are a helpful agent." Use global rules or KB topics to set tone and boundaries.

    **Example:** "Adopt a professional tone. Answer only questions about `<service>`."
  </Accordion>
  <Accordion title="Is defining a role necessary?" icon="user-tag">
    It's helpful for clarity. Add this in the "About" section or as a global rule.

    **Examples:**
    - "You are a virtual agent for \[Brand], focused on customer support."
    - "You are a helpful hotel concierge, focused on resolving customer problems and managing reservations."
  </Accordion>
</AccordionGroup>

## Technical considerations

<AccordionGroup>
  <Accordion title="Can prompting affect how the agent talks or understands speech?" icon="microphone">
    No. Prompts do not influence automatic speech recognition (ASR) or text-to-speech (TTS). These systems are independent.

    <Note>
      To customize speech recognition, use [ASR biasing](/flows/asr-biasing) and [keyphrase boosting](/speech-recognition/introduction). To customize voice output, see [voice configuration](/voice/introduction).
    </Note>
  </Accordion>
</AccordionGroup>
