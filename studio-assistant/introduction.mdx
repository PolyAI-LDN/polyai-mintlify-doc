Studio Assistant is the AI copilot built into PolyAI Studio. Tell it what you want — *"Add a flow that takes a reservation"*, *"What can my agent do right now?"*, *"Fix the booking flow"* — and it reads your project, plans the work, and (when you say go) makes the changes on a branch you can review before merging.

<Note>
Studio Assistant works on top of the same building blocks you use manually — flows, topics, entities, functions, handoffs, and agent settings. Every edit lands on a **branch**, not on `Main`. Review, revise, or merge on your terms.
</Note>

## A typical session, end to end

The arc you'll repeat hundreds of times:

<Steps>
  <Step title="Start from the home screen">
    A blank chat panel with a prompt input and templates grouped by intent — **Build**, **Test**, **Ask**, **Analyze**. Click a template, edit it, or just type your own request. The cost indicator next to the send button shows your model spend so you can keep an eye on usage.

    ![Studio Assistant home screen with Build/Test/Ask/Analyze templates and the spend indicator](/images/studio-assistant/home-templates.png)
  </Step>

  <Step title="Ask, and let it read your project">
    For exploration, the assistant reads your agent settings, flows, topics, entities, and functions, then replies with project-grounded answers — not generic ones. You'll see the tools it ran inline (e.g. `read`, `explore`) before the final message, so its reasoning is auditable.

    ![Ask-mode response showing a project-specific Agent Overview for a Bella Vista restaurant agent](/images/studio-assistant/ask-mode-response.png)
  </Step>

  <Step title="Spot something to fix? It branches first">
    The moment you ask the assistant to make a change, it **creates a branch** for the work — automatically, named for what you asked it to do. `Main` stays untouched while the assistant explores, drafts, and edits on the branch. The branch chip appears in the top-left, replacing `Main` for the duration of the session.

    ![Studio Assistant creating a fix-booking-flow-wiring branch before making any changes](/images/studio-assistant/branch-created.png)
  </Step>

  <Step title="Approve the change, see exactly what it touched">
    When the assistant proposes a fix, a single confirmation in chat is enough to apply it. Every edit is summarised in an **N change made** card you can expand to see precisely which steps, flows, or settings were modified — and the assistant tells you which branch the change landed on.

    ![A single-step fix applied to the collect_phone_number step on the fix-booking-flow-wiring branch](/images/studio-assistant/change-applied.png)
  </Step>

  <Step title="Merge when you're happy">
    The **Merge** button in the top-right opens a dialog where you write a merge note and ship the branch back to `Main`. From there, your changes flow through the normal [environments and versions](/environments-and-versions/introduction) process to Sandbox, Pre-release, and Live.

    ![Merge to the main branch dialog with a merge note typed in](/images/studio-assistant/merge-dialog.png)
  </Step>
</Steps>

## Modes

Three modes control what the assistant is allowed to do. Switch between them using the mode toggle above the message input.

| Mode | What it does | Branches? |
|------|--------------|-----------|
| **Agent** | Plans, then executes changes against a branch | Yes — creates a branch on the first edit |
| **Plan** | Produces an implementation plan without making any changes | No |
| **Ask** | Answers questions about your agent and PolyAI Studio concepts, read-only | No |

In **Plan** and **Ask**, the assistant cannot create, update, or delete anything. Ask it to make a change and it'll prompt you to switch to **Agent**.

[See modes →](/studio-assistant/modes)

## Conversations and history

- Chats are saved in the **History** panel on the left, scoped to the project.
- Resume any past chat, switch between concurrent ones, or archive what you no longer need.
- The assistant remembers everything in the current chat — *"now do the same for the cancellation flow"* works without repeating context.
- Drag in transcripts, policy docs, or screenshots to attach files — the assistant reads them as part of the request.

## What Studio Assistant can work on

Studio Assistant operates on the same building blocks you can edit by hand. For any of these, it can read your current configuration, propose changes, and (in **Agent** mode) write the changes to a branch.

| Building block | Examples of what the assistant can do |
|----------------|----------------------------------------|
| **Flows** | Create new flows; restructure steps, transitions, and conditions; rewrite step prompts |
| **Topics** | Create knowledge-base topics from a description; update wording or scope |
| **Entities** | Define entities and their values; update entity definitions used by flows |
| **Functions** | Wire flows to existing functions; describe and create new function specs |
| **Integrations** | Configure integrations and connect them to flows |
| **Handoffs** | Add transfer-to-human steps with the right conditions and metadata |
| **Agent settings** | Set or update purpose, role, and personality |

For complex requests, the assistant breaks the work down internally and delegates to specialised reasoning steps — for example, a planning step that surveys your project and identifies gaps, or a flows step that handles navigation patterns. You don't manage these directly; they show up as plan content and tool calls in the conversation log.

## Working effectively

The same prompt-quality principles that apply to LLMs apply here. Two patterns matter most:

- **State the business rule, not just the action.** *"Add a refund flow with a 30-day window, original payment method only"* lands closer to what you want than *"Add a refund flow"*.
- **One unit of work per request.** One flow, one topic, one set of related changes. Easier to review, faster to iterate.

## Limits

Studio Assistant has a per-account spending limit on model calls. The default is **$5 USD-equivalent per day**, with daily, weekly, monthly, or total reset cadences depending on your plan. The current limit and spend show next to the message input.

When you hit the limit, the assistant pauses (a *Resting* card replaces the input) until the next reset. Manual editing in Studio is unaffected.

[Usage and limits →](/studio-assistant/usage-and-limits)

## What to expect

- **Drafts only.** Studio Assistant edits the branch you're on. It never publishes or promotes between [environments](/environments-and-versions/introduction) on its own.
- **One scoped task at a time.** It works best on focused requests. For very large changes, break them into stages.
- **Business knowledge comes from you.** The assistant won't invent prices, hours, policies, or names — if those are missing, planning will surface them as questions.
- **Always reviewable.** Every change appears on a branch with a tracked-changes summary, so you can inspect, tweak, or revert it before merging.
