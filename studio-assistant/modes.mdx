Studio Assistant runs in one of three modes. The mode controls **what the assistant is allowed to do** in response to your message – nothing else changes.

Switch modes from the toggle directly above the message input. The assistant will tell you if you ask it to do something the current mode doesn't allow.

## Agent mode

The default. Plans, then executes changes against your draft.

**Use it when**

- You're ready to build, modify, or remove something.
- You want the assistant to take action and report back.

**What it can do**

- Create flows, topics, entities, functions, integrations, and handoffs.
- Update prompts, steps, conditions, and settings on existing elements.
- Delete elements you ask it to remove.
- Read everything in your project to ground its work in your existing configuration.

**How it behaves**

- For non-trivial requests, the assistant produces a short plan first and asks you to approve or revise it before any change is written.
- Trivial actions (renaming an item, deleting a single topic, connecting two existing flows) skip the plan and run directly.
- Every change lands on your **draft**. The assistant never publishes or promotes between [environments](/environments-and-versions/introduction).

**Reviewing a plan**

When the assistant produces a plan, two buttons appear underneath it:

- **Approve** – the assistant proceeds with the plan as written.
- **Revise** – opens a small feedback box where you can describe what should change ("scope it to just one flow", "use SMS instead of email", "skip the integration step"). Leave the box empty to reject the plan without feedback.

Approved and revised plans are kept in the conversation, so you can scroll back and see exactly which version of the plan produced each set of changes.

## Plan mode

Read-only planning. Analyzes the request and produces a detailed implementation plan – without making any changes.

**Use it when**

- The change is large, touches multiple flows, or restructures logic.
- You want to review the approach (and gather missing business inputs) before anything is written.
- You're scoping a piece of work with another teammate.

**What it can do**

- Read your full agent configuration.
- Produce an implementation plan that lists:
  - what already exists and is reusable,
  - what would be created, updated, or removed,
  - business-specific information the assistant still needs from you (e.g. opening hours, refund window, pricing).
- Iterate on the plan based on your feedback.

**What it cannot do**

- Create, update, or delete anything. If you ask, it will tell you to switch to **Agent** mode.

When you're happy with the plan, switch to **Agent** mode and tell the assistant to proceed – it will pick up from the approved plan.

## Ask mode

Read-only Q&A. Answers questions about your agent and PolyAI Studio concepts.

**Use it when**

- You're exploring an existing agent and want to understand how it's wired up.
- You're learning the platform and want explanations grounded in your project.
- You're on a production-bound draft and want zero risk of accidental edits.

**What it can do**

- Read your agent configuration to give grounded, project-specific answers.
- Explain how Studio features (flows, topics, entities, functions, handoffs, environments) work.
- Surface relationships – which flows call which functions, where an entity is used, what a topic says.

**What it cannot do**

- Create, update, or delete anything.

### Example questions for Ask mode

- *Which flows can transfer to a human, and under what conditions?*
- *What does my opening-hours topic say?*
- *Which flows reference the `account_lookup` function?*
- *Summarize the personality and role defined in agent settings.*
- *How do entities work in Studio?*

## Choosing a mode

| You want to… | Use |
|--------------|-----|
| Build or change something now | **Agent** |
| Review an approach before any change is made | **Plan** |
| Understand your current agent or learn the platform | **Ask** |
| Work safely on a draft heading to production | **Plan** or **Ask** |

You can switch modes at any time – including mid-conversation. Switching mode does not clear your chat history.
