A **View** is a saved combination of filters and column settings on the [Conversations](/analytics/conversations/review) table. Views turn one-off filtering into a repeatable workflow you can pin, share, and set as your default.

Views are scoped per channel – Voice and Webchat each keep their own set of pinned tabs.

## Concepts

- **View** – a saved combination of filters and column settings.
- **System View** – a built-in view (Production or Test) that cannot be modified or deleted.
- **Custom View** – a user-created view, saved as a pinned tab.
- **Filters** – conditions that narrow the conversations table (PolyScore, duration, environment, custom metrics, etc.).
- **Column configuration** – which columns are visible, in what order.

<Info>
You can save up to **50 Custom Views per project**, per scope. Custom View names are limited to **25 characters**.
</Info>

## System Views

Every project starts with two System Views per scope. They cannot be renamed, edited, or deleted:

| System View | Filter | Use it for |
|---|---|---|
| **Production** | `environment = live` | Default for monitoring real customer conversations |
| **Test** | `environment ∈ {draft, sandbox, pre-release}` | Working in Sandbox, debugging branches, QA before deploying to Live |

If you spend most of your time in Test, save your filters as a Custom View and **set it as default** so the page opens there.

## Create a Custom View

<Steps>
  <Step title="Apply filters">
    Open **Analytics &gt; Conversations**, pick a scope (Voice or Webchat), and apply your filters and column choices to the table.
  </Step>
  <Step title="Save changes">
    When the current view has unsaved changes, a **Save changes** button appears in the views toolbar. Click it.
  </Step>
  <Step title="Name the view">
    Enter a name (up to 25 characters – a counter shows how many you have left) and confirm. The view appears as a new pinned tab with a pin icon.
  </Step>
</Steps>

## Manage views

Open the **three-dot menu** on any pinned tab to access view actions:

| Action | Shortcut | What it does |
|---|---|---|
| **Rename** | – | Update the view name in place. |
| **Set as default** | – | Open this view automatically when you visit the page. |
| **Duplicate** | <kbd>Ctrl</kbd>+<kbd>D</kbd> | Create a new Custom View with the same filters and columns. |
| **Share filters as link** | <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>L</kbd> | Generate a shareable URL encoding the current filter state. |
| **Delete** | – | Remove the Custom View. System Views cannot be deleted. |

Switching between views is a single click – no page reload, no re-navigation.

## Share a view

Use **Share filters as link** to copy a URL encoding the current filter and column state. When a teammate opens the link:

- A banner appears at the top of the page informing them that a shared filter set has been applied **temporarily**.
- They can click **Dismiss** to return to their previous view, or **Save as new view** to add it to their own pinned tabs.

This avoids silently overriding someone's existing configuration.

## Set a default view

Right-click (or open the three-dot menu) on the tab you want as your default and choose **Set as default**. The next time you open the Conversations page, that view loads automatically. Only one Custom View per scope can be the default at a time.

## Known limitations

- View names are capped at **25 characters**.
- Maximum **50 Custom Views per project**, per scope.
- System Views (Production, Test) cannot be renamed, edited, or deleted.
- Views are scoped per channel – a Voice view does not appear on the Webchat scope and vice versa.
