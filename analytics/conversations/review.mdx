Use **Conversation Review** to see exactly what happened on every voice call and webchat session – the full transcript, the topics the agent matched, the tools it called, and how it handled each turn. It is the primary tool for debugging agent behavior and auditing conversation quality.

![conversations-review](/images/analytics/conv-review.png)

<Note>
For end-to-end QA workflows, see [QA and analytics](/learn/maintain/qa-analytics). For programmatic export, see [Call data](/call-data/introduction).
</Note>

## How the page is organized

The page has three layers: **channel scopes** at the top, your **views** below them, and the **conversations table** that opens an integrated **review side panel** when you select a row.

### Voice and webchat scopes

Conversations are split by channel. Switch between them using the navigation in the sidebar under **Analytics > Conversations**:

- **Voice** – calls placed through the voice channel
- **Web chat** – sessions on the [webchat](/webchat/introduction) channel

Each scope has its own column layout (webchat hides voice-only fields), its own set of views, and its own default view, so you can keep voice and webchat workflows completely separate.

### Views

Each scope opens on a built-in **System View** by default:

- **Production** – live environment conversations only
- **Test** – sandbox, draft, and pre-release conversations

System Views can't be renamed or deleted. To save your own filter and column combinations as pinned tabs, use [Views](/analytics/conversations/views).

### Conversations table

Each row in the table is one conversation. Columns are configurable per view. Default visible columns are **Start time** (always visible), **Contact**, **Summary**, **Duration**, and **PolyScore**. Additional columns are hidden by default and can be shown from the **Columns** panel.

- **Live calls** appear at the top with a duration indicator and update automatically as new turns arrive.
- **Sort** any sortable column by clicking its header.
- **Select rows** to create a [test case](/analytics/test-suite/introduction) batch from the toolbar.
- **Click a row** to open the conversation in the review side panel without leaving the table.

### Review side panel

Clicking a conversation row opens the **Conversation Review** side panel. The panel header shows the contact's email address or phone number, the conversation start time, the channel type, and the detected language — for example: `user@example.com • 28 Apr 14:59 • Agent chat • English (UK)`.

The panel has four tabs:

| Tab | What you see |
|---|---|
| **Transcription** | Turn-by-turn transcript, audio waveform (voice only), matched topics, retrieved sources, and the **Diagnosis** toggle group |
| **Scores** | [PolyScore](/analytics/polyscore) and [CSAT](/analytics/csat/introduction) badges with per-dimension breakdowns |
| **Details** | Call info (status, environment, version), pre-built metrics (handoff target and reason, function calls, SMS events, A/B group, group ID, QA), and quick links to latency visualization, data logs, and outbound calls |
| **Custom metrics** | Searchable, sortable list of any [custom metrics](/settings/metrics) configured for the project |

Use the **link icon** next to "Conversation Review" in the panel header to copy a deep link to the conversation. Press **Esc** or click the close icon to dismiss the panel and return to the table.

<Note>
Webchat sessions do not have AI-generated voice summaries. Panels for webchat conversations display a "No summary due to conversation being via chat" notice where the summary would appear.
</Note>

## Columns

Click **Column** in the table toolbar to open the Columns panel. It has two tabs:

- **Visibility** – toggle individual columns on or off. **Start time** is always visible and cannot be hidden. All other columns can be shown or hidden. Columns are grouped under **Pre-built metrics**, which includes: Contact, PolyScore, Summary, Duration, Language, Call SID, Variant, QA, Handoff To, A/B Group, and Environment.
- **Order** – drag columns into your preferred display order. Start time is locked as the first column.

Changes apply to the active view only. You can save a view after adjusting columns to make the layout persistent.

## Filters

Click **Filter** to open the filter builder. You can add up to **10 filters** per view. Available filter types include:

- **Start date** – date range picker (with timezone offset shown)
- **PolyScore** – numeric comparison (less than, greater than, equals)
- **Duration** – time-based comparison (more than, less than), set in hours, minutes, and seconds
- **Environment** – include or exclude specific environments
- **Function calls** – filter by whether any function ran, or by a specific function name (supports *exists*, *equals*, *contains*)
- **SMS events** – filter by whether any SMS was sent, or by a specific SMS event name (supports *exists*, *equals*, *contains*)

Active filters appear as chips above the table. Each chip shows a short summary of the condition (for example, **PolyScore < 5**, **Duration > 1m 0s**). Click the **×** on a chip to remove that filter.

<Tip>
Combine PolyScore and Duration filters to build a focused QA queue – for example, low-scoring calls that ran longer than a minute.
</Tip>

## Caller identity and PII

The panel header displays the caller's email (webchat) or phone number (voice) when available. Personally identifiable details are visible only to users with [PII-level access](/tools/classes/conv-log#pii). When PII masking is enabled for your account, contact details are redacted in both the table and the panel.

## Matched topics and sources

When the agent matches [Knowledge](/managed-topics/introduction) topics or retrieves content from [Connected Knowledge](/connected-knowledge/introduction) sources, both surfaces appear under the relevant turn:

- **Matched topics** – tags listing the topics most relevant to the user's utterance.
- **Sources** – clickable source names. Clicking a source opens an inline preview panel; use **Open in Knowledge** to jump to the source file.

![sources-conversation-review](/images/release-notes/2603/sources-conversation-review.png)

Enable both layers from the **Diagnosis** toggle group on the Transcription tab. See [Diagnosis](/analytics/conversations/diagnosis) for the full list.

## PolyScore at a glance

Every eligible voice call and webchat session displays a [PolyScore](/analytics/polyscore) badge as a column in the table and on the **Scores** tab – a 0–10 quality rating generated automatically by AI. The badge is color-coded (green for 7–10, amber for 4–6, red for 0–3) and expands to show per-dimension breakdowns. PolyScore works the same way across both channels, so you can run the same QA workflows for voice and chat.

Use PolyScore in Conversation Review to:
- **Prioritize review** – sort or filter the table by PolyScore to surface the lowest-quality calls first.
- **Understand failure modes** – expand the breakdown to see whether a low score reflects poor flow, an unresolved task, or caller frustration.
- **Spot-check high scores** – confirm that well-scored calls followed the right process before using them as benchmarks.

<Tip>
Save a [view](/analytics/conversations/views) filtered by low PolyScore to build a focused QA queue.
</Tip>

## Conversation status

The end of every conversation is visually marked with an **"Ended conversation"** indicator. Live calls show an **"In progress"** badge until they close automatically.

## Default filters

Each scope opens on the **Production** System View. If you are working in Sandbox or have not yet deployed to production, switch to the **Test** System View, or save a Custom View with your preferred filters and set it as your default.

## Data retention

<Warning>
Transcripts older than **90 days** are automatically removed in line with PII retention policies. Plan exports accordingly if you need to keep conversation data for longer.
</Warning>
