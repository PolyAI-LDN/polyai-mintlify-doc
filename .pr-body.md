## Summary

Adds a customer-facing **A/B testing** doc and announces it in the **May 2026** release notes.

The page is built from the actual Agent Studio (Jupiter) frontend implementation in `platform_ui` — UI copy, modal flow, traffic split, constraints, and emitted call metrics are all sourced directly from the code so they match what users see.

## Changes

- `environments-and-versions/ab-testing.mdx` — new page covering:
    - Concept (control = Live, variant = Pre-release, 50/50 deterministic split by caller)
    - Prerequisites (incl. `ab_tests` feature flag)
    - Start flow (Pre-release tab → version overflow menu → **Start A/B test** modal)
    - Constraints while active (no promote, no rollback)
    - End flow (winner = control keeps Live, variant auto-promotes to Live)
    - Result metrics (`AB_TEST_ID`, `AB_TEST_GROUP`) for QuickSight / custom dashboards
    - MVP limitations
- `releases/notes/26.05.mdx` — new accordion entry + intro line linking to the page.
- `docs.json` — nav entry under **Environments**.

## Deliberately omitted (per upstream guidance)

- **No mention of the A/B test API** — endpoints exist but aren't a stable customer-facing API yet.
- **No Smart Analyst integration** mentioned (no ticket).
- **No mid-test traffic adjustment** (`PATCH` exists but isn't exposed in the UI).
- Term is **A/B testing**, never "canary" (clashes with the K8s canary system).
