Adds a feature doc set for **Polyphone** – the embeddable voice-only WebRTC widget – mirroring the structure and tone of the existing Webchat widget pages.

## Pages

- `polyphone/introduction.mdx` – overview, when to use, architecture, browser support, Polyphone vs telephony
- `polyphone/widget.mdx` – Studio editor walkthrough: prerequisites, create dialog, Styling / Content / Embed tabs, save & publish lifecycle
- `polyphone/widget-installation.mdx` – HTML and Tag Manager embed, re-embed rules, install verification, cross-tab behaviour
- `polyphone/test-widget.mdx` – preview-token call walkthrough, mic permission outcomes, sharing previews
- `polyphone/widget-troubleshooting.mdx` – launcher / mic / connection / audio quality / cross-tab lock / re-embed banner

## Source of truth

Content is grounded in the live `polyai-ldn/poly_core` and `PolyAI-LDN/platform_ui` code – including the unified `widget_configurations` table, the `WidgetUiCopy` Zod schema (welcome / call CTA / disclaimer / policies), the draft → published lifecycle (`status`, `published_at`, `last_published_script_tag`), the cross-tab `localStorage` lock, and the WebRTC gateway transport described in `poly_core/src/voice/call/webrtc/docs/external-api-guide.md`. Studio screen anatomy follows the Figma flows for the PolyPhone setup page.

## Scope decisions made

- **Polyphone is treated as a sibling of Webchat** under Channels in the sidebar, not as a sub-page of Webchat – matches the Studio top-of-page **Polyphone | Webchat** tabs in Figma and the unified `widget_configurations` model.
- **Hybrid mode is out of scope** for v1 docs – the Add-widget dialog is single-select per Figma.
- **Call-state strings** ("Connecting…", "In call", "Mute", etc.) are documented as **fixed runtime strings**, since they are hardcoded in `PhoneWidget` today and are not in `WidgetUiCopy`.
- **Image placeholders removed** – pending Frances sharing Figma node `1457-1808` for screenshot capture. The `<Frame>` blocks can be reintroduced once images land in `images/polyphone/`.
- **Mic-consent screen documented as live** in the test-widget walkthrough; engineering still needs to un-bypass `ConsentView` in `PhoneWidget` for the launch UX to match.
- **No SDK reference page** included yet (developer-facing `window.PolyphoneAPI` docs) – that's the next slice after the builder docs.
