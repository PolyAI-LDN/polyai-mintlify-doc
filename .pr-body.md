Fixes the documented `limit` query parameter range on `GET /v3/{account_id}/{project_id}/conversations` from `1 <= x <= 5000` to `1 <= x <= 1000`.

## Root cause

The OpenAPI spec at `api-reference/conversations/v3/openapi.json` declared `maximum: 5000` for the `limit` parameter, which does not match the actual backend.

In `PolyAI-LDN/poly_core`:
- `src/agent_v3/jupiter_api/openapi/conversations.yaml` (the spec backing the public Jupiter API that `docs.poly.ai` documents) defines `limit` with `maximum: 1000` and `default: 1000`.
- `src/agent_v3/project_store/openapi/conversations.yaml` likewise uses `maximum: 1000`.
- `src/agent_v3/conv_storage/handlers/conversations_handler.py` reads `limit` with a default of `1000`.

The `5000` value comes from `src/agent_v3/api/openapi/conversations/conversations.yaml`, an internal/admin-style spec that was the wrong source when the Mintlify OpenAPI was authored. This PR aligns the published docs with the actual public API contract.

## Change

`api-reference/conversations/v3/openapi.json` — `limit.maximum`: `5000` → `1000`.
