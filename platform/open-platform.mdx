The PolyAI Open Platform is the self-serve way to try Agent Studio. Sign up with an email address, build a working agent with [Studio Assistant](/studio-assistant/introduction), and embed it on your website with a [Polyphone widget](/polyphone/introduction). No contract, no sales call.

Existing enterprise customers continue to use the same platform with the full feature set. The Open Platform is the same Agent Studio with a focused subset for self-serve users.

<Note>
Sign up at [studio.us.poly.ai](https://studio.us.poly.ai/). The self-serve plan is open to anyone with an email address, no SSO required.
</Note>

### Sidebar at a glance

The Open Platform sidebar is shorter than the enterprise sidebar. You'll see:

| Section | Items |
|---|---|
| **Build** | Agent · Knowledge · Flows · Tools |
| **Configure** | General · Agent voice · Polyphone · Numbers · Integrations |
| **Analytics** | Dashboards · Agent Analysis · Conversations |

Everything else – telephony routing, SMS, handoffs, custom metrics, source hub, real-time configuration, language hub, dashboards builder, enterprise safety – is part of the enterprise tier. Those pages stay browseable in these docs, but in the Open Platform docs view the page content greys out behind a sticky banner so you can read it without acting on it. ## What's gated behind enterprise

These features need an enterprise contract. They're documented in full so you can plan for them, but you won't be able to use them on the self-serve plan until you upgrade. Trying to connect a gated integration shows the upsell directly in Studio:

![Integration Restricted modal showing Twilio is not enabled on self-serve accounts, with Close and Contact Sales buttons](/images/platform/enterprise-restricted-modal.png)

| Capability | Why it's enterprise-only |
|---|---|
| **Outbound and inbound telephony** | Live PSTN routing is contracted per-tenant on the enterprise cluster. The Open Platform runs on a separate cluster without telephony at launch. |
| **Live human handoff** | Requires SIP trunking to your contact-center platform (Genesys, Five9, Salesforce Voice, etc.). |
| **SMS** | Requires a provisioned SMS provider integration. |
| **Real-time configuration** | Schema-driven runtime config (Python schema, multi-environment values). |
| **Smart Analyst** | The page opens on the self-serve plan but actions show a Talk to sales prompt. Querying transcripts at scale needs an enterprise-tier configuration. |
| **Custom metrics, CSAT, dashboards builder** | Bespoke analytics that need account-level configuration. |
| **Enterprise safety filters** | Red-teamed safety rules, custom guardrails. |
| **Source hub (connected knowledge)** | Live document syncing from Confluence, Sharepoint, Google Drive, etc. |
| **Audio management** | Custom recorded prompts and voice cloning. |
| **Language hub** | Multi-language agent variants with shared content. |
| **Workspace API keys** | Use [personal access tokens](/secrets/personal-access-tokens) and the [Agents API](/api-reference/agents/introduction) instead. |

## Usage limits

Studio Assistant has a daily spend cap to keep usage predictable. The default cap, reset cadence, and any per-account adjustments are managed by PolyAI on your account.

See [Usage and limits](/studio-assistant/usage-and-limits) for what counts toward the limit, what happens when you hit it, and how to keep usage low.

## Building locally with the ADK

The [PolyAI ADK](/extend/adk) (`polyai-adk` on PyPI) lets you pull your project as YAML and Python files, edit with any tool, and push changes back. It works for self-serve accounts the same way it works for enterprise. Sign up at [studio.us.poly.ai](https://studio.us.poly.ai/), generate a [personal access token](/secrets/personal-access-tokens), and run `poly init`.

```bash
pip install polyai-adk
export POLY_ADK_KEY=<your-personal-access-token>
poly init
```

Full installation, authentication, and command reference at [polyai.github.io/adk](https://polyai.github.io/adk/).

## Upgrading to enterprise

When you outgrow the self-serve plan – typically when you need telephony, real-time config, or contracted SLAs – you migrate the project you've built to an enterprise cluster.

- Export and import are built into the ADK: `poly pull` your project, then `poly push` to the enterprise account.
- [Talk to sales](https://poly.ai/request-a-demo) when you're ready to discuss an enterprise contract.
- Existing enterprise contracts get the full sidebar with no greyed pages.

## Security

The Open Platform sits behind AWS WAF and CloudFront. Every account is isolated – usage limits, conversation data, and credentials are scoped per-account, and tenant context is derived from the signed JWT issued at sign-in. There is no cross-account access.

Sensitive credentials (API keys, OAuth client secrets) belong in the [Secrets Vault](/secrets/introduction). Functions retrieve them at runtime; they never sit in code, logs, or version history.

## Open Platform docs view

The Open Platform docs view is a streamlined view of these docs for self-serve users. Enterprise-only entries are dimmed in the sidebar so you can see at a glance what's in the self-serve plan and what isn't, but every entry stays clickable. Open one and the page content is greyed out and non-interactive behind a sticky banner explaining why it's gated.

The same upsell pattern applies inside Agent Studio. When you try to connect an enterprise-only integration, the product shows you why it's gated and how to talk to sales.

### How to exit

You can leave the Open Platform docs view at any time in three ways:

- Click the **Exit Open Platform view** pill floating in the top-right corner of any docs page.
- If you land on an enterprise page in the Open Platform view, click **Exit Open Platform view** in the upsell banner at the top of that page.
- Come back to this page and click **Exit Open Platform view**.
