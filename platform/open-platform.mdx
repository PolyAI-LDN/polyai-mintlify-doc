Agent Builder is Agent Studio without the contract. Sign up with any email at [studio.poly.ai](https://studio.poly.ai/), describe what you want on the [home page](/studio-assistant/introduction), and you get a working voice agent you can test in the browser and embed on your site. The first agent typically takes ten minutes.

It runs on the same Raven model and infrastructure that powers enterprise deployments. There's no separate sandbox mode and no toy version. What you build is production-ready voice AI.

<Note>
**60-day free trial.** Sign up at [studio.poly.ai](https://studio.poly.ai/) with any email address. You get 60 days of free access with no billing or metering. If you don't move to a paid plan at the end of the trial, your agents are deactivated rather than deleted, so you can pick them back up later.
</Note>

## Build path

The order you'll actually work in.

1. **Scaffold from the [home page](/studio-assistant/introduction)**: describe the agent in plain language. It builds the first version on a branch you review.
2. **Tune the [knowledge](/managed-topics/introduction)**: Agent Builder seeds Managed Topics from a URL or document. Edit by asking.
3. **Add structure where you need it**: [no-code flows](/flows/no-code/introduction) for bookings, verification, lead capture, and anything multi-step.
4. **Wire up [integrations](/integrations/introduction)**: Microsoft, Zoom, Zendesk, Slack, web search via Paragon, or any MCP server.
5. **Test in the browser, embed [Web Calling](/polyphone/introduction)**: call the agent live, then drop the script tag on your site.
6. **Read the [conversations](/analytics/conversations/introduction), iterate**: real transcripts, real fixes, via the [iterate after a call](/learn/iterate-open-platform) loop.

## What's included

| Feature | What it does |
|---|---|
| [Agent Builder](/studio-assistant/introduction) | Describe an agent in natural language. It scaffolds flows, topics, voice, and integrations on a branch you can review before merging. |
| [Web Calling widget](/polyphone/introduction) | A WebRTC voice widget you embed on any site. End-to-end browser audio. Voice-only on Agent Builder. |
| [Managed topics](/managed-topics/introduction) | The agent's knowledge layer. Auto-populated from your URL or documents, editable in Agent Builder or in the UI. |
| [No-code flows](/flows/no-code/introduction) | Multi-step conversations built visually with Default steps and entity extraction. Booking, verification, routing, FAQ — all without writing code. |
| [Voice library](/voice/introduction) | The standard PolyAI voice catalog. Configurable per agent. |
| [Marketplace integrations](/integrations/introduction) | Microsoft, Zoom, Zendesk, Slack, and web search via Paragon. Plus any MCP server. |
| [Conversations and analytics](/analytics/conversations/introduction) | Every transcript and recording, plus the standard dashboard. Smart Analyst is folded into Agent Builder. |
| [ADK](/extend/adk) | `polyai-adk` on PyPI. Pull your project as YAML and Python, edit anywhere, push it back. Authenticate with a [personal access token](/secrets/api-keys). |

## Usage limits

Agent Builder has a fair-use limit so usage stays predictable. Size and reset cadence are set on your account; the indicator next to the message input shows your current usage. See [Usage and limits](/studio-assistant/usage-and-limits).

## Security and hosting

Agent Builder runs on its own dedicated cluster, separate from enterprise infrastructure. Your account is isolated: conversations, configuration, and credentials are only visible to you. No cross-account access.

Agent Builder is hosted in the US. EU and UK hosting are enterprise.

Sensitive credentials (API keys, OAuth client secrets) belong in the [Secrets Vault](/secrets/introduction). Functions retrieve them at runtime, so they don't end up in your code, logs, or version history.

## Local development with the ADK

The [ADK](/extend/adk) works the same on Agent Builder as it does on enterprise. Pull your project as YAML and Python, edit with any tool, push changes back.

```bash
pip install polyai-adk
export POLY_ADK_KEY=<your-personal-access-token>
poly init
```

Generate the PAT in **Secrets > Personal access tokens**. Full ADK reference at [polyai.github.io/adk](https://polyai.github.io/adk/).

## Moving to enterprise

Your work moves with you. When you need telephony, real-time configuration, contracted SLAs, or any of the enterprise capabilities, `poly pull` the project, then `poly push` onto an enterprise cluster. [Request a demo](https://poly.ai/request-a-demo) when you're ready.
