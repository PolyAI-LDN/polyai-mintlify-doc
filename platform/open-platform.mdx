The PolyAI Open Platform is the self-serve way to try Agent Studio. Sign up with an email address, build a working agent with [Studio Assistant](/studio-assistant/introduction), and embed it on your website with a [Polyphone widget](/polyphone/introduction) — no contract, no sales call.

Existing enterprise customers continue to use the same platform with the full feature set. The Open Platform is the same Agent Studio with a focused subset for the free trial.

<Note>
Sign up at [studio.us.poly.ai](https://studio.us.poly.ai/). The free trial is open to anyone with an email address — no SSO required.
</Note>

See [free trial mode](/platform/free-trial-mode) for how the docs adapt when you turn that filter on.

## What's not in the free trial

These features need an enterprise contract. They're documented in full so you can plan for them, but you won't be able to use them until you upgrade.

| Capability | Why it's enterprise-only |
| --- | --- |
| **Outbound and inbound telephony** | Live PSTN routing is contracted per-tenant on the enterprise cluster. The Open Platform runs on a separate cluster without telephony at launch. |
| **Live human handoff** | Requires SIP trunking to your contact-centre platform (Genesys, Five9, Salesforce Voice, etc.). |
| **SMS** | Requires a provisioned SMS provider integration. |
| **Real-time configuration** | Schema-driven runtime config (Python schema, multi-environment values). |
| **Custom metrics, CSAT, dashboards builder** | Bespoke analytics that need account-level configuration. |
| **Enterprise safety filters** | Red-teamed safety rules, custom guardrails. |
| **Source hub (connected knowledge)** | Live document syncing from Confluence, Sharepoint, Google Drive, etc. |
| **Audio management** | Custom recorded prompts and voice cloning. |
| **Language hub** | Multi-language agent variants with shared content. |
| **APIs page (workspace-scoped API keys)** | Use [personal access tokens](/secrets/personal-access-tokens) and the [Agents API](/api-reference/agents/introduction) instead. |

## Trial limits

Studio Assistant has a daily spend cap to keep usage predictable.

- Default: **$5 USD** of model spend per account, per day.
- Resets every 24 hours.
- Shared across everyone using Studio Assistant on the account.

See [Usage and limits](/studio-assistant/usage-and-limits) for what counts toward the limit, what happens when you hit it, and how to keep usage low.

## Building locally with the ADK

The [PolyAI ADK](/extend/adk) (`polyai-adk` on PyPI) lets you pull your project as YAML and Python files, edit with any tool, and push changes back. It works for free-trial accounts the same way it works for enterprise — sign up at [studio.us.poly.ai](https://studio.us.poly.ai/), generate a [personal access token](/secrets/personal-access-tokens), and run `poly init`.

```bash
pip install polyai-adk
export POLY_ADK_KEY=<your-personal-access-token>
poly init
```

Full installation, authentication, and command reference at [polyai.github.io/adk](https://polyai.github.io/adk/).

## Upgrading to enterprise

When you outgrow the free trial — typically when you need telephony, real-time config, or contracted SLAs — you migrate the project you've built to an enterprise cluster.

- Export and import are built into the ADK: `poly pull` your project, then `poly push` to the enterprise account.
- Talk to sales: [poly.ai](https://poly.ai/) → "Talk to sales".
- Existing enterprise contracts get the full sidebar with no greyed pages.

## Security

The Open Platform sits behind AWS WAF and CloudFront. Every account is isolated — usage limits, conversation data, and credentials are scoped per-account and tenant context is derived from the signed JWT issued at sign-in. There is no cross-account access.

Sensitive credentials (API keys, OAuth client secrets) belong in the [Secrets Vault](/secrets/introduction). Functions retrieve them at runtime — they never sit in code, logs, or version history.
