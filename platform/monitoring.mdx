PolyAI provides multiple layers of production monitoring so you can detect and respond to issues before they affect callers.

## Monitoring layers

| Layer | What it covers | Where to configure |
|-------|---------------|-------------------|
| **Alert rules** | Automated threshold-based monitoring for latency, errors, crashes, and call volume | [Alerts API](/api-reference/alerts/introduction) |
| **Webhook notifications** | Real-time HTTP notifications when alerts trigger or resolve | [Webhooks API](/api-reference/webhooks/introduction) |
| **Dashboards** | Visual performance tracking with standard, safety, and custom dashboards | [Dashboards](/analytics/dashboards/introduction) |
| **Conversation review** | Per-call diagnosis with topic citations, source attribution, and turn-level analysis | [Conversations](/analytics/conversations/introduction) |
| **PolyScore** | Automated conversation quality scoring | [PolyScore](/analytics/polyscore) |

## Alert rules

The [Alerts API](/api-reference/alerts/introduction) lets you create rules that monitor key metrics -- turn latency, API errors, function errors, call crashes, and call volume. When a metric breaches your threshold, PolyAI sends a signed webhook notification to your [registered endpoints](/api-reference/webhooks/introduction).

For setup instructions, available metrics, code examples, and webhook payload formats, see the [Alerts API reference](/api-reference/alerts/introduction).

### Recommended starting configuration

For a typical production deployment:

| Alert | Metric | Threshold | Window | Why |
|-------|--------|-----------|--------|-----|
| High latency | `turn_latency_p95` | 2000ms | 5m | Callers notice delays above 2 seconds |
| Call crashes | `call_crashes` | 0 | 5m | Any crash needs immediate attention |
| Function errors | `function_errors` | 5 | 10m | Indicates integration issues |
| Traffic drop | `call_volume` | Below expected baseline | 60m | May indicate telephony routing issues |

## Dashboards

[Dashboards](/analytics/dashboards/introduction) provide visual monitoring through three types:

- **[Standard dashboards](/analytics/dashboards/standard)** -- pre-built views for call volume, containment, duration, and topic distribution
- **[Safety dashboards](/analytics/dashboards/safety)** -- track safety-related metrics and edge cases
- **[Custom dashboards](/analytics/dashboards/custom)** -- build your own views with configurable metrics and filters

## Conversation-level diagnosis

When an alert fires or you notice a metric change, [Conversation review](/analytics/conversations/introduction) lets you drill into individual calls:

- **[Review conversations](/analytics/conversations/review)** -- read transcripts with full context
- **[Annotations](/analytics/conversations/annotations)** -- tag and categorize conversations for quality tracking
- **[Diagnosis](/analytics/conversations/diagnosis)** -- toggle topic citations, source attribution, and turn-level analysis to understand exactly what happened
