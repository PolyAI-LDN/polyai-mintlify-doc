PolyAI's **Agentic Dialog Platform** is open to every builder across the enterprise. It's the same platform brands use to handle their most complex customer conversations — bookings, billing, escalations, anything an agent needs to actually resolve. Most agents run on [Raven](/agent-settings/raven), PolyAI's proprietary LLM, with sub-300ms latency across 24+ languages.

The fastest way in is [Agent Builder](/studio-assistant/introduction): describe what you want and it configures the agent, conversation tracks, and guardrails for you. Every agent ships with a [shareable test environment](/widgets/test) so you can validate behavior across channels and get stakeholder buy-in before anything goes live. If you'd rather build like an engineer, the [ADK](/extend/adk) gives you self-serve API keys, a CLI, and a Git-native workflow — work in your own IDE and deploy from your terminal.

## Choose your surface

| | Agent Studio | ADK | APIs |
|---|---|---|---|
| **Who it's for** | Non-technical users, visual workflow | Technical teams, engineering workflows | Enterprises with internal tooling |
| **How you work** | Visual UI – point and click | CLI and Python – local dev with Git | REST endpoints – programmatic control |
| **Best for** | Building and tuning agents interactively | Bulk authoring, version control, CI/CD | Programmatic control, automation |
| **Code required** | Optional | Yes (Python) | Yes (any language) |

An agent built in Studio can be updated through the ADK or APIs, and vice versa.
