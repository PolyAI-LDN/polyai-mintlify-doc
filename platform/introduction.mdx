PolyAI gives you three ways to build, deploy, and manage conversational AI agents — all unified by the same underlying platform. Choose the surface that fits how you work.

<CardGroup cols={3}>
  <Card title="Agent Studio" icon="browser" href="/get-started/introduction">
    **Build with or without code**

    A comprehensive UI for building and maintaining agents, viewing analytics, and managing deployments. The primary interface for customer service managers, PS teams, and non-technical users.
  </Card>
  <Card title="PolyAI ADK" icon="terminal" href="/extend/adk">
    **Build like an engineer**

    A CLI and Python package providing a GitHub-native workflow for building agents locally. Built for GSIs, ISVs, and technical teams deploying at scale.
  </Card>
  <Card title="Agent Studio APIs" icon="code" href="/api-reference/introduction">
    **Build into your systems**

    A suite of REST APIs for building, updating, and configuring agents programmatically. Powers deep integration for partners, integrators, and enterprises with internal tooling.
  </Card>
</CardGroup>

## Choose your surface

| | Agent Studio | ADK | APIs |
|---|---|---|---|
| **Who it's for** | CS managers, PS teams, non-technical users | GSIs, ISVs, technical teams | Partners, integrators, internal tooling |
| **How you work** | Visual UI — point and click | CLI and Python — local dev with Git | REST endpoints — programmatic control |
| **Best for** | Building and tuning agents interactively | Bulk authoring, version control, CI/CD | White-labelling, embedding, automation |
| **Code required** | Optional | Yes (Python) | Yes (any language) |

All three surfaces manage the same agents. An agent built in Studio can be updated via ADK or APIs, and vice versa.
