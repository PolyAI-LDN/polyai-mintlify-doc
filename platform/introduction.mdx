PolyAI is an enterprise voice AI platform. Agents built on PolyAI handle real customer calls — answering questions, making bookings, resolving issues, and handing off to humans when needed. Every agent is powered by [Raven](/agent-settings/raven), PolyAI's proprietary LLM built for real-time voice conversations.

You get three ways to build, deploy, and manage agents — all unified by the same underlying platform. Choose the surface that fits how you work.

<CardGroup cols={3}>
  <Card title="Agent Studio" icon="browser" href="/get-started/introduction">
    **Build with or without code**

    A comprehensive UI for building and maintaining agents, viewing analytics, and managing deployments. The primary interface for non-technical users and teams who prefer a visual workflow.
  </Card>
  <Card title="PolyAI ADK" icon="terminal" href="/extend/adk">
    **Build like an engineer**

    A CLI and Python package providing a GitHub-native workflow for building agents locally. Built for technical teams deploying at scale.
  </Card>
  <Card title="Agent Studio APIs" icon="code" href="/api-reference/introduction">
    **Integrate with your systems**

    A suite of REST APIs for updating and configuring agents programmatically. Powers deep integration for enterprises with internal tooling and automation needs.
  </Card>
</CardGroup>

## Choose your surface

| | Agent Studio | ADK | APIs |
|---|---|---|---|
| **Who it's for** | Non-technical users, visual workflow | Technical teams, engineering workflows | Enterprises with internal tooling |
| **How you work** | Visual UI — point and click | CLI and Python — local dev with Git | REST endpoints — programmatic control |
| **Best for** | Building and tuning agents interactively | Bulk authoring, version control, CI/CD | Programmatic control, automation |
| **Code required** | Optional | Yes (Python) | Yes (any language) |

All three surfaces manage the same agents. An agent built in Studio can be updated via ADK or APIs, and vice versa.
