<Info>
  The start function runs before any user input — use it to initialize state, personalize the greeting, or route calls. The end function runs after every call ends — use it to log metrics, create tickets, or clean up state.
</Info>

## What makes them different

Every function in Agent Studio can be called by the LLM during a conversation. The start and end functions are the exception: they are called **deterministically** by the platform.

| | Start function | End function |
|---|---|---|
| **When it runs** | At the very beginning of every call, before the greeting | After every call ends (hang-up, agent hang-up, or handoff) |
| **LLM callable?** | No | No |
| **Has parameters?** | No | No |
| **Has description?** | No | No |
| **Has access to `conv`?** | Yes | Yes |

Because neither function is LLM-callable, they have no description and no parameters. They are pure Python — code that runs once, automatically.

## The start function

### What it is for

The start function runs before the greeting is played. Use it to:

- **Initialize state variables** to default values
- **Personalize the greeting** based on caller number, SIP headers, or agent memory
- **Set the voice** (or randomize it across a pool)
- **Set the language** based on a SIP header or other signal
- **Gate feature flags** by environment
- **Make an API call** to retrieve caller information and route the call accordingly

### Initializing state variables

If you use `$variable_name` in a prompt and the variable has not been initialized, the placeholder appears literally. The start function is the right place to set default values.

```python
def start(conv):
    # Initialize to empty string so it is invisible in prompts until set
    conv.state.order_status_prompt = ""
    conv.state.cart = None
    conv.state.order_finalized = False
```

### Personalizing the greeting

The start function can return an `utterance` that **overrides** the greeting configured in the Agent tab UI.

```python
from datetime import datetime

def start(conv):
    hour = datetime.now().hour
    if hour < 12:
        return {"utterance": "Good morning. Thanks for calling. How can I help?"}
    elif hour < 18:
        return {"utterance": "Good afternoon. Thanks for calling. How can I help?"}
    else:
        return {"utterance": "Good evening. Thanks for calling. How can I help?"}
```

You can also use state variables in the greeting field directly:
```
Hi, thanks for calling. I'm $agent_name. How can I help?
```

If you set `conv.state.agent_name` in the start function, it will be replaced correctly in the greeting.

### Personalizing with a caller lookup

If the caller's phone number maps to a known customer, you can fetch their data upfront and route or greet them accordingly.

```python
def start(conv):
    if conv.caller_number:
        user = api_get_user_by_phone(conv.caller_number)
        if user and user.get("has_pending_order"):
            conv.state.user_name = user["name"]
            conv.go_to_flow("order_tracking")
            return {
                "utterance": f"Hi {user['name']}, I can see you have an order on the way. Is that what you're calling about?"
            }
    # Default greeting handled by Agent tab
```

### Randomizing voice

```python
import random

VOICES = {
    "alex": ElevenLabsVoice(voice_id="...", ...),
    "sam":  ElevenLabsVoice(voice_id="...", ...),
}

def start(conv):
    name, voice_config = random.choice(list(VOICES.items()))
    conv.state.agent_name = name.capitalize()
    conv.set_voice(voice_config)
    conv.write_metric("agent_voice", name)
```

### Gating by environment

```python
def start(conv):
    conv.state.feature_new_flow_enabled = conv.environment in ("draft", "sandbox")
```

In functions that use this feature:
```python
if conv.state.feature_new_flow_enabled:
    # new behaviour
else:
    # existing behaviour
```

## The end function

### What it is for

The end function runs when every call ends — whether the user hangs up, the agent hangs up (via `hangup: True`), or the call is transferred to a live agent.

Use it for:

- **Logging analytics metrics** based on what happened in the conversation
- **Making a final API call** — creating a support ticket, submitting an order, sending a summary
- **Post-call classification** — use `conv.utils.prompt_llm()` with a classification prompt and the conversation history to automatically categorize calls

### Writing end-of-call metrics

```python
def end(conv):
    # Log whether the booking was completed
    if conv.state.booking_confirmed:
        conv.write_metric("booking_completed", True, write_once=True)
    else:
        conv.write_metric("booking_completed", False, write_once=True)

    # Log call duration category based on history length
    turn_count = len(conv.history)
    conv.write_metric("turn_count", turn_count, write_once=True)
```

### Creating a ticket via API

```python
def end(conv):
    ticket_data = {
        "conversation_id": conv.id,
        "caller": conv.caller_number,
        "summary": str(conv.state.cart or "No cart"),
        "outcome": "completed" if conv.state.order_finalized else "abandoned"
    }
    api_create_ticket(ticket_data)
```

### Post-call LLM classification

```python
def end(conv):
    transcript = "\n".join(
        f"{turn['role']}: {turn['text']}" for turn in conv.history
    )
    result = conv.utils.prompt_llm(
        model="claude-sonnet",
        prompt=f"""Classify this call into one of: booking, enquiry, complaint, other.
Transcript:
{transcript}
Respond with just the category name."""
    )
    conv.write_metric("call_category", result.strip(), write_once=True)
```

<Warning>
  The end function runs **after** handoffs as well. If you need access to data at the exact moment of handoff (before the call is fully over), you cannot rely on the end function for it — use the handoff function's return value instead.
</Warning>
