<Info>
  `conv.state` is the agent's memory. This lesson explains how to use it to persist data across turns, share information between functions, and inject dynamic values into prompts.
</Info>

## Why state exists

Python function variables are local. Once a function finishes executing, its variables are gone.

```python
def update_cart(conv, item: str, quantity: int) -> str:
    cart = {}  # this disappears when the function returns
    cart[item] = quantity
    return f"Added {quantity} {item}."
```

If the user adds five items to a cart across five turns, each function call would start with an empty `cart`. There would be no accumulation.

`conv.state` solves this. It is a dictionary that persists for the entire duration of the conversation — across turns, function calls, and flow transitions.

## Reading and writing state

Access state using dot notation or dictionary-style indexing:

```python
# Write
conv.state.cart = {}
conv.state.order_finalized = False

# Read
current_cart = conv.state.cart
is_finalized = conv.state.order_finalized

# Check if a variable exists (returns None if not yet set)
if conv.state.cart is None:
    conv.state.cart = {}
```

<Tip>
  Always check whether a state variable exists before using it. On the first call to any function, variables you have not yet set will return `None`.
</Tip>

## Example: a persistent shopping cart

```python
def update_cart(conv, item_name: str, quantity: int, action: str) -> str:
    """Update the user's shopping cart. Action is 'add' or 'remove'."""

    # Initialize on first call
    if conv.state.cart is None:
        conv.state.cart = {}

    item = item_name.lower()

    if action == "add":
        if item in conv.state.cart:
            conv.state.cart[item] += quantity
        else:
            conv.state.cart[item] = quantity
    elif action == "remove":
        if item not in conv.state.cart:
            return "That item is not in your cart."
        conv.state.cart[item] = max(0, conv.state.cart[item] - quantity)
        if conv.state.cart[item] == 0:
            del conv.state.cart[item]

    return f"Updated. Current cart: {conv.state.cart}"
```

The LLM receives the current cart state as function output on every call. It never has to reconstruct the cart from conversation history — the Python dictionary is always authoritative.

## Using state flags to coordinate functions

State variables are useful for tracking what has happened in a conversation and preventing certain actions from repeating.

```python
def finalize_order(conv) -> str:
    """Finalize the user's order."""
    # ... call API ...
    conv.state.order_finalized = True
    return "Your order has been placed."


def update_cart(conv, item_name: str, quantity: int, action: str) -> str:
    """Update the user's cart."""
    if conv.state.order_finalized:
        return "Your order is already finalized. You cannot modify the cart."
    # ... rest of function ...
```

Once `order_finalized` is set to `True`, the cart function blocks itself. The LLM receives an informative message and cannot modify the cart — regardless of what the user asks.

## Injecting state variables into prompts

You can insert a state variable's current value directly into the Agent behavior prompt using the `$variable_name` syntax.

In the **Rules** field or a step prompt:

```
Current state of the cart: $cart
Do not summarize the cart unless the user asks for it.
```

At runtime, `$cart` is replaced with the current value of `conv.state.cart`. The LLM always sees the up-to-date cart without it being included in every function return.

<Warning>
  If the state variable has not been set yet, the placeholder `$cart` will appear **literally** in the prompt. Initialize variables to an empty string `""` in the start function to avoid this — an empty string is invisible to the LLM.
</Warning>

### Dynamic prompts from state

You can also store entire prompt segments in state and inject them conditionally:

```python
# In finalize_order:
conv.state.order_status_prompt = "The order is finalized. Do not allow cart modifications."
```

In the Rules prompt:
```
$order_status_prompt
```

Before `finalize_order` runs, `$order_status_prompt` is an empty string (invisible). After it runs, the constraint appears in every subsequent LLM request.

## Viewing state in Conversation Review

During testing, enable **Conversation variables** in the Conversation Diagnosis panel. This shows every state variable and its current value at the end of each turn — invaluable for debugging functions that read and write state.

You will see entries like:
```
conv.state.cart = {"banana": 3, "kiwi": 1}
conv.state.order_finalized = false
```

Use this view to confirm state is being set correctly before relying on it in LLM prompts.
