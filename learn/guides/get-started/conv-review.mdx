## 14. Conversation review and diagnosis

[ ] Understand what actually happened during interactions.

Conversation Review shows transcripts, metadata, and diagnostic layers for both live and completed calls.

**Do**
- Open **Conversation review**.
- Filter to conversations you just tested.
- Inspect:
  - Transcript accuracy (ASR quality).
  - Matched KB topics.
  - Function calls and outcomes.
  - Handoff events.
  - Latency and interruptions.

**Use Diagnosis toggles**
- **Topic citations**: confirms which KB topic was used.
- **Function calls**: shows triggered actions and parameters.
- **Flows and steps**: tracks execution paths.
- **Variants**: shows which variant handled each turn.
- **Entities**: validates captured structured data.
- **Turn latency / interruptions**: highlights conversational friction.

**Annotations**
- Mark:
  - Wrong transcription
  - Missing topic
- Use annotations to guide KB expansion, ASR tuning, or rule changes.

**Verify**
- You can explain *why* the agent responded the way it did.
- You can identify at least one concrete improvement from review data.
