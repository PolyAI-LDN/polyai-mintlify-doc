## Quick decision guide

| I need to... | Go here | Time | Skill level |
|--------------|---------|------|-------------|
| Update FAQs or business info | [Managed Topics](/learn/maintain/knowledge-base) | 10-30 min | No code |
| Sync external sources (websites, files) | [Connected knowledge](/learn/maintain/knowledge-connected-knowledge) | 5-15 min | No code |
| Change voice or fix pronunciation | [Voice and audio](/learn/maintain/voice-audio-updates) | 5-20 min | No code |
| Update or debug a function | [Function maintenance](/learn/maintain/function-maintenance) | 15-60 min | Requires Python |
| Maintain multilingual agents | [Multi-language updates](/learn/maintain/multi-language-updates) | 30-90 min | No code |
| Check performance or fix issues | [Performance monitoring](/learn/maintain/performance-monitoring) | 30-120 min | No code |
| Analyze conversations at scale | [Smart Analyst](/smart-analyst/introduction) — deep sampling up to 500 calls | 5-10 min | No code |
| Publish or rollback changes | [Version management](/learn/maintain/version-management) | 5-15 min | No code |
| Daily/weekly health checks | [Health checks](/learn/maintain/health-checks) | 5-60 min | No code |

## Typical workflows

<Tabs>
  <Tab title="Daily" icon="sun">
    **Time:** 5-10 minutes

    <Steps>
      <Step title="Check overnight metrics">
        Review the [Standard dashboard](/analytics/dashboards/standard) for call volume, handoff rate, and latency.
      </Step>
      <Step title="Review recent calls">
        Scan **Analytics > Conversations** filtered to last 24 hours for errors.
      </Step>
      <Step title="Verify critical functions">
        Check function logs for API errors or failures.
      </Step>
      <Step title="Address urgent issues">
        Fix any issues identified, or escalate if needed.
      </Step>
    </Steps>

    **See:** [Health checks](/learn/maintain/health-checks)
  </Tab>
  <Tab title="Weekly" icon="calendar-week">
    **Time:** 30-60 minutes

    <Steps>
      <Step title="Run Smart Analyst deep sampling">
        Use [Smart Analyst](/smart-analyst/introduction) to surface the week's trends and failure patterns across hundreds of conversations.
      </Step>
      <Step title="Update Managed Topics">
        Add or update content based on Smart Analyst insights.
      </Step>
      <Step title="Sync Connected knowledge">
        Refresh external sources to ensure content is current.
      </Step>
      <Step title="Fix identified issues">
        Address knowledge gaps, pronunciation problems, or function errors.
      </Step>
      <Step title="Publish and promote">
        Deploy your changes through [environments](/environments-and-versions/introduction).
      </Step>
    </Steps>

    **See:** [Smart Analyst](/smart-analyst/introduction), [Managed Topics](/learn/maintain/knowledge-base), [Performance monitoring](/learn/maintain/performance-monitoring)
  </Tab>
  <Tab title="Monthly" icon="calendar">
    **Time:** 2-4 hours

    <Steps>
      <Step title="Comprehensive Smart Analyst review">
        Use [Smart Analyst](/smart-analyst/introduction) for a deep sampling review — analyze containment blockers, handoff reasons, and sentiment trends across the full month.
      </Step>
      <Step title="Optimize functions and performance">
        Refactor slow functions, update API integrations, and tune latency.
      </Step>
      <Step title="Update voice and audio">
        Review voice quality, fix pronunciations, and regenerate cached audio.
      </Step>
      <Step title="Address knowledge gaps">
        Fill gaps identified by Smart Analyst with new or updated topics.
      </Step>
      <Step title="Test and validate">
        Run test sets, validate changes in Sandbox, and promote when ready.
      </Step>
    </Steps>

    **See:** [Smart Analyst](/smart-analyst/introduction), [Voice and audio](/learn/maintain/voice-audio-updates)
  </Tab>
</Tabs>

<Tip>
When in doubt, use your [environments](/environments-and-versions/introduction) carefully. Make any changes in the **Sandbox**, test thoroughly, and only then promote up to **Live**.
</Tip>
