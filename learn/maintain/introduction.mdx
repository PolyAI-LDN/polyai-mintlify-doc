## Quick decision guide

| I need to... | Go here | Time |
|--------------|---------|------|
| Update FAQs or business info | [Knowledge base](/learn/maintain/knowledge-base) | 10-30 min |
| Sync external sources (websites, files) | [Connected Knowledge](/learn/maintain/knowledge-connected-knowledge) | 5-15 min |
| Change voice or fix pronunciation | [Voice and audio](/learn/maintain/voice-audio-updates) | 5-20 min |
| Update or debug a function | [Function maintenance](/learn/maintain/function-maintenance) | 15-60 min |
| Maintain multilingual agents | [Multi-language updates](/learn/maintain/multi-language-updates) | 30-90 min |
| Check performance or fix issues | [Performance monitoring](/learn/maintain/performance-monitoring) | 30-120 min |
| Publish or rollback changes | [Version management](/learn/maintain/version-management) | 5-15 min |
| Daily/weekly health checks | [Health checks](/learn/maintain/health-checks) | 5-60 min |

## Typical workflows

### Daily (5-10 minutes)
1. Check overnight metrics
2. Review recent calls
3. Verify critical functions
4. Address urgent issues

**See:** [Health checks](/learn/maintain/health-checks), [Alerts API](/api-reference/alerts/introduction)

### Weekly (30-60 minutes)
1. Update knowledge base content
2. Sync Connected Knowledge sources
3. Review analytics and trends
4. Fix identified issues
5. Publish and promote changes

**See:** [Knowledge base](/learn/maintain/knowledge-base), [Performance monitoring](/learn/maintain/performance-monitoring)

### Monthly (2-4 hours)
1. Deep analytics review
2. Optimize functions and performance
3. Update voice and audio settings
4. Comprehensive knowledge audit
5. Test and validate improvements

**See:** [Function maintenance](/learn/maintain/function-maintenance), [Voice and audio](/learn/maintain/voice-audio-updates)

<Tip>
When in doubt, use your [environments](/environments-and-versions/introduction) carefully. Make any changes in the **Sandbox**, test thoroughly, and only then promote up to **Live**.
</Tip>
