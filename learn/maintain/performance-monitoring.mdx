Monitoring your agent's performance helps you catch issues early and continuously improve the customer experience. This page covers key metrics, common issues, and how to diagnose problems.

## Key performance metrics

### Metrics to monitor

| Metric | What it measures | Target | Red flag |
|--------|------------------|--------|----------|
| **Containment rate** | % of calls resolved without handoff | >80% | <70% |
| **ASR accuracy** | % of utterances transcribed correctly | >95% | <90% |
| **Average call duration** | Time from start to resolution | Varies by use case | Sudden increases |
| **Handoff rate** | % of calls transferred to human | <20% | >30% |
| **Function success rate** | % of function calls that succeed | >98% | <95% |
| **Response latency** | Time from user stops speaking to agent responds | <2s | >3s |
| **Topic coverage** | % of queries matched to knowledge base | >90% | <80% |
| **Customer satisfaction** | Post-call survey scores | >4/5 | <3.5/5 |

### Where to find metrics

- **Conversation Review** - Individual call analysis
- **Analytics** - Aggregate metrics and trends
- **Agent Analysis** - Automated insights and categorization
- **Diagnosis** - Function logs and technical details
- **Annotations** - Flagged issues from manual review

## Common performance issues

### ASR (transcription) problems

**Symptoms:**
- Agent misunderstands user input
- Incorrect responses to clear questions
- Annotations flagged as "Wrong transcription"

**Diagnosis:**
1. Go to Conversation Review
2. Filter for calls with annotations
3. Review transcripts vs. audio
4. Identify patterns (specific words, accents, background noise)

**Solutions:**
- Add ASR corrections for commonly misheard words
- Use ASR biasing for expected input types
- Adjust microphone sensitivity settings
- Add pronunciations for domain-specific terms

**Example:**
```
User says: "I need help with my API key"
ASR hears: "I need help with my 80 key"
Fix: Add ASR correction "80 key" → "API key"
```

### Knowledge base gaps

**Symptoms:**
- Agent says "I don't have that information"
- Handoffs for questions that should be answerable
- Annotations flagged as "Missing topic"

**Diagnosis:**
1. Review Conversation Review for unanswered questions
2. Check Analytics for common unmatched queries
3. Use Agent Analysis to identify topic clusters

**Solutions:**
- Add new topics to Managed Topics
- Expand existing topic coverage
- Add Connected Knowledge sources
- Improve sample questions for better matching

### High handoff rate

**Symptoms:**
- More calls transferred than expected
- Handoffs for routine questions
- Increasing handoff trend over time

**Diagnosis:**
1. Go to Analytics → Handoff metrics
2. Review reasons for handoffs
3. Check if handoffs are appropriate or avoidable

**Solutions:**
- Add missing knowledge base topics
- Improve function reliability
- Clarify when handoffs should occur
- Train agent to handle more scenarios

### Function errors

**Symptoms:**
- Functions failing to execute
- Error messages in Diagnosis logs
- Incomplete transactions

**Diagnosis:**
1. Go to Conversation Review → Diagnosis
2. Filter for function errors
3. Review input parameters and error messages
4. Check `conv.log` entries for details

**Solutions:**
- Fix code errors in functions
- Update API credentials in Secrets
- Add error handling and fallbacks
- Improve parameter validation

**Common function errors:**
- Authentication failures → Update secrets
- Timeout errors → Optimize code or add delay controls
- Type errors → Validate inputs
- API rate limits → Add caching or retry logic

### Latency issues

**Symptoms:**
- Long pauses before agent responds
- Users complaining about slow responses
- Calls feel unnatural or robotic

**Diagnosis:**
1. Review call recordings for timing
2. Check Diagnosis logs for function execution times
3. Identify slow functions or API calls

**Solutions:**
- Switch to faster TTS provider (e.g., Cartesia)
- Use cached audio for common phrases
- Optimize slow functions
- Add delay controls with filler phrases
- Adjust interaction style (Swift or Turbo mode)

### Poor response quality

**Symptoms:**
- Responses are inaccurate or off-brand
- Agent repeats itself or loops
- Responses are too verbose or too brief

**Diagnosis:**
1. Review call transcripts
2. Check Agent Behavior rules
3. Review knowledge base content quality

**Solutions:**
- Update Agent Behavior with clearer instructions
- Improve knowledge base content
- Add Response Controls to prevent unwanted patterns
- Refine prompts in topic actions

## Using Conversation Review

### Filtering and searching

Find specific issues quickly:

1. **Filter by date** - Focus on recent calls
2. **Filter by annotations** - Find flagged issues
3. **Search by keyword** - Find calls about specific topics
4. **Filter by outcome** - Handoffs, completions, errors

### Annotations

Flag issues for later review:

- **Wrong transcription** - ASR problems
- **Missing topic** - Knowledge gaps
- **Custom annotations** - Your own categories

**Workflow:**
1. Review calls regularly (daily or weekly)
2. Annotate issues as you find them
3. Build a list of fixes
4. Prioritize by frequency and impact
5. Implement fixes in batches

### Diagnosis panel

The Diagnosis panel shows:

- **Function calls** - Inputs, outputs, errors
- **conv.log entries** - Custom logging from functions
- **API calls** - Requests and responses
- **Timing information** - Latency breakdown

**Use Diagnosis to:**
- Debug function errors
- Verify API integrations
- Identify performance bottlenecks
- Understand agent decision-making

## Using Analytics

### Key reports

- **Call volume** - Track usage trends
- **Containment rate** - Measure self-service success
- **Handoff analysis** - Understand why calls transfer
- **Topic coverage** - See which topics are used most
- **Function performance** - Track success rates

### Agent Analysis

Automated analysis with pre-built templates:

1. Go to Agent Analysis
2. Select a template (e.g., "Call categorization", "Issue detection")
3. Run on recent calls
4. Review insights and trends
5. Schedule for automatic daily/weekly runs

**Benefits:**
- Identify patterns across many calls
- Track changes over time
- Spot emerging issues early
- Quantify improvement efforts

## Troubleshooting workflows

### Diagnosing a spike in handoffs

1. Go to Analytics → Handoff metrics
2. Identify when the spike started
3. Review calls from that period
4. Look for common patterns:
   - New type of question?
   - Function failures?
   - ASR issues?
   - Knowledge base gap?
5. Implement fix based on root cause
6. Monitor to verify improvement

### Fixing ASR accuracy issues

1. Review calls with wrong transcriptions
2. Identify commonly misheard words or phrases
3. Go to Global ASR → ASR Corrections
4. Add corrections for misheard terms
5. Test in Agent Chat
6. Publish and monitor improvement

### Improving response quality

1. Identify calls with poor responses
2. Determine root cause:
   - Wrong topic retrieved?
   - Topic content inadequate?
   - Agent Behavior unclear?
   - Function returned wrong info?
3. Update the relevant component
4. Test in Agent Chat
5. Publish and verify improvement

### Reducing latency

1. Identify slow calls in Conversation Review
2. Check Diagnosis for timing breakdown
3. Find the bottleneck:
   - Slow TTS provider?
   - Long-running function?
   - External API delay?
4. Implement solution:
   - Switch TTS provider
   - Optimize function code
   - Add caching
   - Use delay controls
5. Test and measure improvement

## Testing and validation

### Before promoting to Live

- [ ] Test in Agent Chat with realistic scenarios
- [ ] Make test calls to Sandbox number
- [ ] Review Diagnosis logs for errors
- [ ] Verify functions execute correctly
- [ ] Check ASR accuracy
- [ ] Confirm response quality
- [ ] Test edge cases
- [ ] Run Test Sets if configured

### A/B testing with variants

Use variants to test changes:

1. Create variant with proposed change
2. Route portion of traffic to variant
3. Compare metrics between variants
4. Choose better-performing version
5. Roll out to all traffic

## Best practices

### Monitoring routine

- **Set up alerts** - Get notified of critical issues
- **Review regularly** - Don't wait for problems to escalate
- **Track trends** - Monitor changes over time
- **Document issues** - Keep a log of problems and fixes
- **Share insights** - Keep your team informed

### Data-driven improvements

- **Measure before and after** - Quantify impact of changes
- **Focus on high-impact issues** - Fix frequent problems first
- **Test hypotheses** - Validate assumptions with data
- **Iterate continuously** - Small, frequent improvements

### Quality assurance

- **Listen to calls** - Don't just read transcripts
- **Get user feedback** - Ask customers about their experience
- **Test edge cases** - Don't just test happy paths
- **Monitor after changes** - Verify fixes work as expected

## Common workflows

### Weekly performance review

1. Open Analytics dashboard
2. Review key metrics vs. targets
3. Identify any concerning trends
4. Go to Conversation Review
5. Filter for annotated calls
6. Review and categorize issues
7. Create fix list prioritized by impact
8. Implement top 3-5 fixes
9. Test in Sandbox
10. Publish and promote

### Responding to a performance issue

1. Identify the issue (alert, report, observation)
2. Quantify the impact (how many calls affected?)
3. Review affected calls in Conversation Review
4. Use Diagnosis to understand root cause
5. Develop and test fix in Sandbox
6. If urgent, promote to Live immediately
7. If not urgent, batch with other fixes
8. Monitor to verify resolution

### Continuous improvement cycle

1. **Measure** - Review metrics and call quality
2. **Analyze** - Identify patterns and root causes
3. **Improve** - Implement fixes and enhancements
4. **Test** - Validate changes in Sandbox
5. **Deploy** - Promote to Live
6. **Monitor** - Verify improvement
7. **Repeat** - Start the cycle again
