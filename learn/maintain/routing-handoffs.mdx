If the agent can't finish a caller's request, it should **hand off** the call to a human or another system. You can update these handoff destinations yourself.

## Quick reference

| I need to... | Action | Time estimate |
|--------------|--------|---------------|
| Update a phone number | Edit destination → Change number → Save | 2 min |
| Add a new handoff destination | Add handoff → Fill details → Save | 5 min |
| Add SIP headers | Edit destination → Add SIP header → Save | 3 min |
| Test a handoff | Call Sandbox → Trigger transfer → Verify | 5 min |
| Connect Twilio number | Configure > Numbers > Twilio → Enter credentials | 10 min |
| Fix failed transfer | Check number format, firewall, SIP headers | 10 min |

<Note>This page covers the **UI-based Call Handoff** feature. Ask if your project uses the code-level `transfer_call` [function](/tools/introduction).</Note>

## Edit an existing destination

![overview](/images/learn/maintain-handoff.png)

<Steps>
  <Step title="Open Call handoffs">
    In the sidebar, select **Call handoffs**.
  </Step>
  <Step title="Select and edit">
    Hover over the destination you want and click **Edit**.
  </Step>
  <Step title="Update details">
    Change the **Number / SIP URI** or **Description**.
  </Step>
  <Step title="Save">
    Click **Save**.
  </Step>
  <Step title="Test and promote">
    Make a quick test call in *Sandbox* and confirm the transfer works, then [promote your version](/environments-and-versions/introduction).
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Common reasons to update" icon="rotate">
    | Reason | What to update |
    | ------- | -------------- |
    | Front-desk line changed | Replace the number in **Route** |
    | Routing after-hours | Add a note in **Description** so team mates know when to switch destinations |
  </Accordion>
</AccordionGroup>

## Add a new handoff destination

<Steps>
  <Step title="Click Add handoff">
    Click **Add handoff**.
  </Step>
  <Step title="Fill in details">
    - **Name** — e.g. "VIP host desk"
    - **Method** — leave **SIP REFER** unless your telephony team says otherwise
    - **Route / Number** — `+1XXXXXXXXXX` or a SIP URI
    - *(Optional)* **SIP headers** if your PBX needs extra context
  </Step>
  <Step title="Add, test, promote">
    Click **Add**, test in Sandbox, then promote.
  </Step>
</Steps>

## Add SIP headers (optional)

SIP headers let you pass metadata — account ID, language, VIP flag, etc.

<Steps>
  <Step title="Open header settings">
    While creating or editing a destination, click **Add SIP header**.
  </Step>
  <Step title="Enter header name">
    Custom headers start with `X-`.
  </Step>
  <Step title="Enter a value">
    Use a static value or a variable such as `$caller_id`.
  </Step>
  <Step title="Save">
    Save the destination.
  </Step>
</Steps>

## Using your own Twilio number

If you bring your own Twilio DID:

<Steps>
  <Step title="Connect Twilio">
    Go to **Configure > Numbers > Twilio** and enter your Account SID + Auth Token.
  </Step>
  <Step title="Configure the webhook">
    In Twilio, point the number's **Voice webhook** at your agent URL.
  </Step>
  <Step title="Set up handoff">
    Back in **Call handoffs**, use that number as the **Route** field.
  </Step>
</Steps>

<Warning>US numbers need [A2P 10DLC registration](https://www.twilio.com/docs/messaging/compliance/a2p-10dlc) for SMS.</Warning>

## Test before you ship

<Steps>
  <Step title="Call the Sandbox number">
    Call the *Sandbox* number.
  </Step>
  <Step title="Trigger the transfer">
    Trigger the scenario that should transfer.
  </Step>
  <Step title="Verify the behavior">
    Confirm the call operates how you are expecting.
  </Step>
  <Step title="Promote to Live">
    Promote the new version to *Live*.
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Transfer failed? Check these" icon="wrench">
    - Number format — include **+** and country code
    - Firewalls or other rules on the destination side
    - SIP header spelling
  </Accordion>
</AccordionGroup>

## When to escalate

- You need more conditional logic than the UI can express
- The destination uses a non-standard SIP method
- You are seeing [4XX/5XX SIP errors](https://datatracker.ietf.org/doc/html/rfc3261#section-13.2.2.3)
