Regular health checks help you catch issues before they impact customers. This page provides checklists and routines for proactive agent maintenance.

## Daily health check (5-10 minutes)

Perform these checks each morning to catch overnight issues:

### Quick metrics review

- [ ] **Call volume** - Is it within normal range?
- [ ] **Handoff rate** - Any unusual spikes?
- [ ] **Function errors** - Check for new failures
- [ ] **System alerts** - Review any notifications

### Conversation spot check

- [ ] Review 3-5 recent calls in Conversation Review
- [ ] Check for new annotation types
- [ ] Verify responses are accurate and on-brand
- [ ] Look for unusual patterns or errors

### Critical function check

- [ ] Test critical functions in Agent Chat
- [ ] Verify API integrations are working
- [ ] Check that secrets haven't expired
- [ ] Confirm external services are accessible

**Time estimate: 5-10 minutes**

**Red flags:**
- Handoff rate >30%
- Multiple function errors
- ASR accuracy drop
- Unusual call patterns

## Weekly health check (30-60 minutes)

Perform a deeper review once per week:

### Analytics review

- [ ] **Containment rate** - Compare to previous week
- [ ] **Top topics** - What are users asking about?
- [ ] **Handoff reasons** - Why are calls transferring?
- [ ] **Call duration** - Any significant changes?
- [ ] **ASR accuracy** - Review transcription quality

### Knowledge base audit

- [ ] Review unanswered questions
- [ ] Check for knowledge gaps from annotations
- [ ] Verify recent content updates are working
- [ ] Test top 10 most common queries

### Function health

- [ ] Review function success rates
- [ ] Check for timeout or latency issues
- [ ] Verify API credentials are current
- [ ] Test error handling paths

### Voice and audio

- [ ] Listen to sample calls
- [ ] Check for pronunciation issues
- [ ] Verify voice quality is consistent
- [ ] Review cached audio usage

### Version status

- [ ] Confirm Live version is correct
- [ ] Check for unpublished drafts
- [ ] Review recent promotions
- [ ] Verify environment configurations

**Time estimate: 30-60 minutes**

## Monthly health check (2-4 hours)

Comprehensive monthly audit:

### Deep analytics dive

- [ ] Month-over-month metric comparison
- [ ] Trend analysis (improving or degrading?)
- [ ] Customer satisfaction scores
- [ ] Peak usage patterns
- [ ] Seasonal variations

### Knowledge base deep clean

- [ ] Review all topics for accuracy
- [ ] Remove outdated content
- [ ] Update seasonal information
- [ ] Expand coverage for common gaps
- [ ] Sync all Connected Knowledge sources

### Function optimization

- [ ] Review all function code
- [ ] Optimize slow functions
- [ ] Update API integrations
- [ ] Rotate secrets and credentials
- [ ] Remove unused functions

### ASR and voice optimization

- [ ] Review ASR corrections effectiveness
- [ ] Add new corrections for common errors
- [ ] Update pronunciations
- [ ] Test voice quality across devices
- [ ] Review response controls

### Agent Behavior review

- [ ] Review and update rules
- [ ] Test edge cases
- [ ] Verify tone and style consistency
- [ ] Update examples and prompts

### Testing and validation

- [ ] Run full Test Set suite
- [ ] Test all critical user journeys
- [ ] Verify handoff flows
- [ ] Test error scenarios
- [ ] Cross-device testing

**Time estimate: 2-4 hours**

## Pre-promotion checklist

Before promoting any version to Live:

### Testing

- [ ] All changes tested in Sandbox
- [ ] UAT completed in Pre-release
- [ ] Test Sets run successfully
- [ ] Critical paths verified
- [ ] Edge cases tested
- [ ] Error handling validated

### Review

- [ ] Version description is clear
- [ ] Stakeholders have approved
- [ ] No critical issues in logs
- [ ] Performance is acceptable
- [ ] No breaking changes

### Preparation

- [ ] Rollback plan ready
- [ ] Team notified of promotion
- [ ] Monitoring plan in place
- [ ] Off-hours promotion scheduled (if major)
- [ ] Documentation updated

### Post-promotion

- [ ] Monitor first 10-20 calls
- [ ] Check error logs
- [ ] Verify metrics are stable
- [ ] Confirm no unexpected behavior
- [ ] Be ready to rollback if needed

**Time estimate: 15-30 minutes**

## Seasonal maintenance

### Before peak seasons

- [ ] Update seasonal content (hours, policies, FAQs)
- [ ] Add seasonal Connected Knowledge sources
- [ ] Test high-volume scenarios
- [ ] Verify capacity and performance
- [ ] Update Agent Behavior for seasonal context

### After peak seasons

- [ ] Disable seasonal content
- [ ] Review peak performance
- [ ] Document lessons learned
- [ ] Update for next year
- [ ] Archive seasonal versions

### Holiday preparation

- [ ] Update business hours
- [ ] Add holiday-specific FAQs
- [ ] Update handoff messages
- [ ] Test holiday scenarios
- [ ] Schedule promotions around holidays

## Monitoring dashboards

### Key metrics to track

| Metric | Daily | Weekly | Monthly | Alert threshold |
|--------|-------|--------|---------|-----------------|
| Containment rate | ✅ | ✅ | ✅ | <70% |
| Handoff rate | ✅ | ✅ | ✅ | >30% |
| ASR accuracy | ✅ | ✅ | ✅ | <90% |
| Function success rate | ✅ | ✅ | ✅ | <95% |
| Average call duration | ✅ | ✅ | ✅ | >2x baseline |
| Call volume | ✅ | ✅ | ✅ | >2x or <0.5x normal |
| Customer satisfaction | - | ✅ | ✅ | <3.5/5 |
| Topic coverage | - | ✅ | ✅ | <80% |

### Setting up alerts

Configure alerts for:

- **Critical errors** - Function failures, API errors
- **Performance degradation** - Latency spikes, timeout increases
- **Volume anomalies** - Unusual call volume
- **Quality issues** - Handoff rate spikes, satisfaction drops

## Proactive issue detection

### Early warning signs

Watch for these patterns:

**Knowledge gaps:**
- Increasing "I don't have that information" responses
- Rising handoff rate for specific topics
- Repeated unanswered questions

**ASR problems:**
- Specific words consistently misheard
- Accuracy drop for certain accents or regions
- Background noise causing issues

**Function issues:**
- Increasing timeout errors
- Authentication failures
- Slow response times

**Voice quality:**
- Pronunciation complaints
- Voice clarity issues
- Inconsistent audio quality

### Trending analysis

Compare week-over-week:

- **Improving** - Containment up, handoffs down, satisfaction up
- **Stable** - Metrics within normal range
- **Degrading** - Any metric trending wrong direction

**If degrading:**
1. Identify when trend started
2. Review changes made around that time
3. Check for external factors (API changes, etc.)
4. Develop and test fix
5. Monitor after fix deployed

## Incident response

### When issues arise

1. **Assess severity**
   - Critical: Affects all users, immediate rollback
   - High: Affects many users, fix within hours
   - Medium: Affects some users, fix within days
   - Low: Minor issue, fix in next release

2. **Immediate action**
   - Critical: Rollback immediately
   - High: Disable affected feature or route to handoff
   - Medium: Monitor closely, prepare fix
   - Low: Add to backlog

3. **Root cause analysis**
   - What happened?
   - When did it start?
   - What changed recently?
   - How many users affected?

4. **Fix and validate**
   - Develop fix in Sandbox
   - Test thoroughly
   - Promote through Pre-release
   - Monitor after Live promotion

5. **Post-mortem**
   - Document what happened
   - Identify prevention measures
   - Update monitoring/alerts
   - Share learnings with team

## Best practices

### Consistency

- **Stick to schedule** - Don't skip health checks
- **Document findings** - Keep a maintenance log
- **Track trends** - Use spreadsheets or dashboards
- **Share with team** - Keep everyone informed

### Efficiency

- **Automate where possible** - Use Test Sets, scheduled analytics
- **Create templates** - Standardize checklists
- **Batch similar tasks** - Group related maintenance
- **Prioritize by impact** - Fix high-impact issues first

### Quality

- **Don't rush** - Thorough checks prevent bigger issues
- **Test before promoting** - Always validate in Pre-release
- **Monitor after changes** - Verify fixes work
- **Learn from issues** - Improve processes continuously

## Common workflows

### Morning routine

1. Check overnight metrics (2 min)
2. Review recent calls (3 min)
3. Test critical functions (2 min)
4. Address any urgent issues (varies)
5. Plan day's maintenance tasks (3 min)

### Weekly review meeting

1. Present key metrics (5 min)
2. Review annotations and issues (10 min)
3. Discuss improvement priorities (10 min)
4. Assign action items (5 min)
5. Plan next week's work (5 min)

### Monthly planning

1. Review previous month's performance (30 min)
2. Identify improvement opportunities (30 min)
3. Plan content updates (30 min)
4. Schedule major changes (15 min)
5. Set goals for next month (15 min)
