The **Connected** tab within the Knowledge area (**Build > Knowledge > Connected**) allows your agent to pull information from external sources like websites, files, and integrations. This page covers how to maintain these sources to ensure your agent always has access to current, accurate information.

## Quick reference

| I need to... | Action | Time estimate |
|--------------|--------|---------------|
| Refresh a single URL source | Click sync icon next to source | 2-5 min |
| Update a file source | Upload new file version | 3 min |
| Add a new source | Add source → Configure → Enable | 5-10 min |
| Disable a source temporarily | Toggle source off in environment | 1 min |
| Remove outdated source | Delete source from list | 1 min |
| Check when source was last synced | View "Last synced" timestamp | 30 sec |

## Understanding source updates

Connected knowledge sources can become outdated when:

- **Website content changes** — product pages, FAQs, or documentation are updated
- **Files are modified** — PDFs, CSVs, or documents are revised
- **Integration data changes** — external systems update their information
- **URLs become inaccessible** — pages move, are deleted, or require new authentication

<Note>Unlike the Managed Topics tab (which you edit directly), Connected knowledge sources must be **synced** to pull in the latest content.</Note>

## How to sync sources

<Steps>
  <Step title="Navigate to Connected knowledge">
    Go to **Build > Knowledge > Connected**.
  </Step>
  <Step title="Find the source">
    Locate the source you want to update.
  </Step>
  <Step title="Click sync">
    Click the **sync icon** next to the source name.
  </Step>
  <Step title="Wait for completion">
    Wait for the sync to complete (typically 1-5 minutes depending on source size). The "Last synced" timestamp will update when complete.
  </Step>
</Steps>

### When to sync

Sync your sources when:

- You know the external content has changed
- You're preparing to publish a new version
- You're troubleshooting incorrect or outdated responses
- It's been more than a week since the last sync (for frequently changing content)
- Before promoting to Pre-release or Live environments

<Tip>Set a regular schedule for syncing high-traffic sources (e.g., weekly for product catalogs, daily for pricing pages).</Tip>

## Managing sources across environments

Connected knowledge sources can be enabled or disabled per environment (Sandbox, Pre-release, Live).

<Steps>
  <Step title="Open Connected knowledge">
    Go to **Build > Knowledge > Connected tab**.
  </Step>
  <Step title="Find the source">
    Locate the source in your list.
  </Step>
  <Step title="Toggle by environment">
    Use the environment toggles to control where the source is active.
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Common environment scenarios" icon="code-branch">
    - **Testing new sources** — enable only in Sandbox until validated
    - **Seasonal content** — disable holiday hours sources after the season ends
    - **Deprecated information** — disable in Live while keeping in Sandbox for reference
    - **Gradual rollout** — enable in Sandbox → Pre-release → Live as you gain confidence
  </Accordion>
  <Accordion title="Source scope and variants" icon="copy">
    If you're using [variant management](/variant-management/introduction), sources can be scoped to specific variants. This allows different locations to reference different knowledge sources (e.g., each restaurant location can have its own menu PDF).

    To scope a source to a variant:
    1. Add or edit a Connected knowledge source
    2. Select which variants should have access to this source
    3. Save and sync
  </Accordion>
</AccordionGroup>

## Troubleshooting sync failures

<AccordionGroup>
  <Accordion title="'Failed to sync' error" icon="triangle-exclamation">
    **Likely cause:** URL is inaccessible or requires authentication.

    **Fix:** Verify URL is publicly accessible or update credentials.
  </Accordion>
  <Accordion title="Source syncs but content seems old" icon="clock-rotate-left">
    **Likely cause:** Cached version being served.

    **Fix:** Wait 5 minutes and sync again; check source URL directly.
  </Accordion>
  <Accordion title="'File too large' error" icon="file-exclamation">
    **Likely cause:** File exceeds size limits.

    **Fix:** Split into smaller files or reduce file size.
  </Accordion>
  <Accordion title="Integration source fails" icon="plug-circle-xmark">
    **Likely cause:** API credentials expired or changed.

    **Fix:** Update integration credentials in settings.
  </Accordion>
  <Accordion title="Partial content synced" icon="file-circle-minus">
    **Likely cause:** Source has restricted sections.

    **Fix:** Ensure entire document is publicly accessible.
  </Accordion>
</AccordionGroup>

### Checking sync status

After syncing, verify the update worked:

<Steps>
  <Step title="Check timestamp">Check the "Last synced" timestamp.</Step>
  <Step title="Test in Agent Chat">Ask a question that should use the updated content.</Step>
  <Step title="Review Diagnosis">Check **Conversation Review → Diagnosis** to see which sources were retrieved.</Step>
  <Step title="Compare content">Compare the agent's response to the actual source content.</Step>
</Steps>

## Best practices

<Tabs>
  <Tab title="File sources" icon="file">
    - **Keep files under 10MB** for faster syncing
    - **Use clear, structured formatting** (headings, bullet points) for better retrieval
    - **Name files descriptively** so you can identify them easily in the source list
    - **Version your files** (e.g., `menu-2025-02.pdf`) to track changes over time
  </Tab>
  <Tab title="URL sources" icon="globe">
    - **Use stable URLs** that won't change or redirect
    - **Avoid dynamic content** that requires JavaScript to load
    - **Test URLs in incognito mode** to ensure they're publicly accessible
    - **Prefer direct content URLs** over landing pages with navigation
  </Tab>
  <Tab title="Integration sources" icon="plug">
    - **Monitor API rate limits** to avoid sync failures
    - **Keep credentials up to date** and rotate them securely
    - **Test integrations in Sandbox** before enabling in Live
    - **Document integration dependencies** for your team
  </Tab>
  <Tab title="General" icon="check">
    - **Audit sources regularly** — remove outdated or unused sources
    - **Document source purposes** — add notes about what each source covers
    - **Sync before major releases** — ensure all sources are current before promoting to Live
    - **Monitor retrieval patterns** — use analytics to see which sources are actually being used
  </Tab>
</Tabs>

## Update vs. sync

<Info>
  - **Sync** = Refresh a Connected knowledge source to pull in the latest external content
  - **Update** = Publish a new version of your agent configuration (including which sources are enabled)

  You need to do both: **sync** the source, **publish** a new version, then **promote** through environments.
</Info>

## Monitoring source freshness

Go to **Build > Knowledge > Connected tab** and review the "Last synced" column. Sort by sync date to identify stale sources.

<Warning>Sources that haven't been synced in 30+ days may contain outdated information. Review and sync regularly.</Warning>

## When to use Connected vs. Managed Topics

| Use the Connected tab when... | Use the Managed Topics tab when... |
|----------------------------------|----------------------------|
| Content lives in external systems | You control the content directly |
| Information changes frequently | Information is relatively stable |
| Multiple sources need to be aggregated | You have a single source of truth |
| Content is maintained by other teams | Your team owns the content |
| You want automatic updates from URLs | You prefer manual control over every change |

## Common workflows

<Tabs>
  <Tab title="Weekly maintenance" icon="calendar-week">
    <Steps>
      <Step title="Review analytics">Identify knowledge gaps from analytics.</Step>
      <Step title="Sync high-traffic sources">Refresh pricing, hours, and FAQ sources.</Step>
      <Step title="Test the agent">Test with recent customer questions.</Step>
      <Step title="Update sources">Add or update sources as needed.</Step>
      <Step title="Publish and promote">Deploy if changes were made.</Step>
    </Steps>
  </Tab>
  <Tab title="Content change response" icon="bolt">
    <Steps>
      <Step title="Identify the source">Find which Connected knowledge source contains the changed content.</Step>
      <Step title="Sync immediately">Sync the source right away.</Step>
      <Step title="Test in Sandbox">Verify the update in Agent Chat.</Step>
      <Step title="Deploy">If urgent, publish and promote to Live. If not urgent, batch with other updates.</Step>
    </Steps>
  </Tab>
  <Tab title="Seasonal content" icon="snowflake">
    <Steps>
      <Step title="Add seasonal source">Add new source with seasonal information.</Step>
      <Step title="Test in Sandbox">Enable only in Sandbox initially and test with seasonal queries.</Step>
      <Step title="Promote through environments">Move to Pre-release for UAT, then Live when the season begins.</Step>
      <Step title="Disable after the season">Disable the source when the season ends — don't delete it, you'll need it next year.</Step>
    </Steps>
  </Tab>
</Tabs>
