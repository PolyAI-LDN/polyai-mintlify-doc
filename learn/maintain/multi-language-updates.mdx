Multilingual agents detect the caller's language, switch languages mid-conversation, and use language-appropriate voices and knowledge. This page covers how to maintain these capabilities.

## Quick reference

| I need to... | Action |
|---|---|
| Add a new language | Add a voice in the target language from the [Voice Library](/voice/voice-library) |
| Update language-specific knowledge | Edit topics with language variants |
| Change voice for a language | **Channels > Voice > Agent Voice** → Change |
| Fix a translation issue | Edit the language-specific response in the topic |
| Test language switching | Agent Chat → switch language mid-call |

## Adding or removing languages

<Tabs>
  <Tab title="Adding a language" icon="plus">
    <Steps>
      <Step title="Select a voice">
        Go to **Channels > Voice > Agent Voice**, click **Change**, and filter by your target language.
      </Step>
      <Step title="Apply the voice">
        Select and apply a voice.
      </Step>
      <Step title="Add language-specific knowledge">
        Create or update topics with language variants (see below).
      </Step>
      <Step title="Update behavior rules">
        Update [behavior rules](/agent-settings/rules) for the new language.
      </Step>
      <Step title="Test and publish">
        Test in Agent Chat, then publish.
      </Step>
    </Steps>

    <Tip>Use a multilingual voice model (such as ElevenLabs multilingual or Cartesia sonic) for proper pronunciation across languages. See [voice classes](/tools/classes/voice) for available providers.</Tip>
  </Tab>
  <Tab title="Removing a language" icon="minus">
    Remove the voice for that language in **Channels > Voice > Agent Voice**, then update any language-specific knowledge and functions. The agent will fall back to the default language for callers speaking the removed language.
  </Tab>
</Tabs>

## Language-specific knowledge

<AccordionGroup>
  <Accordion title="Managed Topics with language variants" icon="book" defaultOpen={true}>
    <Steps>
      <Step title="Open Managed Topics">Go to **Build > Knowledge > Managed Topics**.</Step>
      <Step title="Create or edit a topic">Select or add a topic.</Step>
      <Step title="Add language variants">Add **language variants** for each supported language with translated or culturally-appropriate content.</Step>
      <Step title="Save and test">Save and test in Agent Chat.</Step>
    </Steps>
  </Accordion>
  <Accordion title="Connected knowledge per language" icon="link">
    Add separate sources per language:
    - **URL sources** — add the language-specific version of your website
    - **File sources** — upload language-specific documents
    - **Integration sources** — configure APIs to return language-specific data
  </Accordion>
  <Accordion title="Shared vs. language-specific knowledge" icon="scale-balanced">
    | Use shared knowledge when... | Use language-specific knowledge when... |
    |---|---|
    | Information is universal (phone numbers, addresses) | Content needs translation |
    | Data is language-agnostic (product IDs, prices) | Cultural context matters |
    | Maintaining multiple versions is impractical | Local regulations differ by region |
  </Accordion>
</AccordionGroup>

## Updating voices per language

<Info>
  - **Use native voices** — don't use an English voice for Spanish
  - **Match regional accents** — Mexican Spanish for Mexico, Castilian for Spain
  - **Test pronunciation** of language-specific characters
  - Multilingual TTS models are convenient but may have slightly lower quality than language-specific models
</Info>

## Language detection and switching

By default, the agent detects the caller's language automatically via ASR and responds in that language.

If detection is too aggressive or not sensitive enough, adjust via [behavior rules](/agent-settings/rules) — for example, instruct the agent to only switch after the caller has spoken consistently in a different language for multiple turns.

For explicit selection, create a Managed Topic that lets callers choose their language and use a function to set it:

```python
def set_language(language_code):
    conv.set_language(language_code)
    return {"utterance": f"Switching to {language_code}."}
```

## Language-specific functions

Access the current language in functions with `conv.language`:

```python
def dynamic_response():
    if conv.language == "es":
        return {"utterance": "Respuesta en español"}
    return {"utterance": "Response in English"}
```

## Fixing translation issues

<AccordionGroup>
  <Accordion title="Awkward phrasing" icon="pen">
    Rewrite for natural language — localize, don't just translate.
  </Accordion>
  <Accordion title="Cultural mismatches" icon="globe">
    Use culturally-appropriate equivalents for idioms and expressions.
  </Accordion>
  <Accordion title="Incorrect terminology" icon="book-open">
    Use domain-specific terms with a glossary.
  </Accordion>
  <Accordion title="Formatting issues" icon="calendar">
    Localize date/time/number formats per language.
  </Accordion>
</AccordionGroup>

<Tip>For bulk translation updates, export Managed Topics as CSV, send to professional translators with context and a domain glossary, then import the translated CSV.</Tip>

## Testing

<Steps>
  <Step title="Open Agent Chat">Select a language to test.</Step>
  <Step title="Verify responses">Confirm the agent responds correctly in the target language.</Step>
  <Step title="Test language switching">Switch languages mid-conversation and confirm detection works.</Step>
  <Step title="Get native speaker feedback">Test with native speakers for naturalness and cultural appropriateness.</Step>
</Steps>

## Maintenance checklist

- **Monthly:** Audit content parity across languages
- **Quarterly:** Review voices for naturalness
- **Ongoing:** Update all languages together — don't let translations lag behind
