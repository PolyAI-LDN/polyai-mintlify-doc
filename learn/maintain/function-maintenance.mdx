Update function code, debug errors, rotate credentials, and optimize performance.

## Quick reference

| I need to... | Action |
|---|---|
| Update function code | Edit in Function Editor → Save → Test in Agent Chat |
| Debug a function error | **Analytics > Conversations** → Diagnosis → check logs |
| Update API credentials | Workspace homepage > Secrets tab → Edit → Save |
| Add logging | Use `conv.log.info()` in code |
| Fix function not triggering | Review KB action or rules → clarify description |
| Update API integration | **Configure > APIs** → edit endpoint |

## Updating function code

<Steps>
  <Step title="Open the function">
    Go to **Build > Tools** and select the function.
  </Step>
  <Step title="Edit the code">
    Edit the Python code in the Function Editor.
  </Step>
  <Step title="Save and test">
    **Save**, then test in Agent Chat.
  </Step>
  <Step title="Review logs">
    Review logs in **Conversation Review → Diagnosis**.
  </Step>
  <Step title="Publish">
    Publish when satisfied.
  </Step>
</Steps>

<Tip>Always test function changes in Sandbox before promoting to Live.</Tip>

### Example: updating an API endpoint

```python
def book_reservation(date, time, party_size, special_requests=None):
    conv.log.info(f"Booking for {party_size} guests on {date} at {time}")

    response = requests.post(
        "https://api.example.com/v2/bookings",
        json={"date": date, "time": time, "guests": party_size,
              "special_requests": special_requests or ""},
        headers={"Authorization": f"Bearer {conv.utils.get_secret('booking_api_key')}"}
    )

    if response.status_code == 200:
        conv.log.info("Booking successful")
        return {"utterance": "Your reservation is confirmed."}
    else:
        conv.log.error(f"Booking failed: {response.text}")
        return {"utterance": "I'm having trouble completing your reservation. Let me transfer you to someone who can help."}
```

## Debugging

### Using conv.log

```python
conv.log.info("Starting payment processing")       # general flow
conv.log.warning("Customer account has low balance") # potential issues
conv.log.error(f"Payment API returned: {error}")     # failures
conv.log.info("Processing order", pii=True)          # sensitive data
```

Logs appear in **Conversation Review → Diagnosis**, **Agent Chat** (during testing), and the [Conversations API](/api-reference/conversations/introduction).

### Common debugging steps

<Steps>
  <Step title="Review Diagnosis logs">
    Check the logs for the failing function in **Conversation Review → Diagnosis**.
  </Step>
  <Step title="Reproduce in Agent Chat">
    Trigger the same scenario in Agent Chat.
  </Step>
  <Step title="Check function inputs">
    Are parameters being passed correctly by the LLM?
  </Step>
  <Step title="Validate external APIs">
    Test APIs directly (Postman, curl) to isolate the issue.
  </Step>
  <Step title="Review the description">
    Is the function description clear about when it should trigger?
  </Step>
</Steps>

### Common errors

<AccordionGroup>
  <Accordion title="Function not triggering" icon="circle-xmark">
    **Likely cause:** Unclear description or KB action.

    **Fix:** Simplify description; clarify when to call it.
  </Accordion>
  <Accordion title="Wrong parameters" icon="circle-question">
    **Likely cause:** LLM misunderstanding.

    **Fix:** Improve parameter names and descriptions.
  </Accordion>
  <Accordion title="Timeout" icon="hourglass-half">
    **Likely cause:** Slow API or complex logic.

    **Fix:** Add [delay controls](/tools/delay-control); optimize code.
  </Accordion>
  <Accordion title="Auth failures" icon="lock">
    **Likely cause:** Expired credentials.

    **Fix:** Update [secrets](/secrets/introduction).
  </Accordion>
  <Accordion title="Import errors" icon="box-open">
    **Likely cause:** Missing library.

    **Fix:** Check [available libraries](/tools/import-library).
  </Accordion>
</AccordionGroup>

## Managing secrets

When API keys or credentials change:

<Steps>
  <Step title="Open Secrets">
    Go to the **Secrets** tab on the workspace homepage.
  </Step>
  <Step title="Edit the secret">
    Find and edit the secret value.
  </Step>
  <Step title="Test dependent functions">
    Save, then test all functions using this secret.
  </Step>
</Steps>

```python
api_key = conv.utils.get_secret("my_api_key")
headers = {"Authorization": f"Bearer {api_key}"}
```

<Info>
  - Rotate credentials every 90 days
  - Use descriptive names (`stripe_live_api_key`, not `key1`)
  - Test all dependent functions after rotation
</Info>

## Optimizing performance

<AccordionGroup>
  <Accordion title="Add delay controls" icon="hourglass-half" defaultOpen={true}>
    Use [delay controls](/tools/delay-control) with filler phrases to keep the caller engaged while processing.

    <Tip>Reference state variables in delay responses using `$`. For example: `Still checking availability at $branch_name...`</Tip>
  </Accordion>
  <Accordion title="Cache frequently-accessed data" icon="database">
    Store common lookups to avoid repeated API calls.
  </Accordion>
  <Accordion title="Reduce unnecessary API calls" icon="bolt">
    Batch operations where possible and skip redundant requests.
  </Accordion>
  <Accordion title="Simplify logic" icon="broom">
    Remove unnecessary processing and streamline code paths.
  </Accordion>
</AccordionGroup>

## Improving function triggering

If the agent isn't calling your function when expected:

<Steps>
  <Step title="Simplify the description">
    Make it clear when to call the function.
  </Step>
  <Step title="Update KB actions">
    Ensure topics reference the function correctly.
  </Step>
  <Step title="Check for conflicts">
    Ensure other functions aren't being called instead.
  </Step>
</Steps>

<Tabs>
  <Tab title="Bad description" icon="xmark">
    "Handles reservations"
  </Tab>
  <Tab title="Good description" icon="check">
    "Call this function when the user wants to book a table. Required: date, time, party_size. Only call after confirming all three with the user."
  </Tab>
</Tabs>
