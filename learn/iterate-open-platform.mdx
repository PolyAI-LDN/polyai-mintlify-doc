The first version of your agent is never the final one. Real callers say things your prompts didn't anticipate. They mispronounce things. They go off-script. The fastest way to get to a good agent is to read the transcripts, spot what went wrong, and ask Studio Assistant to fix it. Then send the share link to someone else and do it again.

Here's the loop.

## Step 1: Find the call you want to look at

Open **Conversations** in the sidebar. You'll see every call (and every test call) the agent has handled. Sort by date, filter by environment, or scan the PolyScore column for the lowest-rated calls.

Click into one. The right side opens the **Conversation Review** panel: transcript on the left, debugging tabs on the right.

## Step 2: Read the transcript

Pay attention to:

- **Where the agent gave a wrong answer.** Look for places where the customer pushed back, repeated themselves, or asked the same question twice.
- **Where the agent got stuck.** Long silences, "I'm sorry, I didn't understand", or repeated requests for the same piece of info usually mean a flow didn't validate input properly.
- **Where the agent went off-topic.** If it answered something it shouldn't have, that's a guardrail issue or a topic that needs tightening.
- **What the caller wanted that you didn't expect.** Sometimes the failure isn't that the agent did the wrong thing; it's that there isn't a flow for what the caller actually wanted yet.

The **Diagnosis** tab on the right shows what matched (topics, flows, function calls) for each turn. That's how you know *why* the agent did what it did.

## Step 3: Ask Studio Assistant to fix it

Go back to the home screen and ask in plain language. Specifics help:

- *"The agent told the caller we offer same-day delivery, but we don't. Update the shipping topic to say next-day at best, and add a confirmation step before quoting any delivery time."*
- *"On the booking flow, the agent kept asking for the date even after the caller said 'tomorrow'. Fix the entity extraction to handle relative dates."*
- *"Add a new topic for 'where is my refund' that says we process refunds in 5-10 business days back to the original payment method."*
- *"The welcome message is too formal. Make it warmer, like a friendly receptionist."*

Studio Assistant reads your project, proposes a plan, and shows you what'll change. Approve the plan, the change lands on a branch, you can merge to `Main` when you're happy.

## Step 4: Test the fix

Open the agent in the browser (the **Test** button in the top-right) and replay the scenario that broke. If it works, promote the version to Live. If not, ask Studio Assistant for another revision.

## Step 5: Send the link to a fresh pair of ears

You know what you fixed, so you're biased about whether it sounds right. Send the share link to a colleague, your customer, or even a friend who isn't in the loop. They'll find the next thing.

## Common patterns

**The agent answers a question it shouldn't have.** Add a guardrail in **Build > Agent > Behavior**, or tighten the topic that the question matched against.

**The agent misses a topic that should match.** Open the topic in **Build > Knowledge > Managed Topics** and add more sample questions. Three to five varied phrasings is usually enough.

**The agent collects the right info but doesn't do anything with it.** That's a missing action on the topic, or a missing handoff step. Ask Studio Assistant to wire it up.

**The agent says weird things.** Usually means the LLM is filling in gaps because the topic content was thin. Either add more detail to the topic, or add a rule in **Behavior** that constrains what the agent's allowed to say.

**The voice is off.** Switch voices under **Configure > Agent voice**. It's a one-line change and you can test it on the next call.

## Don't do this in one big batch

The temptation is to gather ten bug reports and ask Studio Assistant to fix them all at once. Don't. One change at a time is easier to plan, easier to review, easier to roll back. The fair-use allowance favors small focused requests, too.
