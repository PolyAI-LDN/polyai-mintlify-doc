The **March 2026** PolyAI Agent Studio release improves how teams review and debug conversations, with deeper visibility into source content, LLM prompts, and large-scale conversation analysis.

Expand the items below for details:

<AccordionGroup>

<Accordion title="Source Hub file preview in Conversation Review" icon="file-magnifying-glass" id="source-hub-file-preview">

You can now view source documents directly in the **Conversation Review** page. When a conversation matches a Connected Knowledge source, the relevant file — such as a PDF — is displayed inline alongside the citation.

**What's new:**
- Preview source files (for example, PDFs) directly in Conversation Review.
- See the exact document content retrieved for a given conversation turn.
- Available for knowledge sources added through **Connected Knowledge** (Source Hub).
- Displayed alongside existing topic citation information on the **Diagnosis** tab.

**Why it matters:**
- Faster debugging when retrieved content is unexpected or incorrect.
- No need to leave Conversation Review to check the source document.
- Clearer understanding of what the agent was working with during a turn.

</Accordion>

<Accordion title="LLM prompt visibility" icon="code" id="llm-prompt-visibility">

The **LLM Request** section in Conversation Review now displays the raw prompt in a cleaner, more readable format.

**What's new:**
- Improved formatting for the LLM prompt text, with markdown-style rendering.
- Tidied whitespace and structure for easier scanning.
- Applies to all turns visible in the **Diagnosis** tab.

**Why it matters:**
- Easier to spot what instructions or context were passed to the LLM for a given turn.
- Supports faster root-cause analysis when agent responses are unexpected.

</Accordion>

<Accordion title="Smart Analyst deep sampling" icon="chart-scatter" id="deep-sampling">

Smart Analyst now uses a sub-agent to analyze up to **500 conversations** for patterns — a 10× increase from the previous 50-conversation limit.

**What's new:**
- Analyze up to **500 conversations** per query (previously 50).
- Conversations are sampled either **randomly** or **based on a metric / PolyScore**.
- Ask questions like *"What do customers complain about?"* or *"How can we reduce handoffs where users ask to speak to a human?"*
- Get **directional insights on behavior frequency** where no metric is present.
- Deep sampling helps uncover patterns that smaller samples might miss.

**Example use cases:**
- Identify common complaint themes across a large sample
- Analyze handoff reasons with statistically meaningful data
- Discover edge cases and rare conversation patterns
- Validate hypotheses about customer behavior at scale

</Accordion>

</AccordionGroup>
