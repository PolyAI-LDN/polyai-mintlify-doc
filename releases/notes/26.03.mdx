The **March 2026** PolyAI Agent Studio release introduces MCP tool connections, a public Deployments API, new integration support, and improvements across Smart Analyst, voice, and no-code flows.

Expand the items below for details:

<AccordionGroup>

<Accordion title="MCP tool connections" icon="plug" id="mcp-tool-connections">

You can now connect [MCP](https://modelcontextprotocol.io/) (Model Context Protocol) servers to your agent, giving it access to external tools and data sources during conversations.

**What's new:**
- Discover and browse available tools from connected MCP servers.
- Filter which tools your agent can use per server.
- Supports multiple authentication methods including API key and OAuth.
- Configure MCP connections alongside your existing [knowledge](/managed-topics/introduction) and [functions](/function/introduction).

</Accordion>

<Accordion title="Public Deployments API" icon="rocket" id="public-deployments-api">

A new public Deployments API lets you manage environment promotions programmatically, so you can integrate deployment workflows into your existing CI/CD pipelines.

**What's new:**
- List current deployments across sandbox, pre-release, and live environments.
- Promote or roll back deployments via authenticated API calls.
- Publish changes to target environments without using the Studio UI.

See the [API reference](/api-reference/introduction) for endpoint details.

</Accordion>

<Accordion title="NICE CXone integration" icon="headset" id="nice-cxone-integration">

Agent Studio now supports **NICE CXone DFO** as a handoff destination for chat conversations, expanding the range of contact center platforms you can route to.

**What's new:**
- Hand off chat conversations to NICE CXone agents.
- Configure the integration from the [Integrations](/integrations/introduction) page.
- Handoff state is tracked end-to-end, with clean session termination when the handoff completes.

</Accordion>

<Accordion title="Webhook improvements" icon="bell" id="webhook-improvements">

Webhooks are now more flexible and secure, with secret rotation and resolved-alert notifications.

**What's new:**
- **Rotate signing secrets** without deleting and recreating the endpoint. See the [Webhooks API](/api-reference/webhooks/introduction) for details.
- **`alerts.resolved` event** — receive a webhook when an alert recovers from a triggered state back to normal.
- **Resource limits** — up to 10 alert rules and 10 webhook endpoints per account, enforced server-side.

</Accordion>

<Accordion title="Smart Analyst updates" icon="chart-scatter" id="smart-analyst-updates">

Smart Analyst gains broader channel coverage, new language support, and built-in CSAT awareness.

**What's new:**
- **Webchat channel** — run Smart Analyst queries against webchat conversations in addition to voice and chat.
- **CSAT metrics included by default** — CSAT survey results are now surfaced automatically in Smart Analyst analyses.
- **Brazilian Portuguese** — Smart Analyst now supports pt-BR as a test language.
- **Configurable deep sampling prompts** — customize the prompt used for deep sampling analyses.

</Accordion>

<Accordion title="Voice and multilingual improvements" icon="microphone" id="voice-multilingual-improvements">

March delivers smoother voice experiences for multilingual agents and long-running function calls.

**What's new:**
- **Automatic voice switching** — in multilingual projects, the TTS voice now switches automatically when the conversation language changes.
- **Filler utterances across consecutive functions** — filler timing is now tracked across back-to-back function calls, so cumulative silence correctly triggers filler utterances instead of leaving gaps.
- **ASR corrections in no-code flows** — [transcript corrections](/speech-recognition/introduction) and ASR biasing now apply to steps built with the [no-code flow editor](/flows/no-code/introduction).

</Accordion>

<Accordion title="Dashboard and analytics updates" icon="chart-line" id="dashboard-analytics-updates">

Dashboards now respect your local timezone, and agent memory gains more flexibility.

**What's new:**
- **Local timezone support** — dashboard charts and aggregations now use your local timezone instead of UTC, including correct handling of daylight saving time transitions.
- **Custom agent memory keys** — configure [agent memory](/function/classes/agent-memory) to look up returning callers by SIP header, integration attribute, or conversation state instead of phone number alone.

</Accordion>

<Accordion title="Bug fixes" icon="wrench" id="bug-fixes">

- Fixed an issue where running all test cases at once could fail due to a pagination error.
- Improved reliability of Zendesk integration actions.

</Accordion>

</AccordionGroup>
