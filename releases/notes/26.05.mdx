The **May 2026** PolyAI Agent Studio release brings the redesigned **Conversations** workspace to general availability, ships **PolyScore v2** across voice and webchat, surfaces Raven Omni voice-agent runtime data in Conversation Review, and adds **Tripleseat** as a one-click integration for restaurant agents.

Expand the items below for details:

<AccordionGroup>

<Accordion title="Conversations workspace revamp" icon="comments" id="conversations-revamp">

The Conversations page is now organized around channel scopes, a Views system, and a side-panel review experience – purpose-built for high-volume, repeatable analysis.

**What's new:**

- **Voice and Webchat scopes** – the page opens with separate pinned tabs for **Voice** and **Webchat**. Each scope has its own column registry (webchat omits voice-only columns like direction, outbound call status, and CSAT) and its own saved views.
- **Views system** – save filter and column combinations as **Custom Views**. Pinned tabs at the top of the table make it a single click to switch contexts. Built-in **System Views** (Production and Test) cover the two most common starting points and cannot be edited or deleted.
- **Share filters as link** – generate a shareable URL for any view (<kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>L</kbd>). Recipients see a banner letting them apply the filters temporarily or save them as a new view, so shared links never silently override their setup.
- **Duplicate, rename, set as default** – three-dot menu on every pinned tab, with <kbd>Ctrl</kbd>+<kbd>D</kbd> to duplicate. View names are limited to 25 characters; up to 50 Custom Views per project per scope.
- **Integrated review side panel** – clicking a row opens a 4-tab side panel (**Transcription / Scores / Details / Custom metrics**) over the table. No more bouncing between the table and a full-page review – context, sorting, and filters stay in place.

See [Conversation review](/analytics/conversations/review) and the new [Views](/analytics/conversations/views) page for the full walkthrough.

</Accordion>

<Accordion title="Diagnosis as a toggle group" icon="stethoscope" id="diagnosis-toggle-group">

The Diagnosis dropdown is now a **toggle group** on the Transcription tab, so you can layer multiple debugging views on top of the transcript at once.

**What's new:**

- **Layered toggles** – flip Conversation variables, Flows and steps, Tool calls, Topic citations, Sources, Transcript corrections, Turn latency, Interruptions, Variants, and Logs on or off independently.
- **Logs layer** – surfaces structured `conv.log.info()`, `conv.log.warning()`, `conv.log.error()`, and `conv.log_api_response()` output from your functions directly under the relevant turn. See [Conversation log](/tools/classes/conv-log).
- **Sources panel** – clicking a Source tag opens an inline preview of the retrieved content with **Open in Knowledge** to jump straight to the source file.

See [Diagnosis](/analytics/conversations/diagnosis) for every available layer.

</Accordion>

<Accordion title="PolyScore v2" icon="star" id="polyscore-v2">

[PolyScore](/analytics/polyscore) has been rebuilt with simplified, less-biased ratings and now scores webchat conversations alongside voice.

**What's new:**

- **Omni-channel scoring** – the same PolyScore badge and breakdown are available for both voice calls and webchat sessions.
- **Simplified ratings** – the underlying scoring rubric was reworked to remove dimensions that introduced bias. The 0–10 scale and color coding (green / amber / red) are unchanged.
- **Smart Analyst integration** – Smart Analyst now references normalized PolyScore values when reasoning about conversation quality.

The **Scores** tab in the review panel groups PolyScore and CSAT together with their per-dimension breakdowns.

</Accordion>

<Accordion title="Raven Omni runtime parity in Conversation Review" icon="microphone-lines" id="raven-omni-parity">

Conversations from the new voice-agent runtime (Raven Omni / voice agent v2) now appear in Conversation Review with the same diagnostic data as v1 calls.

**What's new:**

- **RAG matched topics** – knowledge base matches now appear under each turn for v2 calls. Previously, v2 calls used a different runtime store that did not write the metadata Conversation Review reads.
- **Function and flow events (rolling out)** – function calls and flow transitions are being wired into the Diagnosis panel for v2 calls, matching the v1 experience. Some accounts may see staged availability during the rollout.
- **Improved barge-in handling** – fixes a regression where a barged-in agent utterance could remain in the conversation history; the displayed transcript now reflects what the caller actually heard.

</Accordion>

<Accordion title="Tripleseat integration" icon="champagne-glasses" id="tripleseat-integration">

Connect PolyAI to **Tripleseat** to capture event and large-party leads from voice and webchat conversations without writing custom code.

**What's new:**

- **One-click connection** – on the Integrations page, find **Tripleseat** and click **Connect** to authorize PolyAI through OAuth 2.0.
- **Large-party handling** – set a max party number under your site settings; when a caller asks for a party larger than that threshold, the agent collects event details and creates a lead in Tripleseat instead of attempting a standard reservation.
- **Multi-channel capture** – the same lead capture works on **Voice** (callers speak event details), **Webchat** (guided conversation), and **SMS** (for contact details that are awkward to capture over voice).
- **Restaurant template support** – projects built from the PLG restaurant template can connect Tripleseat directly from Agent Studio. Other projects continue to be supported through Managed Services.

See [Tripleseat](/integrations/tripleseat) for the full setup guide.
