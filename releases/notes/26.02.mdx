The **February 2026** PolyAI Agent Studio release focused on telephony integrations, multi-channel configuration, webchat rendering, safer version promotion, and a range of improvements to testing, configuration, and review workflows.

Expand the items below for details:

<AccordionGroup>

<Accordion title="Channels and navigation reorganisation" icon="bars-staggered" id="channels-navigation-reorg">

<iframe src="https://www.loom.com/embed/6492739064e644668f459f75b9c9643f" title="Loom video player" frameborder="0" className="w-full aspect-video rounded-xl" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" allowfullscreen />

Agent Studio now has a dedicated **Channels** section in the left-hand navigation, making it easier to manage channel-specific configuration separately from shared project assets.

This update introduces a clearer information architecture for multi-channel projects, with dedicated spaces for **Voice**, **Chat**, and **Widget** configuration while keeping shared resources such as flows, knowledge base content, and variants elsewhere in the project.

**What's new:**
- New **Channels** section added to the sidebar below **Build**.
- Channel-specific configuration now lives in dedicated pages under **Channels**.
- **Chat configuration** was moved out of **Settings** and into the new **Channels** section.
- Greeting and disclaimer settings were moved from the **Agent** page into dedicated **Voice Configuration** and **Chat Configuration** pages.
- **Widget** configuration for webchat now lives under **Channels**.
- The new structure supports future channels being added in the same section.

**Other navigation changes:**
- New **Deployments** section created in the sidebar.
- **Environments** moved into **Deployments**.
- **Project History** moved into **Deployments**.
- **Real-time Configuration** moved into **Build**.
- **Configuration Builder** moved into **Build**.
- User permissions were updated to reflect the new sidebar structure.

</Accordion>

<Accordion title="Webchat enhancements" icon="message" id="webchat-markdown">

![chat-config](/images/release-notes/2602/chat-config.png)

February brings lot of updates to Webchat deployment, along with usability and rendering improvements.

**Chat configuration:**
- Chat can now be enabled from the **Chat Configuration** page under **Channels**.
- **Widget** configuration is managed separately and pushes changes directly to environments.
- In-product banners guide users to enable chat before configuring the widget.
- Chat configuration aligns with branching behaviour.

**Markdown rendering:**
- Agent responses render markdown correctly in the **Test Agent Chat** panel.
- Webchat conversations display formatted markdown in **Conversation Review**.
- Supports **bold**, *italics*, lists, links, and code blocks.

</Accordion>

<Accordion title="Easy telephony integrations" icon="phone" id="ccaas-integrations">

![integrations](/images/release-notes/2602/integrations.png)

Agent Studio now supports direct integrations with selected CCaaS providers, making it easier to connect voice agents to existing contact center infrastructure.

This reduces setup friction for teams that want to route calls through existing telephony systems rather than manage telephony separately.

**What's new:**
- Click-based integration support for **Dialpad**, **Twilio**, and **Five9** directly from the **Integrations** page.
- After setup, users are directed to the **Handoffs** page to complete routing configuration.
- The **Handoffs** page now reflects telephony integration options for more unified call routing management.

**Implementation notes:**
- **Zendesk** support was planned but did not fully ship in this release due to authentication-related changes.

</Accordion>

<Accordion title="Compare versions and environments" icon="code-compare" id="version-comparison">

![chat-config](/images/release-notes/2602/compare-versions.png)

A new comparison tool helps teams review differences between versions and environments before promoting changes.

This makes it easier to understand what changed between environments and reduces the risk of promoting the wrong configuration.

**What's new:**
- Compare **versions and environments side-by-side** before promotion.
- Review changes across **Sandbox**, **Pre-release**, and **Live** environments.
- Improved visibility into version history and environment state.
- Updated **Versions** page with clearer ordering and navigation.
- Versions now appear in **reverse chronological order**, with the newest versions first.

</Accordion>

<Accordion title="In-app calling with variants" icon="phone-volume" id="in-app-call-variants">

![chat-config](/images/release-notes/2602/in-app-calling-variants.png)

In-app test calling now supports variant selection, making it easier to validate variant-specific behaviour directly in Agent Studio.

This is particularly useful for teams running multi-site or multi-configuration deployments from a single project.

**What's new:**
- Select a specific [variant](/variant-management/introduction) when making in-app test calls.
- Test variant-specific content, routing, and responses without switching phone numbers.
- The variant selector only appears when a project has variants configured.
- Call settings are grouped into a collapsible **Call configuration** section for a cleaner testing experience.
- Variant naming is now more consistent across the product, including on the **Numbers** page, where the variant name now matches the site name.

</Accordion>

<Accordion title="Voice and conversation flow improvements" icon="microphone" id="voice-and-conversation-flow">

Voice configuration and conversation review now better reflect how agents behave in practice.

These updates improve consistency across channels and make delay-related behaviour easier to inspect during QA.

**What's new:**
- Voice configuration now syncs more consistently across channels.
- **Disclaimer voice** updates automatically when the main agent voice changes.
- Transition utterances and filler phrases now appear **inline** with the agent’s final response in **Conversation Review**.
- This makes [high-latency functions](/function/delay-control) easier to review and debug.

</Accordion>

<Accordion title="Security and usability updates" icon="wand-magic-sparkles" id="security-usability-updates">

February also included a range of smaller improvements across testing, review, and in-product guidance.

Together, these changes improve day-to-day usability while tightening default protections for sensitive information.

**What's new:**
- Increased the maximum number of **sample questions per topic** from **10 to 20**.
- The **Live** filter was removed from conversations for a more unified review experience.
- For projects without live calls, the conversations page now shows all environments by default.
- **Secret values** are now masked in the Secrets UI by default.
- Click into a secret value field to unmask and edit it, then click away to re-mask it.
- Added a **copy-to-clipboard** button for secret values.

</Accordion>
