The **February 2026** PolyAI Agent Studio release introduces a new multi-channel navigation structure, expands Webchat capabilities, and improves telephony, version control, and testing workflows.

Expand the items below for details:

<AccordionGroup>

<Accordion title="Channels and navigation reorganisation" icon="bars-staggered" id="channels-navigation-reorg">

<iframe src="https://www.loom.com/embed/6492739064e644668f459f75b9c9643f" title="Loom video player" frameborder="0" className="w-full aspect-video rounded-xl" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" allowfullscreen />

Agent Studio now includes a dedicated **Channels** section in the sidebar. It separates channel-specific configuration from shared project assets such as flows, knowledge base content, and variants.

**What’s new:**
- Added a new **Channels** section below **Build**.
- Created dedicated pages for:
  - **Voice Configuration**
  - **Chat Configuration**
  - **Widget Configuration**
- Moved **Chat configuration** from **Settings** into **Channels**.
- Moved greeting and disclaimer settings from the **Agent** page into **Voice** and **Chat** configuration pages.
- Placed **Widget** configuration under **Channels**.
- Structured the section to support additional channels in future.

**Navigation updates:**
- Added a new **Deployments** section.
- Moved **Environments** and **Project History** into **Deployments**.
- Moved **Real-time Configuration** and **Configuration Builder** into **Build**.
- Updated user permissions to reflect the new structure.

</Accordion>

<Accordion title="Webchat enhancements" icon="message" id="webchat-markdown">

![chat-config](/images/release-notes/2602/chat-config.png)

February expands Webchat deployment and improves chat configuration and rendering.

**Chat configuration:**
- Enable chat from the **Chat Configuration** page under **Channels**.
- Manage **Widget** configuration separately and push changes directly to environments.
- Display in-product banners to guide chat enablement before widget setup.
- Align chat configuration with branching behaviour.

**Markdown rendering:**
- Render formatted markdown in the **Test Agent Chat** panel.
- Display formatted markdown in **Conversation Review**.
- Support **bold**, *italics*, lists, links, and code blocks.

</Accordion>

<Accordion title="Easy telephony integrations" icon="phone" id="ccaas-integrations">

![integrations](/images/release-notes/2602/integrations.png)

Agent Studio now supports click-based integrations with leading CCaaS providers.

**What’s new:**
- Integrate **Dialpad**, **Twilio**, and **Five9** directly from the **Integrations** page.
- Route directly to the **Handoffs** page after setup.
- Update the **Handoffs** page to reflect telephony integrations for unified routing.

**Note:** Zendesk support was planned but did not ship in this release due to authentication changes.

</Accordion>

<Accordion title="Compare versions and environments" icon="code-compare" id="version-comparison">

![chat-config](/images/release-notes/2602/compare-versions.png)

Compare versions and environments before promoting changes.

**What’s new:**
- View side-by-side differences across **Sandbox**, **Pre-release**, and **Live**.
- Show versions in **reverse chronological order** (newest first).
- Improve version history visibility and navigation on the **Versions** page.

</Accordion>

<Accordion title="In-app calling with variants" icon="phone-volume" id="in-app-call-variants">

![chat-config](/images/release-notes/2602/in-app-calling-variants.png)

Test calls now support variant selection directly in Agent Studio.

**What’s new:**
- Select a specific [variant](/variant-management/introduction) when making test calls.
- Display the variant selector only when variants exist.
- Group call settings in a collapsible **Call configuration** section.
- Align variant naming across the **Numbers** page.

</Accordion>

<Accordion title="Security and usability updates" icon="wand-magic-sparkles" id="security-usability-updates">

February also delivers smaller refinements across testing and review.

**What’s new:**
- Increase sample question limit from **10 to 20** per topic.
- Remove the **Live** filter in Conversations for unified review.
- Mask secret values by default in the **Secrets** UI.
- Unmask on focus and re-mask on blur.
- Add copy-to-clipboard support for secret values.

</Accordion>
