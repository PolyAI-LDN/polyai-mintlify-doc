The **April 2026** PolyAI Agent Studio release adds MCP integrations for connecting external tools and a new multilingual configuration UI for managing languages, voices, and translations without custom code.

Expand the items below for details:

<AccordionGroup>

<Accordion title="MCP integrations" icon="plug" id="mcp-integrations">

Agent Studio now supports [Model Context Protocol](https://modelcontextprotocol.io/) (MCP) integrations, letting you connect external tool servers to your agent directly from the UI.

**What's new:**
- A dedicated **MCP** tab is now available under **Configure > Integrations**.
- Add an MCP server by providing a URL, authentication details (Header, Query parameter, or OAuth), and an optional timeout.
- Agent Studio **auto-discovers** the tools available on the connected server and displays them in a configuration panel.
- **Toggle individual tools** on or off to control which functions are exposed to the agent, helping manage context window usage.
- Supports **Header**, **Query parameter**, and **OAuth** authentication methods using secrets from the Secrets Vault.

**Why it matters:**
- Connect your agent to external systems without writing custom functions or API configurations.
- Reuse MCP-compatible tools across multiple agents.
- Control exactly which tools are available to your agent to keep prompts focused and efficient.

<Note>
Projects that previously configured MCP through experimental config will continue to work, but those servers won't appear in the new UI. Configuring a new server through the UI will replace the experimental config. See [MCP integrations](/integrations/mcp) for setup instructions.
</Note>

</Accordion>

<Accordion title="Multilingual configuration" icon="language" id="multilingual-configuration">

Agent Studio now includes a dedicated UI for configuring multilingual agents. You can add languages, assign per-language voices, manage translation overrides, and set pronunciation rules — all without editing function code.

**What's new:**
- **Add languages in the UI** — add up to 10 additional languages per project from the Agent settings page, replacing the previous start function workaround.
- **Per-language voice selection** — configure separate voices for the **Agent** voice and **Disclaimer** voice in each language, with multi-voice support per language.
- **Auto voice switching** — the agent automatically switches to the configured voice when the conversation language changes. Unsupported languages keep the current voice.
- **Language Hub** — a new page for managing translation overrides where auto-translation isn't accurate enough.
  - Create **translation cards** with auto-translate and optional manual overrides per language.
  - Insert translations using the action menu in greetings, disclaimers, behavior, prompts, delay control responses, and SMS templates.
  - Reference translations in functions using `conv.translations.{language_key}`.
- **Per-language pronunciation rules** — configure pronunciation corrections specific to each language.
- **Language visibility** — language information now appears in **Conversation Review** and **Audio Management**.

**Why it matters:**
- Configure multilingual agents through the UI instead of managing state variables and conditional logic in code.
- Override specific translations where the LLM auto-translation doesn't match cultural nuances — for example, replacing "Please hold" with "Please bear with me" for British English.
- Assign appropriate voices per language so the agent sounds natural in every supported language.

</Accordion>
