The **April 2026** PolyAI Agent Studio release adds multi-language support with a dedicated UI, a new Translations page for fine-tuning multilingual content, and MCP integrations for connecting external tools to your agent.

Expand the items below for details:

<AccordionGroup>

<Accordion title="Multi-language support" icon="globe" id="multilingual">

Agent Studio now supports configuring multilingual agents directly from the UI, replacing the previous approach of defining languages in `start_function` or maintaining separate projects per language.

**What's new:**
- **Add languages in the UI** — go to **Build > Agent > Multi-language** to add up to 10 additional languages. No code changes required.
- **Per-language voice configuration** — configure separate Agent and Disclaimer voices for each language under **Channels > Voice > Agent Voice**.
- **Auto voice switching** — the agent automatically switches voices when the conversation language changes to a configured language.
- **Conditional content filtering** — use `<language:en>` / `<language:es>` tags to serve language-specific content within a single prompt.
- **Language variants on Managed Topics** — manage multilingual knowledge base content within a single agent using per-language topic variants.
- **Language-specific pronunciation rules** — pronunciation rules are now organized by language with separate collapsible cards per language.
- **Language dropdown in Agent Chat** — select a language from a dropdown to test your agent in different languages.
- **Language metadata in conversations** — language information is now visible in conversation review lists and detail pages.

**Why it matters:**
- Build and manage a single multilingual agent instead of maintaining separate projects per language.
- Built-in language detection and mid-conversation switching simplifies the user experience.
- No more workarounds in code — configure everything from the UI.

See [Multi-language](/agent-settings/multilingual) for setup instructions.

</Accordion>

<Accordion title="Translations" icon="language" id="translations">

A new **Translations** page under **Channels > Response Control** gives you fine-grained control over auto-translated content in multilingual projects.

**What's new:**
- **Translation cards** — create cards that store language-specific versions of content, each with a unique Translation Key.
- **Auto-translation on save** — when you create a card, all configured languages are auto-translated immediately.
- **Manual overrides** — edit any auto-translated text to fix awkward phrasing, cultural mismatches, or domain-specific terminology. Overridden translations are marked as "Manually Translated".
- **Insert anywhere** — reference translation cards via the action menu in greetings, disclaimers, behavior rules, prompts, delay control responses, and SMS templates.
- **Use in functions** — access translations programmatically with `conv.translations.tn_key` for hard-coded utterances that need language-specific versions.

**Why it matters:**
- Override only what needs fixing — if auto-translation is good enough, you don't need to touch it.
- Handle cultural nuances like formality levels, regional phrasing, and idioms that don't translate directly.
- Keep all translation overrides in one place instead of scattered across functions and prompts.

See [Translations](/response-control/translations) for setup instructions.

</Accordion>

<Accordion title="MCP integrations" icon="plug" id="mcp-integrations">

Agent Studio now supports [Model Context Protocol](https://modelcontextprotocol.io/) (MCP) integrations, letting you connect external tool servers to your agent directly from the UI.

**What's new:**
- A dedicated **MCP** tab is now available under **Configure > Integrations**.
- Add an MCP server by providing a URL, authentication details (Header, Query parameter, or OAuth), and an optional timeout.
- Agent Studio **auto-discovers** the tools available on the connected server and displays them in a configuration panel.
- **Toggle individual tools** on or off to control which functions are exposed to the agent, helping manage context window usage.
- Supports **Header**, **Query parameter**, and **OAuth** authentication methods using secrets from the Secrets Vault.

**Why it matters:**
- Connect your agent to external systems without writing custom functions or API configurations.
- Reuse MCP-compatible tools across multiple agents.
- Control exactly which tools are available to your agent to keep prompts focused and efficient.

<Note>
Projects that previously configured MCP through experimental config will continue to work — the old MCP tab is hidden once the feature is enabled, but existing servers keep running in the background. The old config only stops being used if you add a new MCP server through the new UI. See [MCP integrations](/integrations/mcp) for setup instructions.
</Note>
