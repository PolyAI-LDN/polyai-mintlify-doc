CSV imports allow you to bulk-create or update content in Agent Studio. This is useful for:

- Updating multiple Managed Topics at once
- Managing variants across multiple sites or locations
- Migrating content between agents or environments
- Keeping your knowledge base in sync with external tools

## How CSV imports work

1. **Export** your current data to get the correct format
2. **Edit** the CSV file in your spreadsheet application
3. **Import** the updated file back into Agent Studio
4. **Review** the diff to see what will change
5. **Confirm** to apply the changes

<Tip>
Always export first to ensure you have the correct column headers and format.
</Tip>

## Managed Topics CSV format

### Required columns

When importing Managed Topics, your CSV must include these columns:

| Column | Description | Required | Example |
|--------|-------------|----------|---------|
| `ID` | Unique identifier for the topic | Yes (for updates) | `topic_123` |
| `Title` | Topic name | Yes | `Store hours` |
| `Answer` | The response text | Yes | `We're open Monday to Friday, 9am to 5pm.` |
| `Sample questions` | Example user queries (pipe-separated) | No | `What are your hours?\|When are you open?\|What time do you close?` |
| `Active` | Whether the topic is active | No | `Y` or `N` |

### File format rules

- **Column headers**: Must match exactly as exported (case-sensitive)
- **IDs**: 
  - Leave existing IDs intact when editing topics
  - Leave ID blank or omit for new topics (system will auto-generate)
- **Sample questions**: Separate multiple questions with pipe character (`|`)
- **Active status**: Use `Y` for active, `N` for inactive
- **Encoding**: UTF-8
- **Line endings**: Any (Windows, Mac, or Unix)

### Example CSV

```csv
ID,Title,Answer,Sample questions,Active
topic_001,Store hours,We're open Monday to Friday from 9am to 5pm.,What are your hours?|When are you open?|What time do you close?,Y
topic_002,Return policy,You can return items within 30 days of purchase.,What's your return policy?|Can I return this?|How long do I have to return?,Y
,New topic,This is a new topic that will be created.,Sample question 1|Sample question 2,Y
```

## Variants CSV format

### Required columns

When importing variants, your CSV must include:

| Column | Description | Required | Example |
|--------|-------------|----------|---------|
| `variant_id` | Unique identifier | Yes (for updates) | `location_001` |
| `variant_name` | Display name | Yes | `Downtown Store` |
| Custom columns | Your variant-specific data | Varies | `phone_number`, `address`, etc. |

### File format rules

- **Column headers**: Must match your variant schema
- **IDs**: Leave intact for updates, blank for new variants
- **Custom columns**: Add any columns needed for your use case
- **Encoding**: UTF-8

### Example CSV

```csv
variant_id,variant_name,phone_number,address,opening_hours
var_001,Downtown Store,555-0100,123 Main St,9am-6pm Mon-Fri
var_002,Uptown Store,555-0200,456 Oak Ave,10am-8pm Mon-Sat
,New Location,555-0300,789 Pine Rd,9am-5pm Mon-Fri
```

## Best practices

### Before importing

1. **Export first**: Always start with an export to get the correct format
2. **Back up**: Keep a copy of your original export
3. **Test small**: Try importing 1-2 rows first to verify format
4. **Use Sandbox**: Import to Sandbox environment first

### During editing

1. **Don't change headers**: Column names must match exactly
2. **Preserve IDs**: Don't modify existing IDs unless you want to create new entries
3. **Check encoding**: Save as UTF-8 to preserve special characters
4. **Validate data**: Check for missing required fields before importing

### After importing

1. **Review the diff**: Check what will change before confirming
2. **Test thoroughly**: Use Agent Chat to verify changes
3. **Promote carefully**: Follow [version management](/learn/maintain/version-management) best practices

## Common issues

### Import fails with "Invalid format"

**Cause**: Column headers don't match expected format

**Solution**: Export a fresh CSV and copy your data into it

### Special characters appear as �

**Cause**: File encoding is not UTF-8

**Solution**: Save your CSV as UTF-8 in your spreadsheet app

### Changes don't appear after import

**Cause**: You're viewing a different environment

**Solution**: Check you're in the correct environment (Sandbox/Pre-release/Live)

### Duplicate topics created instead of updating

**Cause**: ID column is missing or empty for existing topics

**Solution**: Keep the ID column intact when editing existing topics

## File naming convention

Exported files follow this naming pattern:

```
{agentName}_{environment}_{timestamp}_{versionCode}_KB.csv
```

Example: `MyAgent_Sandbox_20250227_v123_KB.csv`

<Note>
Draft exports exclude the version code from the filename.
</Note>
