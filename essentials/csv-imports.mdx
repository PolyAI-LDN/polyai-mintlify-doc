CSV imports allow you to bulk update or create content in Agent Studio. You can export data, edit it in a spreadsheet, and reimport it to update your agent.

## What you can import/export

### Managed Topics

Export and import your knowledge base content as CSV files.

**Use cases:**
- Bulk editing of answers and questions
- Adding multiple topics at once
- Professional translation workflows
- Migrating content between agents or environments

### Variants

Export and import variant data for multi-site configurations.

**Use cases:**
- Updating contact numbers or opening hours across multiple locations
- Scaling configurations across sites
- Syncing with external tools
- Migrating variants between agents

## How to export

1. Navigate to the relevant section (Managed Topics or Variant Management)
2. Click **Export CSV**
3. Download the CSV file

**File naming convention for Managed Topics:**
```
{agentName}_{Env}_{Timestamp}_{versionCode}_KB.csv
```

Note: Draft exports exclude the version code.

## How to import

1. Edit your CSV file in a spreadsheet application
2. **Keep column headers exactly as exported**
3. For updates: Leave IDs intact for existing items
4. For new items: Add new rows (IDs will be auto-generated)
5. Save as CSV
6. Click **Import CSV** in Agent Studio
7. Review the diff showing what will change
8. Click **Confirm import**

## CSV format requirements

### General rules

- Keep all column headers exactly as they appear in the export
- Use UTF-8 encoding
- Preserve the structure of the exported file
- Don't modify ID columns for existing items

### Managed Topics columns

When you export Managed Topics, the CSV includes:

- **ID** - Unique identifier (leave intact for updates, blank for new topics)
- **Topic** - The topic name
- **Questions** - Sample questions that trigger this topic
- **Answer** - The response content
- **Active** - `Y` or `N` to activate/deactivate the topic
- Additional metadata columns

### Variant columns

When you export variants, the CSV includes variant-specific fields configured for your agent.

## Best practices

<AccordionGroup>
  <Accordion title="Test in Sandbox first">
    Always import to your Sandbox environment first to verify changes before promoting to Live.
  </Accordion>

  <Accordion title="Keep backups">
    Export and save a copy before making bulk changes so you can revert if needed.
  </Accordion>

  <Accordion title="Use the diff review">
    Carefully review the diff before confirming import to catch any unintended changes.
  </Accordion>

  <Accordion title="Professional translations">
    For high-quality translations:
    1. Export knowledge as CSV
    2. Send to translators with context and domain glossary
    3. Import translated CSV
    4. Review and test before publishing
  </Accordion>
</AccordionGroup>

## Common workflows

### Bulk editing 10-20 topics

**Time estimate:** 15 minutes

1. Export CSV
2. Edit in spreadsheet
3. Import CSV
4. Review diff
5. Confirm

### Bulk editing 50+ topics

**Time estimate:** 30 minutes

1. Export CSV
2. Make extensive edits in spreadsheet
3. Import CSV
4. Carefully review diff
5. Confirm

### Multi-site variant updates

1. Export variants from one agent
2. Update location-specific data (phone numbers, hours, etc.)
3. Import to other agents
4. Test in each environment

## Troubleshooting

<AccordionGroup>
  <Accordion title="Import fails with validation error">
    - Check that column headers match exactly
    - Verify data types (e.g., Active column should be Y or N)
    - Ensure required fields are not empty
    - Check for special characters that need escaping
  </Accordion>

  <Accordion title="Changes not appearing">
    - Verify you're in the correct environment
    - Check if topics are marked as Active (Y)
    - Ensure you clicked Confirm import after reviewing the diff
  </Accordion>

  <Accordion title="Duplicate topics created">
    - Make sure you kept the ID column intact for existing topics
    - Only leave ID blank for genuinely new topics
  </Accordion>
</AccordionGroup>
