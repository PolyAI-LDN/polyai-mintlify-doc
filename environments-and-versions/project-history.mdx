The **Project history** page (**Deployments > Project history**) provides a chronological record of all project events — including published versions, draft branch merges, and deleted drafts.

## Overview

Project history lets you review every event in your project's lifecycle: published versions, draft branch merges, and deleted drafts. Each entry shows who performed the action and when. Use it to audit changes, understand when specific updates went live, track branch sync activity, and compare versions side by side.

## Viewing history

1. Go to **Deployments > Project history** in the sidebar.
2. Versions appear in **reverse chronological order** (newest first).
3. Each entry shows the event type (published version, branch merge, or deleted draft), description, author, and timestamp.

## Comparing versions

Select any two versions to view a side-by-side diff of changes. This is useful for:

- Auditing what changed between deployments
- Reviewing changes before promoting to a new environment
- Diagnosing issues introduced in a specific version

For more details on version comparison, see [Compare versions](/environments-and-versions/diffs).
