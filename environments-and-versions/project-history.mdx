<Warning>
**Project history has been sunset (24 June 2026).** The standalone Project history page has been removed. Version history, including event type, author, timestamp, and version diffs, now lives directly on the [Deployments](/environments-and-versions/introduction) page. See [Compare versions](/environments-and-versions/diffs) for side-by-side diffs between any two versions.
</Warning>

The information below describes the previous Project history page and is preserved for reference only.

Use Project History to audit when changes were made, who made them, and what went into each version. When diagnosing regressions, you can identify exactly which version introduced a problem and roll back if needed. A complete audit trail supports compliance requirements and team accountability.

The **Project history** page was previously found under **Deployments > Project history**. Each entry showed the event type, author, and timestamp.

<img src="/images/environments-and-versions/project-history.png" alt="Project history page showing a list of project events including deployments and branch merges." />

## Viewing history

1. Go to **Deployments > Project history** in the sidebar.
2. Versions appear in **reverse chronological order** (newest first).
3. Each entry shows the event type (published version, branch merge, or deleted draft), description, author, and timestamp.

## Comparing versions

Select any two versions to view a side-by-side diff of changes.

Compare versions to:

- Audit what changed between deployments
- Review changes before promoting to a new environment
- Diagnose issues introduced in a specific version

For more details on version comparison, see [Compare versions](/environments-and-versions/diffs).
