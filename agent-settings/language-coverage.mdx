This page is the canonical reference for which languages PolyAI supports and which models cover them across LLM, speech recognition (ASR), and text-to-speech (TTS).

For setup and runtime behavior, see [Multi-language](/agent-settings/multilingual). For model selection, see [Model](/agent-settings/model-use) and [Raven](/agent-settings/raven).

<Note>
**Source of truth.** LLM model availability and channel support are read from `platform_ui` model metadata (`supports_voice`, `supports_webchat`) and the per-region `llm_gateway` model registry in `poly_core`. Raven language list reflects internal research team announcements. Provider lists reflect the implementations in `poly_core/src/agent_v3/nlu/asr_providers/` and `poly_core/src/poly_platform/llm_gateway/`. The 73-language response table is mirrored from `agent-settings/multilingual.mdx`, which was originally compiled from public docs rather than an engineering-authoritative list – flagged for confirmation by Research/Runtime.
</Note>

## LLM language coverage

### Raven (recommended)

[Raven](/agent-settings/raven) is purpose-built for customer service across voice and chat. Raven 3.5 supports the following 24 languages:

Arabic, Bulgarian, Cantonese, Croatian, Czech, Dutch, English, French, German, Greek, Hindi, Hindi (Romanized/Hinglish), Italian, Japanese, Korean, Mandarin (China), Mandarin (Taiwan), Polish, Portuguese (Brazil), Portuguese (Portugal), Serbian, Spanish (US), Swedish, Turkish.

**Strongest performance** relative to general-purpose models: Cantonese, Italian, Korean, Mandarin (China), Mandarin (Taiwan), Spanish (US).

You can keep prompts and knowledge in English and set the response language to your target language – Raven responds consistently in the target language. Quality improves further if you translate prompts and add examples in the target language.

<Warning>
**Known Raven gaps.** Raven does **not** cover Danish – use GPT-5 (or another third-party model that supports Danish) for Danish deployments. Other languages in the 73-language table that are absent from the Raven list above also require a third-party LLM. When in doubt, check with Research before committing to a Raven deployment for a language not listed here.
</Warning>

### Third-party LLMs

When you select an OpenAI or Amazon Bedrock model on the [Model](/agent-settings/model-use) page, language coverage matches that provider's official support. Use a third-party LLM when:

- Your target language is not on the Raven list above, or
- You need a capability only the third-party model provides.

### LLM models

The table below reflects the model gateway registry in `poly_core/src/poly_platform/llm_gateway/dynamic_configs/models.*.yaml` (US-1, EU-W-1, UK-1 production). Per-channel support is gated by `supports_voice` / `supports_webchat` metadata served by the model options API – the channel column reflects today's defaults but a deployment-specific override can change it.

| Provider | Model | Channel | Regions | Notes |
| --- | --- | --- | --- | --- |
| PolyAI | Raven 3.5 (`polywhirl-3-5`) | Voice + Chat | All | Recommended. Auto-reasoning, out-of-domain detection, custom style following, built-in safety. |
| PolyAI | Raven V3 (`polywhirl-v3`) | Voice | All | Legacy. Superseded by Raven 3.5. No chat support. |
| PolyAI | Raven V2 (`polywhirl-v2`) | Voice | All | Legacy. Retained for existing deployments. |
| PolyAI | Raven Omni (`raven-omni`) | End-to-end voice | Dev / Apollo only | Not yet generally available. End-to-end speech-to-speech model. |
| OpenAI (Azure / OpenAI direct) | GPT-5 | Voice + Chat | All | Strong reasoning. |
| OpenAI (Azure / OpenAI direct) | GPT-5 mini | Voice + Chat | All | Lower latency for mid-complexity workloads. |
| OpenAI (Azure / OpenAI direct) | GPT-5 nano | Voice + Chat | All | Fast, lightweight responses. |
| OpenAI (Azure / OpenAI direct) | GPT-5 chat | Chat | All | Optimized for extended dialogue. |
| OpenAI (Azure) | GPT-5.2 chat | Chat | All | Latest chat-optimized model. |
| OpenAI (Azure / OpenAI direct) | GPT-4.1 | Voice + Chat | All | Strong reasoning with improved cross-task performance. |
| OpenAI (Azure / OpenAI direct) | GPT-4.1 mini | Voice + Chat | All | Cost-effective, latency-focused. |
| OpenAI (Azure / OpenAI direct) | GPT-4.1 nano | Voice + Chat | All | Minimal compute, high throughput. |
| OpenAI (Azure / OpenAI direct) | GPT-4o | Voice + Chat | All | Balanced reasoning, speed, and cost. |
| OpenAI (Azure / OpenAI direct) | GPT-4o mini | Voice + Chat | All | High-volume / everyday workloads. |
| OpenAI | GPT realtime / realtime-mini | End-to-end voice | All | Speech-to-speech; behaves differently from text LLMs. |
| Amazon Bedrock | Claude Sonnet 4 (`aws-claude-4-sonnet`) | Voice + Chat | US-1, EU-W-1, UK-1 | Strong reasoning and safety alignment. |
| Amazon Bedrock | Claude 3.5 Haiku (`aws-claude-35-haiku`) | Voice + Chat | US-1 only | Predictable tasks with strong safety alignment. Not currently configured in EU-W-1 / UK-1. |

<Note>
**Channel column caveats.** "Voice + Chat" means the model is generally available for both channels via the model metadata flags. Some accounts disable specific models per channel, and end-to-end voice models (`gpt-realtime`, `raven-omni`) are filtered out of the traditional model selector by `filterTraditionalModels` in `platform_ui`. Confirm with Runtime / Research before relying on this column for a new launch.
</Note>

## Choosing a language and model

1. **Pick the response language** from the table above using its BCP 47 code.
2. **Check Raven coverage** – if your language is on the 24-language Raven list, Raven 3.5 is the recommended LLM.
3. **If Raven does not cover it** (for example Danish), select a third-party LLM in [Voice configuration](/voice/voice-configuration) or [Chat configuration](/webchat/chat-configuration). Verify the chosen model officially supports the language.
4. **Confirm regional availability** for Bedrock models if you are deploying outside US-1.
5. **Confirm a voice exists** for the language in the [Voice library](/voice/voice-library). Prefer native voices over multilingual fallbacks.
6. **Configure ASR** – defaults usually work; for domain terms see [Speech recognition](/speech-recognition/introduction) and ASR biasing.

## Maintaining this page

This page is the canonical reference. When language or model coverage changes:

- **Raven language added / removed** → update the Raven list above and `agent-settings/raven.mdx`, cite the Research announcement.
- **New response language** → add a row to the full language table above and update `agent-settings/multilingual.mdx`.
- **New / removed LLM model** → check `poly_core/src/poly_platform/llm_gateway/dynamic_configs/models.*.yaml`, update the model table above, and update `agent-settings/model-use.mdx`.
- **Channel / regional availability change** → check `platform_ui` model metadata (`supports_voice`, `supports_webchat`) and the per-region gateway YAMLs.
- **New ASR or TTS provider** → confirm against `poly_core/src/agent_v3/nlu/asr_providers/` (ASR) and TTS provider classes, then update the lists above and `glossary/architecture.mdx`.
- **Deprecation** → mark the row and link to the release note in [Release notes](/releases).
