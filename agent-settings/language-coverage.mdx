This page is the single source of truth for which languages PolyAI supports and which models cover them. Use it to confirm what you can ship today across LLM, speech recognition (ASR), and text-to-speech (TTS), and how language coverage varies by model.

For setup and runtime behavior, see [Multi-language](/agent-settings/multilingual). For model selection, see [Model](/agent-settings/model-use) and [Raven](/agent-settings/raven).

## At a glance

| Layer | Coverage | Notes |
| --- | --- | --- |
| LLM (response language) | 73 languages | Full union across Raven and supported third-party LLMs. |
| Raven (recommended LLM) | 24+ languages | Conversation-tuned, near-perfect language consistency. |
| ASR (speech-to-text) | Provider-dependent | Routed automatically per language with fallback. |
| TTS (text-to-speech) | Provider-dependent | Voice availability varies by provider and language. |

## LLM language coverage

### Raven (recommended)

[Raven](/agent-settings/raven) is purpose-built for customer service across voice and chat. Raven 3.5 supports the following languages:

Arabic, Bulgarian, Cantonese, Croatian, Czech, Dutch, English, French, German, Greek, Hindi, Hindi (Romanized/Hinglish), Italian, Japanese, Korean, Mandarin (China), Mandarin (Taiwan), Polish, Portuguese (Brazil), Portuguese (Portugal), Serbian, Spanish (US), Swedish, Turkish.

**Strongest performance** relative to general-purpose models: Cantonese, Italian, Korean, Mandarin (China), Mandarin (Taiwan), Spanish (US).

You can keep prompts and knowledge in English and set the response language to your target language – Raven responds consistently in the target language. Quality improves further if you translate prompts and add examples in the target language.

### Third-party LLMs

When you select an OpenAI or Amazon Bedrock model on the [Model](/agent-settings/model-use) page, language coverage matches that provider's official support. PolyAI exposes the full union of those languages – 73 in total – so any language in the table below works with a supported third-party model even if Raven does not cover it.

## Choosing a language and model

1. **Pick the response language** from the table above using its BCP 47 code.
2. **Check Raven coverage** – if your language is on the Raven list, Raven 3.5 is the recommended LLM.
3. **If Raven does not cover it**, select a third-party LLM in [Voice configuration](/voice/voice-configuration) or [Chat configuration](/webchat/chat-configuration).
4. **Confirm a voice exists** for the language in the [Voice library](/voice/voice-library). Prefer native voices over multilingual fallbacks.
5. **Configure ASR** – defaults usually work, but for domain-specific terms see [Speech recognition](/speech-recognition/introduction) and [ASR biasing](/speech-recognition/introduction).

## Maintaining this page

This page is the canonical reference. When language or model coverage changes:

- **New Raven language** → update the Raven list above and `agent-settings/raven.mdx`.
- **New response language** → add a row to the full language table above and update `agent-settings/multilingual.mdx`.
- **New LLM model** → add a row to the LLM model table above and update `agent-settings/model-use.mdx`.
- **New ASR or TTS provider** → add to the provider lists above and update `glossary/architecture.mdx`.
- **Deprecation** → mark the row and link to the release note in [Release notes](/releases).
