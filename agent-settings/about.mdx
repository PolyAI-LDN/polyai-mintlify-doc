Configure your agent's greeting, personality, and role in **Build > Agent > About**.

## Greeting

The agent's opening line — the first thing a user hears (voice) or sees (chat). The greeting goes directly to text-to-speech without LLM processing (see [conversation flow](/essentials/order)), so it needs to be exactly right.

You can include [function calls](/function/introduction) to add dynamic content (e.g., time-based greetings) and [variant calls](/variant-management/introduction) to customize greetings per location.

**Example:**
> "Thank you for calling Acme Hotels. How can I help you today?"

## Personality

A free-text description that sets the tone and communication style for your agent. The personality is provided to the LLM as context for generating responses.

**Example values:**
- "Friendly and conversational, uses casual language"
- "Professional and concise, avoids small talk"
- "Empathetic and patient, especially with frustrated callers"

## Role

A description of the agent's job function. The role gives the LLM context about what the agent is responsible for. Use [Rules](./rules) to enforce specific behavioral constraints beyond what the role description provides.

**Example values:**
- "Front desk receptionist for a hotel chain"
- "Customer service representative for a telecommunications company"
- "Technical support agent for a SaaS product"
