---
title: Agent behavior best practices
description: Structure your agent behavior prompt for consistent, reliable conversations.
---

The behavior prompt defines how your agent interacts, responds, and manages conversations. A well-structured behavior prompt ensures consistent, helpful interactions while addressing the challenges of voice communication.

## Recommended structure

Organize your behavior prompt into these sections:

### 1. Task and context

Establish the agent's identity and functional scope:

```text
Your task is to assist users with their queries about [Organization].
[Organization] is a [brief description] that specializes in [services].
You have the ability to [list of capabilities].
```

Include tool usage instructions:

```text
You have the ability to call functions when explicitly instructed.
Always execute tool calls properly when instructed. Do not output
tool calls as text. In a given turn, output either a tool call or
text — never both.
```

<Warning>
Prompting for only one of tool call or text per turn is critical. Returning both leads to worse performance.
</Warning>

### 2. Conversational style

Keep answers short and conversational. Specify formatting conventions for voice:

```text
Keep your answers short and conversational. Always be polite but assertive.
Format responses as natural conversational paragraphs rather than lists.
When reading phone numbers, convert digits to words.
Do not ask more than one question in a single turn.
```

### 3. Special case handling

**Out of scope queries:** Acknowledge limitations and offer to transfer.

**ASR mistranscriptions:** Use a graduated approach — ask the user to repeat up to 2-3 times before transferring to a human agent.

**Jailbreak attempts:** Redirect firmly but professionally to the agent's intended purpose.

### 4. Smalltalk

Define concise responses for common social interactions:

```text
- <the user says hello> "Hi! How can I help you today?"
- <the user asks how you are> "I'm doing great, thanks! How can I help?"
- <the user asks if you can hear them> "I can hear you loud and clear.
  What can I do for you today?"
```

### 5. Silence handling

<Info>
Agent Studio has a default silence prompt that handles repetitions automatically. You may not need silence handling in your behavior prompt, but you should handle silence-triggered hangups.
</Info>

```text
- <silence after a question> Rephrase the last question
- <silence after a statement> "Is there anything else I can do for you?"
- <continued silence> "Hi, are you still on the line?"
- <prolonged silence> Call the end_call function
```

### 6. Call transfer and deflection

Define when to transfer and how to handle early transfer requests:

- **Start of call:** Attempt to deflect — the user may not know the agent's capabilities
- **Later in call:** Transfer immediately — the user likely has a specific need

```text
# At the start of the call
ASSISTANT: "Hi! How can I help you today?"
USER: "Speak to someone"
ASSISTANT: "I might be able to help you myself. Could you tell me
what you need?"
USER: "No, speak to someone"
ACTION: Call transfer_call with destination="DEFAULT", reason="SPEAK_TO"
```

### 7. Goodbye handling

Use the `end_call` function to control goodbye behavior:

```python
def end_call(conv: Conversation):
    # Optionally transition to a CSAT flow first
    if not conv.state.csat_flow_in_history:
        conv.goto_flow("csat")
        conv.state.csat_flow_in_history = True
        return "Before ending the call, move on to the CSAT survey first."

    return {
        "utterance": "Ok. I hope you have a great rest of your day. Goodbye!",
        "hangup": True
    }
```

### 8. Backout behavior

Allow users to exit flows they didn't intend to start:

```text
If you've misunderstood the user's intent and started a flow they
don't want, and they indicate they want to stop, immediately call
the backout function to exit the flow.

Examples:
- "No, I'm not rescheduling."
- "I don't want to make a booking."
- "Stop, that's not what I wanted."
```

### 9. Dynamic information

Use `$variable` syntax to insert information that changes per conversation:

```text
Today is $formatted_date_time.

"Next [day]" refers to that day in the following week.
"This [day]" or just "[day]" refers to the upcoming instance.
```

<Tip>
Place variable information at the end of the prompt to facilitate efficient caching. Only the dynamic portions need updating each conversation.
</Tip>
