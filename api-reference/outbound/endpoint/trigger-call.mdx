---
title: 'Trigger an outbound call'
api: 'POST /v1/agents/{project_id}/telephony/outbound-calls'
description: 'Initiate an outbound call to a phone number'
---

Triggers a new outbound call to the specified phone number. When the request is accepted, the call is placed close to immediately.

The API should only be called to make a call. The call is always attempted, and the [start tool](/tools/start-tool) (`start_function`) is executed **after** the call connects.

<Warning>
It is your responsibility to ensure that calls are placed in compliance with applicable laws and regulations.
</Warning>

## Request

### Headers

<ParamField header="x-api-key" type="string" required>
  API key provisioned by your PolyAI representative
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

### Path parameters

<ParamField path="project_id" type="string" required>
  The project identifier. Found in your Agent Studio URL: `https://studio.{region}.poly.ai/{account_id}/{project_id}/...`
</ParamField>

### Body

<ParamField body="toNumber" type="string" required>
  Phone number to call in E.164 format (e.g., `+442012345678`). If the number lacks a country prefix, also supply `countryCode`.
</ParamField>

<ParamField body="environment" type="string" required>
  Target environment for the call. One of:
  - `sandbox`
  - `pre-release`
  - `live`
</ParamField>

<ParamField body="countryCode" type="string">
  ISO 3166-1 alpha-2 country code (e.g., `GB`, `US`) used to parse a non-E.164 `toNumber`. Leave unset when `toNumber` is already in E.164 format.
</ParamField>

<ParamField body="variantId" type="string">
  Variant ID to use for the call. If omitted, the default connector for the environment is used.
</ParamField>

<ParamField body="encryption" type="string" default="tls_srtp">
  Telephony encryption mode for the outbound leg. Defaults to `tls_srtp`.
</ParamField>

<ParamField body="metadata" type="object">
  Arbitrary key/value pairs to attach to the call. **All values must be strings.** The base64 representation of the metadata must be less than 26 KB.

  Each key is delivered to the agent as a SIP header and can be read inside `start_function` (or any tool) with:

  ```python
  conv.sip_headers.get('key_name')
  ```
</ParamField>

## Response

<ResponseField name="callSid" type="string">
  Unique identifier for the triggered call. Always prefixed with `OUT-`. Use this to check call status.
</ResponseField>

<ResponseField name="status" type="string">
  Initial call status. Will be `QUEUED` on success.
</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl -X POST "https://api.us.poly.ai/v1/agents/PROJECT-xxx/telephony/outbound-calls" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "toNumber": "+16176451984",
    "environment": "live",
    "metadata": {
      "patient_first_name": "Paul",
      "patient_last_name": "Smith",
      "patient_dob": "1981-August-26",
      "referred_by_name": "Dr. Alex Deerman"
    }
  }'
```

```python Python
import requests

url = "https://api.us.poly.ai/v1/agents/PROJECT-xxx/telephony/outbound-calls"
headers = {
    "x-api-key": "YOUR_API_KEY",
    "Content-Type": "application/json",
}
payload = {
    "toNumber": "+16175551212",
    "environment": "live",
    "metadata": {
        "patient_first_name": "Paul",
        "patient_last_name": "Smith",
        "patient_dob": "1981-August-26",
        "referred_by_name": "Dr. Alex Deerman",
    },
}

response = requests.post(url, json=payload, headers=headers)
data = response.json()
print(f"Call triggered: {data['callSid']}")
```

```javascript JavaScript
const response = await fetch(
  'https://api.us.poly.ai/v1/agents/PROJECT-xxx/telephony/outbound-calls',
  {
    method: 'POST',
    headers: {
      'x-api-key': 'YOUR_API_KEY',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      toNumber: '+16175551212',
      environment: 'live',
      metadata: {
        patient_first_name: 'Paul',
        patient_last_name: 'Glynn',
        patient_dob: '1981-August-26',
      },
    }),
  }
);

const data = await response.json();
console.log('Call triggered:', data.callSid);
```

</CodeGroup>

## Error responses

<ResponseField name="400 Bad Request">
  Invalid request parameters — for example, an unparseable `toNumber`, an invalid `environment`, a metadata value that isn't a string, or metadata that exceeds the 26 KB base64 size limit.
</ResponseField>

<ResponseField name="401 Unauthorized">
  Invalid or missing API key.
</ResponseField>

<ResponseField name="404 Not Found">
  No connector found for the specified environment, or connector token could not be resolved.
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Failed to place the outbound call. Retry with exponential backoff.
</ResponseField>

## Notes

- Phone numbers must be in E.164 format unless you also pass `countryCode`.
- All `metadata` values must be strings; nested objects, numbers, and booleans are rejected.
- The `environment` field is required and must be one of `sandbox`, `pre-release`, or `live`.
- Use the [Get call status](/api-reference/outbound/endpoint/get-call-status) endpoint to monitor call progress.
- Call status data is retained for approximately 2 hours after the call ends.
