Track all PolyAI API and feature deprecations in one place. Each entry names the affected feature, its replacement, and the cutover deadline.

## Active deprecations

### Conversations API v1

| Field | Detail |
|---|---|
| **Affected** | Conversations API v1 — `GET /v1/{account_id}/{project_id}/conversations` |
| **Replacement** | [Conversations API v3](/api-reference/conversations/v3/endpoint/get-conversations) |
| **Announced** | January 2026 |
| **Soft deprecation** | 2 March 2026 — best-effort support, no SLA guarantees |
| **End of support** | 31 August 2026 — v1 remains available but unsupported |

**What changed:** v3 runs on PolyAI's event-sourced data platform with improved reliability at scale, additional turn-level fields (`latency`, `translated_user_input`, `english_agent_response`), consistent ISO8601 timestamps, and empty strings instead of `null` for empty text fields.

**Migration guide:** [Migrate from v1 to v3](/api-reference/conversations/migrate-v1-to-v3)

<Warning>
If you are still on v1, plan your migration before 31 August 2026. After that date, v1 will receive no bug fixes or maintenance.
</Warning>

### Tool variable deprecations

| Field | Detail |
|---|---|
| **Affected** | `handoff_reason` and `handoff_number` tool variables |
| **Replacement** | [`handoff` object](/tools/variables) |
| **Status** | Deprecated — use the `handoff` object instead |

**What changed:** The standalone `handoff_reason` and `handoff_number` variables have been replaced by the structured `handoff` object. Existing references continue to work but should be migrated.

## How deprecations work

PolyAI follows a structured deprecation process:

<Steps>
  <Step title="Announcement">
    The deprecation is announced in [release notes](/releases/overview) and added to this page with a timeline.
  </Step>
  <Step title="Soft deprecation">
    The feature moves to best-effort support. It still works, but SLA guarantees no longer apply. This is the signal to start migration.
  </Step>
  <Step title="End of support">
    The feature receives no further bug fixes or maintenance. It may continue to work, but PolyAI makes no guarantees.
  </Step>
  <Step title="Removal (if applicable)">
    The feature is removed entirely. This is announced separately with at least 90 days' notice after end-of-support.
  </Step>
</Steps>
