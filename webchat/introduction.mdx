## Getting started

Setting up webchat involves two main steps:

<Steps>

<Step title="Enable chat">
Go to **Channels > Chat > Chat Configuration** and toggle **Enable chat** to on. Configure your greeting message and disclaimer settings.
</Step>

<Step title="Configure and deploy the widget">
Go to **Channels > Chat > Widget** to style the widget appearance, then generate a script tag to embed on your website.
</Step>

</Steps>

## Key features

### Separate configuration pages

Chat configuration and widget settings are managed on separate pages, allowing you to:

- Enable/disable chat independently from widget styling
- Push widget changes directly to specific environments
- Test configuration changes before going live

### Markdown support

Chat messages support formatted markdown:

- **Bold** and *italics*
- Bulleted and numbered lists
- [Links](https://example.com)
- `Code blocks`

Formatted markdown renders in both the **Test Agent Chat** panel and **Conversation Review**.

### Variant support

Select which [variant](/variant-management/introduction) the webchat connects to using the variant dropdown. This allows location-specific or configuration-specific chat experiences.

### Environment alignment

Chat configuration follows the same [branching behavior](/environments-and-versions/introduction) as other project settings. Test changes in Sandbox before promoting to Pre-release and Live.
