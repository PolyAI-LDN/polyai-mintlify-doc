The **Chat Configuration** page (found under **Channels > Chat** in the sidebar) controls whether your agent responds to chat messages and how it behaves in chat conversations.

## Enabling chat

To enable chat for your project:

1. Navigate to **Channels > Chat > Chat Configuration** in the sidebar.
2. Toggle **Enable chat** to on.

<Warning>
Chat must be enabled before you can configure the widget or deploy to any environment. If chat is disabled, the widget will not function.
</Warning>

## Chat greeting

Configure what your agent says when a chat session begins.

<ParamField path="Greeting message" type="string">
  The first message your agent sends when a user opens the chat widget.
  
  **Example:** "Hi there! I'm here to help. How can I assist you today?"
</ParamField>

<Tip>
Keep your greeting short and welcoming. Avoid long introductions that push the conversation off-screen.
</Tip>

## Chat disclaimer

Display a disclaimer message before users can start chatting. This is useful for legal transparency and data handling notices.

<ParamField path="Disclaimer message" type="string">
  The message shown to users before they begin chatting.
  
  **Example:** "This is an AI-powered assistant. All conversations are recorded for quality and training purposes."
</ParamField>

<ParamField path="Require consent" type="boolean">
  When enabled, users must click "I consent and start chat" before the conversation begins.
</ParamField>

## Branching behavior

Chat configuration aligns with [environment branching](/environments-and-versions/introduction). Changes made in Sandbox can be promoted to Pre-release and Live environments, ensuring consistent testing and deployment workflows.

## Markdown support

Chat messages support formatted markdown rendering:

- **Bold** and *italics*
- Bulleted and numbered lists
- [Links](https://example.com)
- `Code blocks`

Formatted markdown appears in both the **Test Agent Chat** panel and **Conversation Review**.
