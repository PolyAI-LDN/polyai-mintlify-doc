The **Widget** configuration page (found under **Channels > Chat** in the sidebar) controls how the chat widget looks and behaves when embedded on your website.

<Note>
You must [enable chat](/webchat/chat-configuration) before configuring the widget. Changes to widget configuration can be pushed directly to environments.
</Note>

## Styling

Use the live preview to adjust how the widget looks. Changes appear in real-time on the right side of the screen.

### Header text

The title shown in the widget header. Use your brand name or assistant name to help users identify who they're chatting with.

<ParamField path="Header text" type="string">
  **Example:** "PolyAI Assistant"
</ParamField>

### Primary color

Sets the color for the header and action elements. Enter a hex code or use the color picker.

<ParamField path="Primary color" type="string">
  **Example:** `#161617`
</ParamField>

<Tip>
[Check color contrast](https://webaim.org/resources/contrastchecker/) to ensure text remains readable against your chosen color.
</Tip>

![Styling tab with live preview](/images/webchat/main.png)

## Content

Configure what users see before a chat session begins.

### Disclaimer message

Provide a message explaining how chats are recorded and processed. This helps meet data-handling and legal transparency requirements.

<ParamField path="Disclaimer message" type="string">
  **Example:** "This is an AI-powered system and responses should be verified. All chats are recorded in order to respond to your query and for training and quality control purposes."
</ParamField>

### Consent button

<ParamField path="Enable consent button" type="boolean">
  When enabled, users must click "I consent and start chat" before chatting.
</ParamField>

![Webchat content tab showing disclaimer toggle and preview](/images/webchat/content-panel.png)

### Company policies

Add clickable links to your Privacy Policy and Terms & Conditions. These appear below the consent button.

<ParamField path="Privacy policy URL" type="string">
  **Example:** `https://example.com/privacy-policy`
</ParamField>

<ParamField path="Terms & conditions URL" type="string">
  **Example:** `https://example.com/terms-conditions`
</ParamField>

## Test and deploy

### Preview demo

Test the widget directly inside Agent Studio without embedding anything:

1. Click **Preview Demo** to open the chat in an inline iFrame.
2. Interact with the widget to verify greetings, styling, and consent flow.
3. The floating launcher (bottom-right) opens and closes the chat window.

![Preview Demo panel](/images/webchat/preview-test-panel.png)

### Generate script tag

Deploy the widget to your website:

1. Select an [environment](/environments-and-versions/introduction): **Sandbox**, **Pre-release**, or **Live**.
2. Enter your **Website URL**.
3. Click **Generate script tag**.
4. Copy and paste the script before the closing `</body>` tag on your site.

![Script tag generation panel](/images/webchat/script-tag.png)

<Warning>
If you update widget settings, you must regenerate and redeploy the script tag for changes to take effect.
</Warning>

## Variant selection

Use the **variant dropdown** in the top-right of the preview to select which [variant](/variant-management/introduction) the webchat connects to.

Both **Preview Demo** and the **generated script tag** reflect the variant you select.

![Variant dropdown](/images/webchat/variant-dropdown.png)

## Environment deployment

Widget changes can be pushed directly to specific environments:

- **Sandbox** - For development and testing
- **Pre-release** - For user acceptance testing (UAT)
- **Live** - For production deployment

This allows you to test widget changes independently before deploying to production.
