Create and deploy chat widgets in **Channels > Chat**. [Configure chat](/webchat/chat-configuration) first.

## Prerequisites

Before installing the widget, ensure you have:

- The web domain where you'll embed the agent
- Access to your website's HTML templates _or_ your tag manager (e.g., Google Tag Manager)
- [Chat configuration](/webchat/chat-configuration) completed in Agent Studio

## Creating a widget

If no widgets exist, you'll see an empty state with a **Create chat widget** button.

Click **Create chat widget** to open the creation modal:

<ParamField path="Widget name" type="string" required>
  A name to identify this widget (e.g., "Main website", "Support portal").
</ParamField>

<ParamField path="Website URL" type="string" required>
  The URL where this widget will be deployed. The widget will only load on this domain.
</ParamField>

<ParamField path="Variant" type="dropdown" required>
  Select which [variant](/variant-management/introduction) this widget connects to.
</ParamField>

<ParamField path="Environment" type="dropdown" required>
  Select the deployment environment: Sandbox, Pre-release, or Live.
</ParamField>

<Tip>
You can create multiple widgets for different websites or variants, each with unique styling and configuration.
</Tip>

## Widget editor

After creating a widget, use the editor to configure its appearance and behavior. The editor has three tabs: **Styling**, **Content**, and **Embed**.

A live preview panel on the right shows your changes in real-time.

### Header controls

- **Widget selector** - Dropdown to switch between widgets
- **Settings icon** - Access widget settings
- **Save and publish** - Deploy changes to the selected environment

## Styling tab

Customize the visual appearance of your widget.

### Header

<ParamField path="Header text" type="string">
  The title shown in the widget header. Use your brand name or assistant name.
  
  **Example:** "PolyAI Assistant"
</ParamField>

<ParamField path="Header image" type="file">
  Upload a logo or image for the widget header.
</ParamField>

### Agent

<ParamField path="Agent name" type="string">
  The name displayed for the agent in chat messages.
  
  **Example:** "Support Assistant"
</ParamField>

<ParamField path="Agent image" type="file">
  Upload an avatar image for the agent.
</ParamField>

### Color

<ParamField path="Primary color" type="string">
  Sets the color for the header and action elements. Enter a hex code or use the color picker.
  
  **Example:** `#161617`
</ParamField>

<Tip>
[Check color contrast](https://webaim.org/resources/contrastchecker/) to ensure text remains readable against your chosen color.
</Tip>

## Content tab

Configure disclaimers and consent requirements.

### Disclaimers

<ParamField path="Enable disclaimers" type="boolean">
  Toggle to show a disclaimer message before users can start chatting.
</ParamField>

<ParamField path="Disclaimer message" type="string">
  The message shown to users before they begin chatting.
  
  **Example:** "This is an AI-powered system. All chats are recorded for quality and training purposes."
</ParamField>

<ParamField path="Enable consent button" type="boolean">
  When enabled, users must click "I consent and start chat" before the conversation begins.
</ParamField>

<ParamField path="Reset to default" type="button">
  Restore the disclaimer message to its default text.
</ParamField>

### Company policies

<ParamField path="Enable company policies" type="boolean">
  Toggle to show links to your Privacy Policy and Terms & Conditions below the consent button.
</ParamField>

When enabled, add your policy URLs:

<ParamField path="Privacy policy URL" type="string">
  **Example:** `https://example.com/privacy-policy`
</ParamField>

<ParamField path="Terms & conditions URL" type="string">
  **Example:** `https://example.com/terms-conditions`
</ParamField>

## Embed tab

Deploy your widget to production.

### Instructions

The embed tab provides step-by-step instructions for adding the widget to your website.

### Widget status

Each widget displays:

- **Widget name** - The name you assigned
- **Status badge** - Shows "Live" when published, or the current state
- **Script tag** - The code to copy and embed on your site
- **Last published** - Timestamp of the most recent deployment

## Live preview

The preview panel on the right side of the editor shows:

- "You're previewing [widget name]" banner
- **Test widget** link to interact with the widget
- Real-time updates as you adjust settings

Click **Test widget** to open a functional preview and verify greetings, styling, and consent flow.

## Managing multiple widgets

You can create multiple widgets for different use cases:

- Different websites or domains
- Different variants for location-specific experiences
- Different environments for testing vs. production

Use the widget selector dropdown in the header to switch between widgets.
