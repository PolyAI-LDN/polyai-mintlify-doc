The Agent Development Kit (ADK) is a Python CLI that lets you manage Agent Studio projects from your local machine. Pull your project as YAML files, edit them with any tool, and push changes back — using a Git-like workflow.

## Why use ADK

ADK is designed for developers who want:

- **Bulk authoring** — generate topics, flows, or entities programmatically (e.g., from a CSV or script) and push them in one command
- **Version control** — YAML files commit cleanly to Git, giving you PR reviews, blame history, and rollback
- **CI/CD pipelines** — run `poly push` in GitHub Actions to automate agent configuration deployments
- **AI-assisted editing** — use coding assistants to edit YAML files directly, then push the result

## Installation

Install the ADK from PyPI:

```bash
pip install polyai-adk
```

## Authentication

You need an ADK API key to connect to Agent Studio. Generate one from your project's **Data access** page in Agent Studio.

## Getting started

### Initialize a project

Link a local directory to an Agent Studio project:

```bash
poly init
```

Follow the prompts to select your account and project.

### Pull your project

Download the current project state as YAML files:

```bash
poly pull
```

This creates a directory structure like:

```
topics/
  Frequently Asked Questions.yaml
  Billing Issues.yaml
flows/
  greeting/
    flow_config.yaml
    flow_step_welcome.yaml
agent_settings/
  role.yaml
  personality.yaml
  rules.yaml
entities/
  billing_amount.yaml
```

### Edit locally

Each YAML file represents a single resource. A topic file looks like:

```yaml
enabled: true
actions: ""
content: |
  We accept Visa, Mastercard, and PayPal.
  Refunds take 3–5 business days.
example_queries:
  - What payment methods do you accept?
  - How long do refunds take?
  - Can I pay with PayPal?
```

Edit files with any editor or IDE. Cross-references between resources use readable tags like `{{fn:check_billing}}` that ADK resolves to internal IDs at push time.

### Push changes

Send your local changes back to Agent Studio:

```bash
poly push
```

ADK automatically:
1. Computes a diff between your local files and the last-known remote state
2. Orders operations by dependency (e.g., creates entities before flows that reference them)
3. Sends all changes in a single batch
4. Creates a branch if pushing to `main` — it never writes directly to the main branch

## Commands

| Command | Description |
|---------|-------------|
| `poly init` | Link a local directory to an Agent Studio project |
| `poly pull` | Download the current project state as YAML files |
| `poly push` | Push local changes to Agent Studio |
| `poly docs` | Print documentation for any ADK resource type |

## What you can manage

ADK supports the full range of Agent Studio resources:

- **Knowledge base topics** — content, example queries, actions
- **Flows** — flow configurations, steps, and transitions
- **Entities** — entity definitions and values
- **Agent settings** — role, personality, rules, greeting
- **Functions** — start functions, end functions, and custom functions

## Branch workflow

ADK integrates with Agent Studio's branch system:

- `poly pull` pulls from your current branch
- `poly push` pushes to your current branch
- Pushing to `main` automatically creates a new branch for safety

## Best practices

<Tip>
Pull before editing to ensure you're working with the latest state. If two developers edit simultaneously, the last push wins.
</Tip>

- Keep YAML files in a Git repository for history and collaboration
- Use scripts to generate YAML from spreadsheets or databases for bulk operations
- Test in sandbox before pushing to pre-release or production branches
- Use `poly docs` to check the expected format for any resource type
