The Agent Development Kit (ADK) is a Python CLI and package that lets you build PolyAI agents from your local machine. Pull your project as YAML and Python files, edit with any tool, and push changes back — using a Git-like workflow.

ADK is designed for technical teams who need to build and deploy agents at scale: GSIs, ISVs, implementation partners, and engineering teams managing multiple projects.

## How it works

ADK uses a pull-edit-push workflow similar to Git:

<Steps>
  <Step title="Pull">
    Pull your Agent Studio project to your local machine as YAML configuration and Python function files.
  </Step>
  <Step title="Edit">
    Edit files with any tool — your IDE, scripts, or AI coding assistants. YAML defines agent configuration; Python defines functions and logic.
  </Step>
  <Step title="Push">
    Push changes back to Agent Studio. Your agent updates immediately in the target environment.
  </Step>
</Steps>

Agents built or updated via ADK are fully compatible with Agent Studio and the Agent Studio APIs. You can switch between surfaces at any time.

## Who it's for

- **GSIs and implementation partners** — deliver agent projects faster without depending on PS involvement for every deal
- **ISVs** — build and manage agents programmatically as part of your product
- **Technical teams** — use engineering workflows (Git, CI/CD, code review) for agent development
- **Anyone at scale** — when you need to create or update many agents or topics efficiently
