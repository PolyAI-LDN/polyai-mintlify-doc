**This section requires Python familiarity.** If you are a non-technical operator, share this page with your developer. For no-code alternatives, see [Managed Topics](/managed-topics/introduction) (actions) or [no-code flows](/flows/no-code/introduction).

Tools let your agent act, not just talk – look up a booking, call an API, validate input, or write to a CRM.

<Note>
**Tools and functions refer to the same thing.** The feature appears as **Tools** in Agent Studio navigation (**Build > Tools**) and in Conversation Diagnosis (**Tool calls** layer). In Python code, each tool is defined as a `function`, and related identifiers keep that name – `start_function`, `end_function`, `conv.functions`, `{"{{fn:…}}"}`, `FUNCTION_*` error codes, and so on. This documentation uses "tool" when referring to the feature in the UI and "function" when referring to the Python identifier or code-level concept.
</Note>

If you are looking for a tool that only exists inside a single [flow](/flows/introduction), visit the [transition functions](/flows/transition-functions) page.

Tools are Python functions used for deterministic logic and API integrations. Use [`conv.log`](./classes/conv-log) to write logs visible in **[Conversation review](/analytics/conversations/review) → [Diagnosis](/analytics/conversations/diagnosis)**.

## Best practices

- **Use descriptive names** – Action-oriented names like `book_reservation` or `send_notification` help the LLM understand when to call the tool.
- **Write clear descriptions** – Explain what the tool does, its parameters, and when it should be triggered.
- **Use meaningful parameter names** – `reservation_date` is clearer than `r_date`.
- **Control triggering** – Define specific [rules](/agent-settings/rules) to prevent over-triggering or under-triggering.
- **Test thoroughly** – Use [Test Cases and Test Sets](/analytics/test-suite/introduction) to validate tool behavior across scenarios.

## Troubleshooting

- **Tool not triggering** – Enable the **Tool calls** layer in **Conversation review → Diagnosis** to see whether the tool was considered. Refine the tool description or add clearer rules.
- **Wrong parameters** – Review `conv.log` entries to see what the LLM extracted. Adjust parameter descriptions or add validation.
- **Unexpected behavior** – Simplify the tool's logic and test in isolation before integrating with the agent.
