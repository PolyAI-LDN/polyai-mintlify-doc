Polyphone is a voice channel that runs in the browser. Visitors click a widget on your website, grant microphone access, and have a real-time voice conversation with your agent over WebRTC – no phone number, no dial-out.

Configure Polyphone from **Channels > Polyphone** in the sidebar.

<Note>
Polyphone is a way for customers to **reach** your voice agent. Configure how the agent **sounds** in [Channels > Voice](/voice/introduction). Both work together – Polyphone routes browser audio into the same agent, using the same TTS voice, that callers hear when they dial in.
</Note>

## Polyphone in the channel mix

| Channel | What it is | Where to configure |
| --- | --- | --- |
| **[Polyphone](/polyphone/introduction)** | Voice over WebRTC, embedded on your website | Channels > Polyphone |
| **[Numbers](/telephony/introduction)** | Voice over the phone network (PSTN/SIP) | Configure > Numbers |
| **[Chat](/webchat/introduction)** | Text chat embedded on your website | Channels > Chat |
| **[Voice](/voice/introduction)** | Your agent's TTS voice (used by Polyphone _and_ Numbers) | Channels > Voice |

Polyphone and phone numbers are two transports for the same voice agent – the agent runtime, voice settings, safety filters, and analytics are shared.

## When to use Polyphone

Use Polyphone when you want:

- A **voice-first experience** on your website (a "Talk to us" button rather than a phone number).
- **One-click calling** without asking visitors to dial.
- **Branded voice** that matches your site's look and feel.
- A widget that **reuses your existing voice agent** – no separate flows or configuration.

Run **Polyphone alongside phone numbers** when you want both web visitors and traditional callers to hit the same agent. Run **Polyphone alongside [Webchat](/webchat/introduction)** when you want both voice and text on your site.

## How it works

```
Visitor's browser                           PolyAI
─────────────────────                       ──────────────
Polyphone iframe (mic, autoplay)
   │
   ├── HTTPS  ──────► Iris script endpoint  (loads widget JS)
   │
   └── WSS    ──────► WebRTC gateway        (signaling + Opus audio)
                            │
                            └─► your voice agent
```

A single `<script>` tag on your site loads the widget. The script injects an iframe pinned to the bottom-right of the page. When a visitor clicks the launcher, the widget requests microphone access, opens a WebSocket to PolyAI's WebRTC gateway, and starts an Opus 48 kHz audio call with your agent.

Only **one Polyphone call can be active per browser** at a time. The widget enforces this with a cross-tab lock so a visitor with multiple tabs open cannot start parallel calls.

## Prerequisites

Before creating a Polyphone widget you'll need:

- An Agent Studio project with a **published variant**.
- A **voice agent** configured in [Channels > Voice](/voice/introduction) – Polyphone uses these TTS settings.
- The **website domain(s)** where the widget will be embedded (Polyphone only loads on domains you allow-list).
- Access to your website's HTML templates _or_ your tag manager.
- An **HTTPS** site – browsers require a secure context for microphone access.

Polyphone has **no dependency on Webchat**. You can ship a Polyphone-only deployment without configuring a chat agent.

## Browser support

Polyphone uses standard WebRTC primitives. Verified browsers:

| Browser | Minimum version |
| --- | --- |
| Chrome / Chromium | 72 |
| Firefox | 60 |
| Safari (macOS) | 14.1 |
| Safari (iOS) | 16 |
| Edge | 79 |

Visitors on unsupported browsers see a disabled call button with a brief explanation.

## Getting started

Setting up Polyphone involves three main steps:

<Steps>

<Step title="Configure the agent's voice">
Go to **Channels > Voice** and pick a TTS voice for the agent. Polyphone visitors will hear this voice. See [Voice introduction](/voice/introduction).
</Step>

<Step title="Create and style a Polyphone widget">
Go to **Channels > Polyphone > Widget** to:
- Create a new widget for your website
- Style the appearance (header, colors, agent avatar)
- Configure disclaimers and policy links
</Step>

<Step title="Embed and test">
Use the **Embed** tab to copy the script tag and paste it into your site. Use the **Test widget** link from the editor to verify the call experience before publishing.
</Step>

</Steps>
