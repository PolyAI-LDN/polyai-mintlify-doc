You've built something. Now you need someone to try it. There are three ways to do that on the Open platform, in order of effort.

## A link you can paste anywhere

Every agent gets a hosted page on PolyAI's domain. Send the URL, the recipient clicks it, their browser opens, they hit the call button. No login. No app. No phone number.

This is the fastest way to demo what you've built. Use it for:

- Sending to your team for a first reaction.
- Posting in a Slack channel for opt-in testing.
- Embedding in a sales deck or a support ticket.
- Anywhere a clickable URL goes.

To get the link, open your agent and go to **Deployment > Widget**. The shareable page link is on the **Test** tab. Copy it and send it.

[More on the shareable page →](/widgets/test)

## The embed on your own site

When you're ready for real visitors, drop PolyPhone into your site's HTML. The whole install is one script tag, generated for you under **Deployment > Widget > Embed**:

```html
<script src="https://messaging.poly.ai/widget/YOUR_UNIQUE_CODE.js"></script>
```

Paste it just before `</body>` on whichever page you want the launcher to show up on. Visitors see a small button in the bottom-right; clicking it opens a call.

Tweak the look, position, language, and consent text from **Deployment > Widget > Styling** and **Content**. You can run multiple variants per brand, region, or page.

[Full install reference →](/widgets/install)

## The PolyPhone API for custom triggers

If you want your own button to start the call (not the default launcher), the widget script exposes a `window.PolyphoneAPI` object. Use it to start a call from your existing UI, hand off context from your app, or react to call lifecycle events.

```js
// Start a call from a custom button
document.querySelector("#talk-to-us").addEventListener("click", () => {
  window.PolyphoneAPI.connectAndCall();
});

// Pass context to the agent mid-call
window.PolyphoneAPI.sendInfo("user_signed_in", { userId: "abc123" });
```

## What to do before you share

A few things worth checking before you send anyone a link:

- **Test it yourself first.** Click the share link in an incognito tab; that's what your reviewer will experience.
- **Set a clear welcome.** The first thing the agent says is the first impression. If it's still the default scaffold message, ask Studio Assistant to make it warmer or more specific.
- **Confirm the voice is the one you want.** Voice is set under **Configure > Agent voice**.
- **Check the consent text** if you've turned consent on. It's in **Deployment > Widget > Content**.

## Watching what happens

Every call placed through any of these channels lands in **Conversations**. Read the transcript, listen to the recording, see which topics matched and which flows ran. Then [iterate](/learn/iterate-open-platform): ask Studio Assistant to fix whatever went wrong.
