After [creating and configuring your Polyphone widget](/polyphone/widget), deploy it to your website using one of the methods below.

## Installation methods

Choose **one** installation method:

- **Option A: Direct HTML embed** – recommended for most sites.
- **Option B: Tag Manager** – if your organization manages scripts through Google Tag Manager or similar.

<Tip>
If you're unsure which route to use, start with **Option A**.
</Tip>

## Step 1: Generate a widget tag

1. Go to **Channels > Widgets** in Agent Studio.
2. Open the **Polyphone** tab.
3. Configure [Styling](/polyphone/widget#styling-tab) and [Content](/polyphone/widget#content-tab).
4. Click **Save and publish**.
5. Open the **Embed** tab.
6. Copy the unique script tag – it looks like:

```html
<script src="https://messaging.poly.ai/widget/YOUR_UNIQUE_CODE.js"></script>
```

The tag is bound to the widget's domain, variant, and environment. If you change any of those, re-publish and re-embed.

## Option A: Direct HTML embed

1. Open the HTML template that loads on **every page** (often called your global layout, base template, or `index.html`).
2. Paste the script tag **just before the closing `</body>` tag**.
3. Publish your website changes.

```html
<!DOCTYPE html>
<html>
  <head>
    <!-- Your existing head content -->
  </head>
  <body>
    <!-- Your page content -->

    <!-- Paste the Polyphone script here, just before </body> -->
    <script src="https://messaging.poly.ai/widget/YOUR_UNIQUE_CODE.js"></script>
  </body>
</html>
```

You only need to embed the script once. Updates to styling, content, or the agent take effect on the next page load – no re-embed required.

<Note>
The script tag must be served over **HTTPS** and the host page must be in a [secure context](https://developer.mozilla.org/en-US/docs/Web/Security/Secure_Contexts). Browsers block microphone access on insecure origins, so the call button will be disabled on `http://` pages.
</Note>

## Option B: Tag Manager

1. Open your tag manager (e.g., Google Tag Manager) and create a new **Custom HTML tag**.
2. Paste the script tag.
3. Set the trigger to **All Pages** (or restrict to the pages where you want the widget).
4. Publish the container.

The widget appears on the next page load.

## Re-embedding

You only need to re-embed when **the script tag itself changes**. The Embed tab in Agent Studio shows a **Snippet changed** banner if the published script differs from the version your team has on the site. Hand the new tag to your dev team or update your tag manager configuration.

You do **not** need to re-embed when you change:

- Branding (header text, logo, agent name, primary colour)
- CTA copy or disclaimer
- Policy links
- The agent or variant the widget connects to (within the same widget configuration)

## Verifying the install

1. Load the page where you embedded the script.
2. You should see the Polyphone launcher (a circular button with a phone icon) in the bottom-right corner.
3. Click the launcher – the widget expands.
4. Click **Start call** – your browser asks for microphone permission.
5. Allow the permission – the widget shows **Connecting…** and then **In call** when your agent answers.

If anything goes wrong, see [Troubleshooting](/polyphone/widget-troubleshooting).

## Cross-tab behaviour

If a visitor opens your site in multiple tabs, only **one Polyphone call can be active at a time**. When a call starts, the widget writes a heartbeat to `localStorage` and other tabs see the call button disabled with a "Call active in another tab" message. Closing the active call tab releases the lock within ~10 seconds.

This is enforced automatically – no configuration required.
